### Phase 112: NEXUS Agent Dispatch Deduplication

**Goal:** Prevent duplicate Linear issues from being created when retried Task tool calls fire
identical agent dispatches.

**Plans:** 1 (112-01)

**Outcome:**
- Added SHA256-based idempotency key generation to `pre-agent-dispatch.ps1` — 16-char hex
  prefix of `SHA256(subagent_type|description[:200])`.
- Dedup store persists at `.claude/scratchpad/dispatch-dedup.json` (auto-created, JSON,
  TTL-managed via `ConvertTo-Json`/`ConvertFrom-Json` — no external dependencies).
- Duplicate detection: same key within 30 min emits `DUPLICATE_DISPATCH` warning to stderr.
- Within-session pruning: entries older than 60 min removed on every dispatch.
- Session-start pruning: `session-start-protocol.ps1` silently clears entries older than
  120 min at each new session.
- Hook remains non-blocking — exits 0 always.

## Milestone Summary

### Key Decisions

- `force-dynamic` must not be added to root layout unless there are actual dynamic calls —
  removing it unblocked CDN caching for the entire site.
- `sendBeacon` preferred over `fetch` for web vitals reporting (fire-and-forget, survives
  page unload).
- 16-char SHA256 prefix sufficient for dedup key uniqueness within a session (2^64 space).
- Separate TTLs for dedup: 30-min warning, 60-min within-session prune, 120-min cross-session.
- CSP `unsafe-inline` removal deferred — requires Next.js 15 nonce infrastructure (separate phase).
- npm audit vulnerability fixes deferred — require dependency major upgrades outside v7.0 scope.
- SocialProof avatars use descriptive alt text (not aria-hidden) — avatars convey social proof
  context to screen reader users.
- Dashboard layout is `'use client'` — route segment config only works in server components;
  not needed anyway as auth/cookies make dashboard pages inherently dynamic.

### Issues Resolved

- Server module leaks in 3 client pages pulling `pg` (270KB) and `ioredis` (180KB) — ~990KB
  total client bundle reduction.
- 8 unbounded Prisma queries across 16 feature areas — added `take` limits and `select` fields.
- Entire site blocked from CDN caching due to misplaced `force-dynamic` on root layout.
- Broken `collectWebVitals` accumulation sending 5 zeroes + 1 real value per call.
- 12 v5.0 feature pages missing file-based Suspense/error boundaries.
- 10 v6.0/nested route pages missing file-based Suspense/error boundaries.
- 2 web-projects pages with no error handling (fetch/create/delete failures silent).
- 5 image alt text WCAG 1.1.1 gaps across bio, marketing, and optimized image components.
- Colour-contrast axe-core rule disabled globally in E2E accessibility spec.
- h1→h3 heading skip in dashboard settings branding tab.
- Login form inaccessible to screen readers on auth error (no aria-invalid/aria-describedby).
- Dead Express/Helmet security config still in TypeScript compilation path.
- NextAuth stub env vars in env-validator despite next-auth being uninstalled.
- Agent dispatch hook producing duplicate Linear issues on retried Task tool calls.

### Issues Deferred

- `pr/coverage/route.ts` Prisma `take` fix — not committable due to pre-existing gitignore
  `coverage/` pattern conflict.
- CSP `unsafe-inline` removal — requires Next.js 15 nonce infrastructure (separate phase).
- `dompurify` moderate XSS vulnerability — fix available, deferred to dependency upgrade cycle.
- `rollup` high path traversal via `@sentry/nextjs` — build-time only (not runtime), requires
  @sentry/nextjs v8→v10 upgrade (separate phase).
- `app/api/auth/resend-verification/route.ts` raw `error.message` exposure — low risk, deferred.
