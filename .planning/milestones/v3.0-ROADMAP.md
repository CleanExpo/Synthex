## Milestone Summary

**Key Decisions:**

- `app/page.tsx` is server component — child `'use client'` components work as client subtrees
- Dynamic OG images via Edge runtime (`app/api/og/route.tsx`) — accepts `?title=` param
- Billing emails use fire-and-forget Resend SDK with lazy singleton (`getResend()`)
- Static `/dashboard/billing` URL for billing portal — avoids async Stripe API call in webhook path
- Unlimited progress bars: check `!limit || limit <= 0` (covers DB 0 and -1)
- HTTP 404 from `/api/user/subscription` = free plan (not error state)
- Feature gate pattern: `{ error: '...', upgrade: true }` 403 response
- `subscription.current_period_end` in Stripe API `2025-07-30.basil` is on `subscription.items.data[0]`
- `credentials: 'include'` mandatory in all SWR fetchers (httpOnly auth-token cookie)
- CSP: remove `unsafe-eval` for production hardening

**Issues Resolved:**

- UNI-633: Free plan UI broken when /api/user/subscription returns 404
- UNI-634: "NaN%" progress bar when subscription limit is 0
- SYN-51: Zero-filled encryption keys security vulnerability
- Orphaned next.config.sentry.mjs causing confusion

**Issues Deferred:**

- 754 console.log statements — structured logger migration (now has logger.ts foundation)
- 14 components over 600 lines — decomposition backlog
- WCAG skip links (Phase 56 work deferred)

**Technical Debt Incurred:**

- NotificationBell.tsx: complex dual-fetch pattern not migrated to SWR (pre-launch risk)
- 25 orphaned API routes marked @internal/@deprecated — future cleanup in v3.1
- Phase 55-02 (inline state audit) deferred from v1.5 — still outstanding
