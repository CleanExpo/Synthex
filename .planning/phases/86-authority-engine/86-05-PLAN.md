Create 5 React components following existing dashboard patterns (glassmorphic styling, Tailwind CSS):

**AuthorityFeatureGate.tsx** — Wrapper that shows children if addon active, or UpgradePrompt if not:
- Props: `{ children: ReactNode; hasAddon: boolean; featureName: string }`
- If `!hasAddon`: render upgrade prompt card with addon-specific messaging ("Unlock Authority Engine — $22/month")
- If `hasAddon`: render children
- Use existing glassmorphic card styling (bg-surface-base/80, border-violet-500/10)

**AuthorityScoreCard.tsx** — Displays authority analysis scores:
- Props: `{ score: number; claimsFound: number; claimsVerified: number; claimsFailed: number; sourceBreakdown: Record<string, number> }`
- Large score number with tier badge (use getTier pattern from GEO page)
- Stats row: claims found, verified, failed
- Source breakdown: coloured pills for gov (emerald), academic (blue), industry (purple), web (gray)
- Glassmorphic card styling consistent with GEO dashboard

**ClaimValidationBadge.tsx** — Per-claim verified/unverified badge:
- Props: `{ claim: ValidatedClaim }`
- Green checkmark + "Verified" if verified, amber warning + "Unverified" if not
- Show confidence as subtle percentage
- Expandable: click to see source details (sources list with URLs)

**CitationList.tsx** — Generated citations display:
- Props: `{ citations: GeneratedCitation[]; format?: string }`
- Numbered list of formatted citations
- Copy-to-clipboard button per citation (use navigator.clipboard.writeText)
- "Copy All" button for the full footnote block
- Source type badges (government=emerald, academic=blue, industry=purple, web=gray)

**AuthoritySourcePanel.tsx** — Available sources + status:
- Props: `{ connectors: Array<{ id: string; name: string; type: string; enabled: boolean }> }`
- Grid of connector cards showing name, type, enabled/disabled status
- Green dot if enabled, gray dot if disabled
- Description of what each connector validates

All components use `'use client'` directive. Use existing Tailwind classes and Radix UI components (Badge, Card, CardContent, CardHeader, CardTitle from components/ui/).

AVOID: Don't create custom hooks in this task. Don't fetch data in components — they receive data via props. Don't use emojis in the UI.
