- Gate behind authority addon: `hasAuthorityAddon(userId)` check, 403 if not active with `{ error: '...', upgrade: true, addon: 'authority' }`

- Return DesignAuditResult as JSON

**DesignAuditCard.tsx**: Design quality score display.
- Props: `{ score: DesignQualityScore; issues: DesignIssue[] }`
- Large total score with tier badge
- Breakdown bars for each sub-score (headingHierarchy, mobileReadiness, etc.)
- Coloured progress bars: green >=80%, amber 50-79%, red <50% (normalised to sub-score max)
- Issues list below with severity icons (error=red, warning=amber, info=blue)
- Glassmorphic card styling, violet accent

**CROScoreCard.tsx**: CRO readiness score display.
- Props: `{ score: CROReadinessScore; issues: DesignIssue[] }`
- Total score with tier badge
- 5 breakdown bars for sub-scores
- Improvement suggestions based on lowest scores
- Glassmorphic card styling

**LLMCitationFitnessCard.tsx**: CITABLE framework compliance display.
- Props: `{ score: LLMCitationFitnessScore; issues: DesignIssue[] }`
- Total score with CITABLE acronym visual (each letter coloured by its sub-score)
- 7 dimension breakdown: C, I, T, A, B, L, E with labels and scores
- Visual: radar/spider chart alternative — just use horizontal bar chart for each dimension
- Highlight weakest dimensions with recommendations
- Glassmorphic card styling

All components use `'use client'` directive, Badge/Card/CardContent/CardHeader from components/ui/.
