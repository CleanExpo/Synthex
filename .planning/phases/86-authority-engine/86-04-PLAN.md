  <action>
Create 4 API routes following the existing route pattern (auth check, Zod validation, org scoping):

**app/api/authority/analyze/route.ts** (POST):
- Auth: require session via `getSession()`
- Zod schema: `{ content: z.string().min(50).max(50000), orgId: z.string(), deepValidation: z.boolean().optional() }`
- If `deepValidation` requested, check for authority addon (call `hasAuthorityAddon(userId)`) — if not active, return `{ error: 'Authority add-on required', upgrade: true, addon: 'authority' }` with 403
- Call `analyzeAuthority(content, { userId, orgId, deepValidation })`
- Return `AuthorityAnalyzeResponse`

**app/api/authority/citations/route.ts** (GET):
- Auth: require session
- Query params: `analysisId` (required)
- Fetch citations from `prisma.authorityCitation.findMany({ where: { analysisId } })`
- Verify the analysis belongs to the user (org scoping)
- Return citations array

**app/api/authority/sources/route.ts** (GET):
- Auth: require session
- Call `getConnectorStatus()` from source-connectors registry
- Return available connectors with enabled/disabled status

**app/api/authority/validate-claim/route.ts** (POST):
- Auth: require session + authority addon check
- Zod schema: `{ claim: z.string().min(10).max(1000) }`
- Search all connectors for this single claim
- Return SourceResult[] with validation status

All routes use `NextResponse.json()`, handle errors with try/catch, log via `logger`.

AVOID: Don't skip auth checks. Don't allow cross-org access. Don't expose raw error messages to clients.
