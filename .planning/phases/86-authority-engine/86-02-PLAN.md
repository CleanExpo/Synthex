  <name>Task 1: Create Semantic Scholar + Australian Government connectors</name>
  <files>lib/authority/source-connectors/semantic-scholar.ts, lib/authority/source-connectors/gov-au.ts</files>
  <action>
**semantic-scholar.ts**: Implement SourceConnector for Semantic Scholar API (214M academic papers).
- API endpoint: `https://api.semanticscholar.org/graph/v1/paper/search`
- Auth: env var `SEMANTIC_SCHOLAR_API_KEY` (optional — API works without key at reduced rate)
- Fields to request: `title,abstract,citationCount,year,authors,tldr,url,externalIds`
- Rate limit: 1 req/s (add 1s delay between requests if batching)
- `search(query)` method: search for papers matching the claim text, return top 5 results
- Confidence scoring: Papers with >50 citations from last 5 years = 0.9, >10 citations = 0.7, else 0.5
- Map to SourceResult: title, url (`https://api.semanticscholar.org/...`), snippet (abstract truncated to 200 chars), sourceType='academic', sourceName='Semantic Scholar'

**gov-au.ts**: Implement SourceConnector for Australian Government data APIs.
- data.gov.au: `https://data.gov.au/data/api/3/action/package_search?q={query}` (CKAN API, public, no auth)
- ABS (Australian Bureau of Statistics): `https://api.data.abs.gov.au/data/` (SDMX API — for statistical claims)
- `search(query)` method: query data.gov.au first, then ABS if statistical claim detected
- Government source = maximum confidence (0.95)
- Map to SourceResult: title, url, snippet, sourceType='government', sourceName='Australian Government' or 'ABS'

Both connectors must implement the `SourceConnector` interface from types.ts. Use `fetch()` directly (server-side). Handle network errors gracefully — return empty results, don't throw. Log errors via `logger.warn()`.

AVOID: Don't install new HTTP client packages. Don't throw on API failures — return empty results with a warning log.
  </action>
  <verify>npm run type-check passes, both files implement SourceConnector interface correctly</verify>
  <done>semantic-scholar.ts queries Semantic Scholar API and maps results to SourceResult[], gov-au.ts queries data.gov.au and ABS APIs, both handle errors gracefully</done>
</task>

<task type="auto">
  <name>Task 2: Create Industry Registry + Claude Web Search connectors</name>
  <files>lib/authority/source-connectors/industry-registry.ts, lib/authority/source-connectors/claude-web-search.ts</files>
  <action>
**industry-registry.ts**: Implement SourceConnector for industry-specific validation.
- ASIC (ABN Lookup): `https://abr.business.gov.au/abrxmlsearch/AbrXmlSearch.asmx/SearchByName?name={query}&maxResults=5` (public, no auth required for basic search)
- IICRC standards: Static lookup map of known restoration industry standards (S500, S520, S700) with URLs to official IICRC pages
- NCC: Static lookup for National Construction Code references
- `search(query)` method: check if query mentions ABN/company → try ASIC, check for IICRC/NCC standards → return static references
- Industry-verified = confidence 0.85
- Map to SourceResult: sourceType='industry', sourceName per registry

**claude-web-search.ts**: Implement SourceConnector using Claude API's built-in web_search tool.
- Uses existing `@anthropic-ai/sdk` package (already installed)
- Create Anthropic client with `ANTHROPIC_API_KEY` env var (already exists in project)
- Call Claude with `web_search_20250305` tool enabled
- System prompt: "You are a fact-checker. Search for authoritative sources that verify or refute the following claim. Focus on government (.gov), educational (.edu), and established news sources."
- `allowed_domains` parameter: `['*.gov.au', '*.edu.au', '*.org.au', 'scholar.google.com', '*.gov', '*.edu']`
- Parse Claude's response: extract `search_result` content blocks, map `cited_text` spans to SourceResult[]
- `search(query)` method: send claim to Claude with web_search, parse citation blocks
- Confidence based on source domain: .gov = 0.95, .edu = 0.85, .org = 0.75, other = 0.6

AVOID: Don't add new npm packages. Don't use MCP SDK — use Claude API directly via `@anthropic-ai/sdk`. Handle Anthropic API errors (rate limits, auth) gracefully with empty results + warning log.
  </action>
