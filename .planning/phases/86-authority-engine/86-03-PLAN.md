  <name>Task 1: Create claim extractor</name>
  <files>lib/authority/claim-extractor.ts</files>
  <action>
Create `claim-extractor.ts` that extracts verifiable claims from content text using NLP heuristics (no external AI call needed — rule-based extraction).

**Extraction rules** — a sentence is a verifiable claim if it contains:
1. **Statistical claims**: Numbers + context (percentages, dollar amounts, counts, dates with figures). Regex: `/\d+[\.,]?\d*\s*(%|percent|million|billion|thousand|AUD|\$)/i`
2. **Factual assertions**: Definitive statements ("is the largest", "was founded in", "according to"). Pattern match for definitive verbs + superlatives.
3. **Comparative claims**: "more than", "less than", "compared to", "versus", "leading", "fastest".
4. **Regulatory claims**: "compliant with", "certified by", "approved by", "meets standard", "AS/NZS", "ISO".
5. **Temporal claims**: "since 2020", "over 10 years", "established in", specific date references with assertions.
6. **Causal claims**: "causes", "results in", "leads to", "due to", "because of" + measurable outcome.

**Implementation**:
- `extractClaims(content: string): ExtractedClaim[]`
- Split content into sentences (handle abbreviations like "Dr.", "Mr.", "U.S.")
- Test each sentence against claim type patterns
- Extract entities mentioned in each claim (proper nouns via capitalisation heuristic)
- Assign confidence based on specificity (numbers + sources = 0.9, vague assertions = 0.4)
- Return sorted by confidence descending
- Limit to top 50 claims (avoid processing entire books)

AVOID: Don't call external APIs for extraction — this must be fast and free. Don't use regex that could catastrophically backtrack on long strings.
  </action>

  <done>extractClaims() extracts 6 claim types from content text using rule-based NLP, returns ExtractedClaim[] sorted by confidence</done>

<task type="auto">
  <name>Task 2: Create authority scorer and citation generator</name>
  <files>lib/authority/authority-scorer.ts, lib/authority/citation-generator.ts</files>
  <action>
**authority-scorer.ts**: Enhanced scoring that replaces the existing heuristic.

`scoreAuthority(claims: ValidatedClaim[], sourceBreakdown: Record<SourceType, number>): number`

Scoring formula (0-100):
- **Claim verification rate** (40 points): `(verified / total) * 40`
- **Source diversity** (20 points): 5 points per unique source type used (max 4 types = 20)
- **Source quality** (25 points): Government sources = 8pts each (max 16), Academic = 5pts each (max 10), Industry = 3pts (max 6), Web = 1pt (max 3) — capped at 25
- **Citation coverage** (15 points): `(claimsWithCitations / total) * 15`

Also export `getAuthorityTier(score: number)` matching existing `getTier()` pattern:
- 80-100: 'Excellent' (emerald)
- 60-79: 'Good' (cyan)
- 40-59: 'Fair' (amber)
- 0-39: 'Needs Work' (red)

**citation-generator.ts**: Generate formatted citations from validated claims.

`generateCitations(claims: ValidatedClaim[], format?: 'footnote' | 'inline' | 'apa' | 'chicago'): GeneratedCitation[]`

Citation formats:
- **footnote** (default): `[1] Source Name. "Title." URL. Accessed DATE.`
- **inline**: `(Source Name, YEAR)`
- **apa**: `Author (YEAR). Title. Source Name. URL`
- **chicago**: `Author. "Title." Source Name. Accessed DATE. URL.`

`generateFootnoteBlock(citations: GeneratedCitation[]): string` — returns full footnote section as markdown text.

Only generate citations for verified claims (verified = true, verificationScore >= 0.6).

AVOID: Don't make scoring depend on external API calls. Scorer must be deterministic given inputs. Don't generate citations for unverified claims.

<task type="auto">
  <name>Task 3: Create authority analyzer orchestrator</name>
  <files>lib/authority/authority-analyzer.ts</files>
  <action>
Create the main orchestrator following the `lib/geo/geo-analyzer.ts` pattern.

`analyzeAuthority(content: string, options: { userId: string; orgId: string; deepValidation: boolean }): Promise<AuthorityAnalysisResult>`

**Orchestration flow**:
1. Extract claims from content via `extractClaims(content)`
2. If `deepValidation` is false (free tier): return basic result with claims found, heuristic score (reuse existing `lib/geo/authority-scorer.ts` logic), and `addonRequired: true` flag
3. If `deepValidation` is true (addon tier):
   a. For each claim, search all source connectors via `searchAllConnectors(claim.text)`
   b. Use `Promise.allSettled` for parallel claim validation (batch 5 at a time to avoid rate limits)
   c. Map SourceResults back to ValidatedClaim objects
   d. Score authority via `scoreAuthority()`
   e. Generate citations via `generateCitations()`
   f. Save to database: create AuthorityAnalysis record + AuthorityCitation records
   g. Return full AuthorityAnalysisResult

**Database persistence**:
- Import prisma from `@/lib/prisma`
- After analysis, create `prisma.authorityAnalysis.create()` with all scores
- Create `prisma.authorityCitation.createMany()` for each generated citation
- Return the analysis ID in the result

**Error handling**:
- If individual connector fails, continue with remaining connectors
- Log connector failures via `logger.warn()`
- Never let connector failure crash the entire analysis

AVOID: Don't run more than 5 concurrent connector searches (rate limit protection). Don't skip database persistence. Don't throw on partial connector failures.
