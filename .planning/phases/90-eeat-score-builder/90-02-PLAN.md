<tasks>
<task id="1" name="Create app/api/eeat/audit/route.ts">
POST — run full E-E-A-T audit.
GET — fetch user's saved EEATAudit records.

POST body: { text: string, contentUrl?: string, save?: boolean, generateAssets?: boolean }
- Call scoreEEAT(text) from lib/eeat/eeat-scorer.ts
- If generateAssets=true: call generateEEATAssets(result)
- If save=true: create EEATAudit record in DB
- Return: { audit: EEATAuditResult, assets?: EEATAssetPlan, auditId?: string }

GET — returns last 50 EEATAudit records for authenticated user:
{ id, overallScore, experienceScore, expertiseScore, authorityScore, trustScore, createdAt }

Auth: getUserIdFromRequest() — 401 if not authenticated
Feature gate: check eeatAudits feature key (from task 3 below)
</task>

<task id="2" name="Create app/api/eeat/assets/route.ts">
POST — generate E-E-A-T asset plan for given content (no DB persistence).
Body: { text: string }
- Calls scoreEEAT(text) then generateEEATAssets(result)
- Returns: EEATAssetPlan

Auth: getUserIdFromRequest()
Rate limiting: 20/min (asset generation is compute-light but spamming should be limited)
</task>

<task id="3" name="Extend lib/geo/feature-limits.ts with eeatAudits">
Read the file. The eeatAudits key already exists in the interface (added in Phase 86 — verify this).
If it already exists: no change needed to interface, just verify plan counts.
If it does NOT exist: add eeatAudits to interface and all plan records.

Target values (if adding):
- free: eeatAudits: 0
- pro: eeatAudits: 10
- growth: eeatAudits: -1
- scale: eeatAudits: -1
- professional: eeatAudits: 10
- business: eeatAudits: -1
- custom: eeatAudits: -1

Add to FEATURE_INFO if not present:
{ label: 'E-E-A-T Audits', description: 'Automated 20-point E-E-A-T scoring and asset generation for Google trust signals' }
</task>
