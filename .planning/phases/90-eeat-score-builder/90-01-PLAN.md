Existing schema has AuthorProfile (with eeatScore field) and SEOAudit (with eeatScore JSON).
Phase 90 adds a dedicated EEATAudit model for granular 20-point automated scoring.

The 20-point audit scores 4 pillars (5 points each):
- Experience: first-hand signals in content text
- Expertise: knowledge depth, technical accuracy, source quality
- Authoritativeness: authority signals, external references
- Trust: transparency, accuracy indicators

All scoring is pure TypeScript — no LLM calls. Asset generation produces templates.

<task id="2" name="Create lib/eeat/types.ts">
All type definitions for the E-E-A-T module:

```typescript
export interface EEATDimension {
  score: number;        // 0-25
  maxScore: number;     // Always 25
  signals: EEATSignal[];
  missing: string[];    // Assets/signals that would improve this dimension
}

export interface EEATSignal {
  name: string;
  detected: boolean;
  weight: number;       // Points this signal contributes when detected
  evidence?: string;    // Snippet of text that triggered detection (max 100 chars)
}

export interface EEATAuditResult {
  experience: EEATDimension;
  expertise: EEATDimension;
  authority: EEATDimension;
  trust: EEATDimension;
  overallScore: number;  // 0-100
  grade: 'A' | 'B' | 'C' | 'D' | 'F';
  wordCount: number;
  auditedAt: string;
}

// Asset generation types
export interface EEATAsset {
  type: 'author-bio' | 'credential-checklist' | 'schema-template' | 'trust-signal' | 'citation-template';
  title: string;
  content: string;    // Ready-to-use template content
  priority: 'high' | 'medium' | 'low';
}

export interface EEATAssetPlan {
  assets: EEATAsset[];
  totalImpact: number;   // Estimated score improvement if all assets implemented
  quickWins: EEATAsset[]; // High priority + easy to implement
}
```

<task id="3" name="Create lib/eeat/eeat-scorer.ts">
20-point automated E-E-A-T scorer from content text. Pure TypeScript regex/pattern matching.

EXPERIENCE dimension (5 signals × up to 5 pts each = 25 max):
1. First-hand language (5pts): /\b(I (tested|tried|found|discovered|noticed|experienced)|we (found|observed|measured|noticed))\b/i
2. Specific dates/times (5pts): /\b(in (January|February|March|April|May|June|July|August|September|October|November|December)|in 20\d\d|last (month|year|week)|on (Monday|Tuesday|Wednesday|Thursday|Friday))\b/i
3. Original data/statistics (5pts): /\b\d+(\.\d+)?%|\d+ (percent|respondents|participants|users|customers|cases)\b/i
4. Personal example sentences (5pts): sentences containing "in my experience", "from my own", "when I", "I've personally"
5. Case study language (5pts): /\b(case study|case studies|real example|actual client|our client|we helped)\b/i

EXPERTISE dimension (5 signals):
1. Technical/jargon depth (5pts): count domain-specific terms (>3 unique technical terms = full points, inferred from avg word length > 5.5 and word diversity)
2. Source citations (5pts): URLs or "(Source:", "[1]", "according to [A-Z]", "per [A-Z]", "cited by" pattern
3. Precise specifications (5pts): specific numbers with units — "3.2 GHz", "128GB", "±0.5mm", ISO standards, RFC references
4. Author credentials in text (5pts): /\b(PhD|MD|MBA|CPA|certified|licensed|registered|accredited|member of)\b/i
5. Academic/research language (5pts): /\b(research (shows|suggests|indicates|found)|study|peer-reviewed|meta-analysis|systematic review)\b/i

AUTHORITY dimension (5 signals):
1. External reference patterns (5pts): mentions of named organisations, publications, government bodies — "according to Google", "per the ABS", "as reported by"
2. Award/recognition language (5pts): /\b(award|recognised|ranked|#1|top 10|industry leader|featured in)\b/i
3. Social proof numbers (5pts): /\b(\d+(,\d{3})* (customers|clients|users|downloads|subscribers|followers))\b/i
4. Expert quotes (5pts): pattern of "Name, Title at Org" followed by quote text
5. Years in industry (5pts): /\b(\d+ years? (of )?(experience|in the industry|in business))\b/i

TRUST dimension (5 signals):
1. Transparency language (5pts): author name pattern, "written by", "reviewed by", "updated"
2. Disclaimer/caveat (5pts): /\b(please consult|seek professional|note that|disclaimer|this (is|article) (not|intended))\b/i — appropriate hedging
3. Contact/about reference (5pts): /\b(contact us|get in touch|about (us|the author|me)|our team)\b/i or email pattern
4. Factual precision (5pts): dates formatted as DD/MM/YYYY or YYYY, specific version numbers, changelog references
5. Privacy/security language (5pts): /\b(privacy policy|GDPR|data protection|secure|encrypted|SSL|certificate)\b/i

Each signal: if pattern matches → award its weight (5 pts). Cap each dimension at 25.
Overall = sum of 4 dimensions (max 100).

Grade: same as humanness scorer (A≥90, B≥75, C≥60, D≥40, F<40).

For "missing" list: for each undetected signal, add a human-readable suggestion like:
"Add first-hand examples using 'I tested' or 'we found' language"
"Include source citations (URLs, named publications, government bodies)"

Export: scoreEEAT(text: string): EEATAuditResult
</task>

<task id="4" name="Create lib/eeat/eeat-asset-generator.ts">
Generates ready-to-use templates to fill identified E-E-A-T gaps.

Input: EEATAuditResult (from scoreEEAT)
Output: EEATAssetPlan

Asset types generated based on what's missing:

If experience < 15:
- type: 'author-bio' → template bio paragraph with [NAME], [YEARS], [SPECIFIC_EXPERIENCE] placeholders
- type: 'citation-template' → "I've personally [tested/used/reviewed] X and found that..."

If expertise < 15:
- type: 'credential-checklist' → ordered checklist of credentials to add: degrees, certifications, publications
- type: 'schema-template' → JSON-LD Person schema with all fields pre-filled with placeholders

If authority < 15:
- type: 'citation-template' → template for adding expert quotes with bylines
- type: 'trust-signal' → template "Featured in [Publication] | Rated [X] by [Y] clients"

If trust < 15:
- type: 'trust-signal' → "Updated: [DATE] | Reviewed by [NAME], [CREDENTIAL] | [N] citations"
- type: 'schema-template' → JSON-LD Article schema with author, datePublished, dateModified

Priority logic:
- high: dimension score < 10 (bottom 40%)
- medium: 10-18
- low: 18-24

quickWins = high priority assets filtered to those under 100 words (easy to implement)

Total impact estimate: sum of max pts per missing signal, scaled to 0-100.

Export: generateEEATAssets(result: EEATAuditResult): EEATAssetPlan
</task>
