<task id="1" name="Create components/eeat/EEATScoreCard.tsx">
Displays EEATAuditResult with 4-pillar breakdown.

Layout:
- Overall score: large number (0-100) + grade badge at top
- 4 pillar cards in 2×2 grid:
  - Experience: score/25, icon: User, colour: blue
  - Expertise: score/25, icon: BookOpen, colour: purple
  - Authority: score/25, icon: Award, colour: amber
  - Trust: score/25, icon: Shield, colour: emerald
- Each pillar card shows: score bar (0-25), detected signals list (green checkmarks), missing items list (amber dots)

Signal pills: small coloured tags showing detected signal names

Props: { result: EEATAuditResult; className?: string }
</task>

<task id="2" name="Create components/eeat/EEATAssetPanel.tsx">
Displays EEATAssetPlan with collapsible asset cards.

Layout:
- Header: "X assets ready to implement — estimated +Y points"
- Quick wins section first (priority: high): highlighted in green-tinted bg
- All assets as collapsible cards:
  - Title + priority badge + type badge
  - Collapsed: title only
  - Expanded: full template content in a styled code/textarea block
  - "Copy Template" button on each card
- Impact indicator: "Implementing all assets could improve your score to ~X"

Props: { plan: EEATAssetPlan; className?: string }
</task>

<task id="3" name="Create app/dashboard/eeat/page.tsx">
E-E-A-T Builder dashboard. Three tabs: Score | Assets | History.

Score Tab:
- Textarea with live word counter
- Optional URL field (not required — content analysis only in MVP)
- "Run E-E-A-T Audit" button
- Toggles: "Generate asset plan" checkbox (checked by default)
- Results: EEATScoreCard

Assets Tab:
- Shows EEATAssetPanel after audit is run
- If no audit yet: EmptyState "Run an audit first to generate your asset plan"
- Copy-to-clipboard for each asset template

History Tab:
- SWR fetch from GET /api/eeat/audit
- List: date, overall score, grade, word count
- Click → expand to show full audit details

State management: three useState buckets (auditResult, assetPlan, historyData)
API calls: useApi() for POST, useSWR for GET history

Also create a GET route for history tab:
Read app/api/eeat/audit/route.ts — if GET is already there (from 90-02), no change needed.
</task>
