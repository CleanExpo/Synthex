<tasks>
<task id="1" name="Create app/api/brand/identity/route.ts">
POST — save/update brand identity and generate entity graph.
GET — fetch user's brand identities.

POST body schema (Zod):
```typescript
z.object({
  entityType: z.enum(['organization', 'person', 'local-business']),
  canonicalName: z.string().min(1).max(200),
  canonicalUrl: z.string().url(),
  description: z.string().max(2000).optional(),
  logoUrl: z.string().url().optional(),
  foundingDate: z.string().optional(),
  hasPhysicalLocation: z.boolean().optional(),
  address: z.object({ street: z.string().optional(), suburb: z.string().optional(), state: z.string().optional(), postcode: z.string().optional(), country: z.string().optional() }).optional(),
  phone: z.string().optional(),
  wikidataUrl: z.string().url().optional(),
  wikipediaUrl: z.string().url().optional(),
  linkedinUrl: z.string().url().optional(),
  crunchbaseUrl: z.string().url().optional(),
  youtubeUrl: z.string().url().optional(),
  twitterUrl: z.string().url().optional(),
  facebookUrl: z.string().url().optional(),
  instagramUrl: z.string().url().optional(),
  declaredHandles: z.array(z.object({ platform: z.string(), url: z.string(), handle: z.string().optional() })).optional(),
})
```

On POST:
1. Auth + Zod validate
2. Check feature limit brandIdentities (count existing BrandIdentity for user)
3. Call buildEntityGraph(input) → entityGraph
4. Call scoreConsistency(input) → { overallScore, consistencyReport }
5. Upsert BrandIdentity (upsert by canonicalUrl + userId)
6. Return { identity, entityGraph }

GET: return all BrandIdentity records for user (omit entityGraph from list, include in detail)
</task>

<task id="2" name="Create app/api/brand/consistency/route.ts">
POST — run consistency audit for a saved brand identity.

POST body: `{ brandId: string }`

1. Auth + validate brandId belongs to userId
2. Fetch BrandIdentity from DB
3. Reconstruct BrandIdentityInput from DB record
4. Call scoreConsistency(input) → ConsistencyReport
5. Update BrandIdentity: consistencyScore, consistencyReport, updatedAt
6. Return ConsistencyReport
</task>

<task id="3" name="Create app/api/brand/mentions/route.ts">
GET — fetch recent BrandMention records for a brand.

Query params: brandId (required), page (default 1), limit (default 20, max 50)

1. Auth + validate brandId belongs to userId
2. Query BrandMention WHERE brandId = brandId ORDER BY publishedAt DESC with pagination
3. Return { mentions, total, page, limit }
</task>

<task id="4" name="Create app/api/brand/mentions/poll/route.ts">
POST — trigger a manual mention poll and store new results.

POST body: `{ brandId: string }`

1. Auth + validate brandId belongs to userId
2. Check feature limit brandMentionPolls
3. Fetch BrandIdentity.canonicalName
4. Call pollMentions(canonicalName, { apiKeys from env vars })
5. For each deduplicated RawMention:
   a. Compute urlHash
   b. Upsert BrandMention (skip on unique constraint violation — already exists)
6. Return { newCount, totalFetched, polledAt }

Rate limit: 10/hour per user (use existing rate limiter middleware)
</task>

<task id="5" name="Create app/api/brand/wikidata/route.ts">
GET — check Wikidata entry for a brand identity.

Query params: brandId (required)

1. Auth + validate
2. Fetch BrandIdentity.canonicalName + wikidataUrl
3. Call checkWikidata(canonicalName, wikidataUrl)
4. If result.qId found, update BrandIdentity.wikidataQId
5. Return WikidataCheckResult
</task>

<task id="6" name="Create app/api/brand/kg-check/route.ts">
GET — check Google Knowledge Graph for entity confidence.

Query params: brandId (required)

1. Auth + validate
2. Fetch BrandIdentity.canonicalName
3. If GOOGLE_KG_API_KEY not set, return { found: false, reason: 'API key not configured' }
4. Call checkKnowledgeGraph(canonicalName, apiKey)
5. If result.kgmid found, update BrandIdentity: kgmid, kgConfidence
6. Return KGCheckResult
</task>

<task id="7" name="Create app/api/brand/calendar/route.ts">
POST — generate brand publishing + maintenance calendar.

POST body: `{ brandId: string, coverageDays?: number }`

1. Auth + validate
2. Fetch BrandIdentity + its BrandCredentials (for expiry-based events)
3. Reconstruct BrandIdentityInput from DB
4. Build credentials array from BrandCredential records
5. Call generateBrandCalendar(input, { credentials })
6. Return BrandCalendar
</task>
