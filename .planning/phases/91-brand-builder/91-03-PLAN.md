<tasks>
<task id="1" name="Create components/brand/BrandIdentityCard.tsx">
Displays a BrandIdentity record with its entity graph status.

Layout:
- Header: entity type badge (Organization/Person/Local Business) + canonicalName
- Score row: Knowledge Panel confidence (kgConfidence as %) + Wikidata status (found/not-found) + Consistency score
- sameAs links: compact grid showing which platforms are connected (green checkmark) vs missing (grey dash)
- Entity graph toggle: "View JSON-LD" expander showing the generated schema markup in <pre> block
- Action buttons: "Re-audit Consistency" | "Check Wikidata" | "Check Knowledge Graph"

Props: { identity: BrandIdentityRecord; onReaudit?: () => void }
(BrandIdentityRecord = BrandIdentity fields from Prisma serialised as JSON)
</task>

<task id="2" name="Create components/brand/ConsistencyAuditPanel.tsx">
Shows ConsistencyReport as a platform-by-platform matrix.

Layout:
- Overall score: large number (0-100) with grade colour (≥80 green, ≥60 amber, <60 red)
- Platform table:
  | Platform | Declared Name | Status | Score |
  - Status badge: "Exact match" (green) | "Variant" (amber) | "Mismatch" (red) | "Not found" (grey)
- Critical issues: red alert list if any sameAs platforms have mismatches
- Recommendations: amber tip list

Props: { report: ConsistencyReport | null; loading?: boolean; onRunAudit: () => void }
</task>

<task id="3" name="Create components/brand/BrandMentionsFeed.tsx">
Chronological list of brand mentions with sentiment badges.

Layout:
- Poll button: "Poll for new mentions" (triggers POST /api/brand/mentions/poll)
- Feed list: each mention as a card:
  - Source domain + date (relative)
  - Title (linked to URL)
  - Description snippet (truncated to 120 chars)
  - Sentiment badge: green "Positive" | grey "Neutral" | red "Negative"
  - API source chip: NewsData.io | GDELT | Guardian
- Empty state: "No mentions found. Click 'Poll for new mentions' to search."
- Pagination: load more button (page-based)

Props: { brandId: string }
Uses useSWR to GET /api/brand/mentions?brandId=XXX and useApi for POST poll.
</task>

<task id="4" name="Create components/brand/WikidataStatusCard.tsx">
Shows Wikidata entry completeness.

Layout:
- Found / Not Found status with Q-ID (if found)
- Score: 0-100 with circular progress or bar
- Properties table:
  - Required: P31 (instance-of), P856 (website), P571 (inception)
  - Recommended: P159 (HQ), P112 (founder), P18 (image), P2671 (KGMID)
  - Each with ✓ (green) or ✗ (red)
- Reference count: "X references — [weak/moderate/strong]"
- Recommendations list (amber tips)
- "Check Wikidata" button

Props: { result: WikidataCheckResult | null; loading?: boolean; onCheck: () => void }
</task>

<task id="5" name="Create components/brand/KnowledgePanelStatus.tsx">
Shows Google Knowledge Graph entity confidence.

Layout:
- Large confidence score: % with colour coding (≥70% green, ≥40% amber, <40% red)
- KGMID display (if found)
- Entity name + description from KG API
- @types list (Organisation, LocalBusiness, etc.)
- "Not found" empty state: explains that 95%+ entity consistency + citations needed
- "Check Knowledge Graph" button (disabled if no GOOGLE_KG_API_KEY env — show "API key not configured")

Props: { result: KGCheckResult | null; loading?: boolean; onCheck: () => void; hasApiKey: boolean }
</task>

<task id="6" name="Create components/brand/BrandCalendarView.tsx">
Shows 90-day brand publishing + maintenance calendar.

Layout:
- Generate button: "Generate Calendar" → POST /api/brand/calendar
- Summary chips: Total Events | High Priority | Publishing | Maintenance
- Event list grouped by week:
  - Week header: "Week of DD MMM"
  - Event row: date + type badge + title + priority indicator
  - Type badges: publish-article (blue), social-post (cyan), credential-refresh (amber), mention-review (slate), schema-audit (purple), wikidata-update (green)
- Empty state when no calendar generated yet

Props: { brandId: string }
</task>
