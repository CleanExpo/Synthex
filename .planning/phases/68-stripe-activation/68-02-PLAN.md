workflow engine, AI insights), then surface upgrade prompts in the UI so free users see

error for free users. `UpgradePrompt` component shows plan upgrade CTA when 403 is received.

- Never expose plan internals to client — return only 403 + message

**Established gate pattern (from app/api/ai/chat/conversations/route.ts):**
```typescript
const ALLOWED_PLANS = ['professional', 'business', 'custom'];

// After auth check, before business logic:
const subscription = await subscriptionService.getSubscription(userId);
if (!subscription || !ALLOWED_PLANS.includes(subscription.plan)) {
  return NextResponse.json(
    { error: 'This feature requires a Professional or Business plan.' },
    { status: 403 }
  );
}
```

**Routes to gate (professional+ required):**
- app/api/seo/audit/* — SEO audit runs consume resources, free tier shouldn't run them
- app/api/workflows/* — Multi-step workflow engine is a premium AI feature
- app/api/insights/* — AI insights agent is premium (cron-triggered, cost centre)

**Note on "insights":** /api/insights is a backend data-read route (not the cron trigger).
Gate the data read so free users can't see insights — they must upgrade.
