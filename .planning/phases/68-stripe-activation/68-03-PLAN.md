  <action>Configure Stripe production keys and register webhook</action>
  <instructions>
    I've built all the Stripe billing infrastructure. To activate real payments, you need
    to configure Stripe in three places:

    **Step 1 — Stripe Dashboard: Create products and prices**
    1. Go to https://dashboard.stripe.com/products
    2. Create "Professional" product → price AUD $249/month (recurring)
    3. Create "Business" product → price AUD $399/month (recurring)
    4. Copy the price IDs (format: price_1xxxxxxxxxxxxxxxxx)

    **Step 2 — Stripe Dashboard: Register webhook**
    1. Go to https://dashboard.stripe.com/webhooks
    2. Add endpoint: https://synthex.vercel.app/api/webhooks/stripe
    3. Select these events: customer.subscription.created, customer.subscription.updated,
       customer.subscription.deleted, invoice.payment_succeeded, invoice.payment_failed,
       checkout.session.completed
    4. Copy the webhook signing secret (format: whsec_xxx)

    **Step 3 — Vercel Dashboard: Add environment variables**
    1. Go to https://vercel.com/dashboard → Synthex project → Settings → Environment Variables
    2. Add for Production:
       - STRIPE_SECRET_KEY = sk_live_xxx (from Stripe dashboard → API keys)
       - NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY = pk_live_xxx
       - STRIPE_WEBHOOK_SECRET = whsec_xxx (from step 2)
       - STRIPE_PROFESSIONAL_PRICE_ID = price_xxx (from step 1)
       - STRIPE_BUSINESS_PRICE_ID = price_xxx (from step 1)
    3. Redeploy Vercel (or trigger deployment) to pick up new env vars

    **For local testing only:** You can use test keys (sk_test_xxx, pk_test_xxx) and
    run `stripe listen --forward-to localhost:3000/api/webhooks/stripe` to test webhooks.

    Type "configured" when complete (or "skip" to proceed without real Stripe keys — feature gates
    and billing UI will still work, but checkout will show a "not configured" error).
