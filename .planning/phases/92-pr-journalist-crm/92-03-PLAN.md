# Plan 92-03: Dashboard UI + Navigation

**Phase:** 92 — Journalist & PR Relationship Manager
**Status:** Pending

## Tasks

All components use:
- SWR for GET data fetching (with `credentials: 'include'` fetcher, per GamificationWidget.tsx pattern)
- `useApi` hook for mutations (POST/PATCH)
- Glassmorphic styling: `bg-white/5 backdrop-blur-xl border border-white/10 rounded-xl`
- Radix UI primitives
- Icons from `@/components/icons`

### Task 1: Add NewspaperIcon to icons adapter

In `components/icons/index.ts`, add:
```typescript
NewspaperIcon as Newspaper,
```
from `@heroicons/react/24/outline` (NewspaperIcon exists in Heroicons v2).

### Task 2: components/pr/PROverviewStats.tsx

Stats summary component. 4 stat cards:
- Total journalists (with tier breakdown: cold/warm/hot/advocate)
- Active pitches (sent + opened + replied)
- Coverage this month
- Press releases (published count)

Uses SWR to fetch from each respective API endpoint. Glassmorphic card styling.

### Task 3: components/pr/JournalistList.tsx

Searchable list of journalist contacts.
- Client-side search filter by name/outlet/beat
- Tier badge: colour-coded (cold=gray, warm=yellow, hot=orange, advocate=green)
- Beat chips: small rounded tags
- Last contacted date (relative: "3 days ago")
- "Enrich" button calls PATCH `/api/pr/journalists/[id]/enrich`
- "New Journalist" button opens JournalistForm

### Task 4: components/pr/JournalistForm.tsx

Modal/panel for creating or editing a journalist contact.
Fields: name, email, outlet, outletDomain, title, location, twitterHandle, linkedinUrl, notes (textarea), tier (select: cold/warm/hot/advocate), beats (comma-separated input).
Hunter.io enrichment button: POST to `/api/pr/journalists/[id]/enrich` — shows email + confidence score result.

### Task 5: components/pr/PitchKanban.tsx

Kanban board with columns by status:
`draft | sent | opened | replied | covered | declined`

Each column shows pitch cards. Click-to-update-status via dropdown (no drag required).
Each pitch card shows: journalist name, subject line, angle preview, created date.
"New Pitch" button in draft column.

### Task 6: components/pr/CoverageFeed.tsx

Chronological list of media coverage entries.
Each entry: outlet name + domain, title (linked), journalist name (if linked), pitch reference (if attributed), sentiment badge (positive=green/neutral=gray/negative=red), published date.
"Poll Coverage" button at top: POST to `/api/pr/coverage/poll` with brand name from local state.
"Add Manual" button opens inline form.

### Task 7: components/pr/PressReleaseEditor.tsx

Two-panel layout:
- Left: form fields (headline, subheading, body textarea, boilerplate, keywords, location, category, contact details, imageUrl)
- Right: JSON-LD preview pane (pre-formatted JSON display of the schema.org output)

Actions: Save Draft, Publish (sets status=published), Archive.

### Task 8: app/dashboard/pr/page.tsx

Main PR Manager dashboard page. `'use client'`.
Uses Radix `Tabs` with 4 tabs:
- `Journalists` — renders `<JournalistList />`
- `Pitches` — renders `<PitchKanban />`
- `Coverage` — renders `<CoverageFeed />`
- `Press Releases` — renders `<PressReleaseEditor />` (list + editor toggle)

Top section: `<PROverviewStats />` always visible.
Tab state synced to URL query param `?tab=` so direct links work.
