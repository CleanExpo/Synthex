# Plan 92-02: API Routes

**Phase:** 92 — Journalist & PR Relationship Manager
**Status:** Pending

## Tasks

All routes follow the established Synthex pattern:
- `getUserIdFromRequest(request)` from `@/lib/auth/jwt-utils`
- Zod validation for request bodies
- Prisma queries scoped by `orgId` (which is the userId in this codebase)
- NextResponse JSON returns with appropriate status codes

### Task 1: app/api/pr/journalists/route.ts

**GET** — List journalist contacts, filtered by beat/tier query params.
**POST** — Create a new journalist contact. Zod schema: name, outlet, outletDomain required; email, title, beats[], tier, twitterHandle, linkedinUrl, notes optional.

### Task 2: app/api/pr/journalists/[id]/route.ts

**GET** — Fetch single journalist with their pitches (last 5).
**PATCH** — Update journalist fields (partial update, all fields optional in Zod).
**DELETE** — Soft delete by setting `doNotContact: true` (never hard delete per CLAUDE.md).

### Task 3: app/api/pr/journalists/[id]/enrich/route.ts

**POST** — Call `hunter-enricher.ts` with journalist name split into first/last + outletDomain.
Returns `{ email, confidence }`. Updates journalist record if email found.
Graceful 200 response if HUNTER_API_KEY not set: `{ message: "Hunter.io not configured" }`.

### Task 4: app/api/pr/pitches/route.ts

**GET** — List pitches, filtered by `status` query param. Include journalist name.
**POST** — Create pitch. Zod: journalistId, subject, angle required; bodyDraft, personalisation, campaignId, tags[] optional.

### Task 5: app/api/pr/pitches/[id]/route.ts

**GET** — Fetch single pitch with journalist detail.
**PATCH** — Update pitch. Allows updating: status, bodyDraft, personalisation, sentAt, repliedAt, followUpAt, tags.
Status transitions validated: draft→sent, sent→opened, opened→replied, replied→covered|declined, any→archived.

### Task 6: app/api/pr/coverage/route.ts

**GET** — List coverage entries, most recent first. Include journalist and pitch linkage.
**POST** — Manual coverage entry. Zod: url, title, outlet, outletDomain required; description, publishedAt, pitchId, journalistId, sentiment optional.
Auto-generate `urlHash` using the same djb2 pattern as `BrandMention` (or SHA-based — match existing pattern).

### Task 7: app/api/pr/coverage/poll/route.ts

**POST** — Trigger coverage polling:
1. Require `brandName` in request body.
2. Call `pollMentions(brandName)` from `lib/brand/mention-poller.ts`.
3. For each raw mention, call `linkCoverage()` from `lib/pr/coverage-linker.ts` against open pitches.
4. Create `MediaCoverage` records for new entries (skip duplicates via urlHash+orgId unique constraint).
5. If coverage linked to a pitch, update pitch status to 'covered'.
6. Return `{ polled: N, linked: N, created: N }`.

### Task 8: app/api/pr/press-releases/route.ts

**GET** — List press releases for org, ordered by createdAt desc.
**POST** — Create press release. Zod: headline, body required; slug auto-generated from headline if not provided (kebab-case, truncated to 80 chars); status defaults to "draft".

### Task 9: app/api/pr/press-releases/[id]/route.ts

**GET** — Fetch single press release.
**PATCH** — Update press release. Publishing: when status changes to "published", set publishedAt = now().
**DELETE** — Move to archived status (set status = "archived"), do not hard delete.

### Task 10: app/api/pr/press-releases/[orgSlug]/[slug]/route.ts

**GET** — Public newsroom endpoint. No auth check.
- Fetch by orgSlug (match against Organization.slug) + release slug.
- Only return if status = "published".
- Returns `{ release, jsonLd }` where jsonLd is generated by `buildPressRelease()`.
- 404 if not found or not published.
