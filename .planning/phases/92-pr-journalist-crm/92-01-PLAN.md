## Tasks

### Task 1: Add Prisma Models

Add 4 models to `prisma/schema.prisma` (append after `BrandMention`):

- `JournalistContact`: id, userId, orgId, name, email, emailVerified, emailScore, outlet, outletDomain, title, location, beats (String[]), recentArticles (Json?), beatsUpdatedAt, twitterHandle, linkedinUrl, notes (@db.Text), tier (default "cold"), lastContactedAt, doNotContact, createdAt, updatedAt + relations to PRPitch[], MediaCoverage[], Organization, User
- `PRPitch`: id, userId, orgId, journalistId, subject, angle (@db.Text), bodyDraft (@db.Text?), personalisation (@db.Text?), status (default "draft"), sentAt, openedAt, repliedAt, followUpAt, followUpCount, campaignId?, tags (String[]), createdAt, updatedAt + relations to JournalistContact, MediaCoverage[], Organization, User
- `MediaCoverage`: id, userId, orgId, pitchId?, journalistId?, url, urlHash, title, description (@db.Text?), outlet, outletDomain, publishedAt?, apiSource, estimatedReach?, sentiment (default "neutral"), tier (default "tier3"), createdAt + @@unique([urlHash, orgId]) + relations to PRPitch?, JournalistContact?, Organization, User
- `PressRelease`: id, userId, orgId, slug, headline, subheading?, body (@db.Text), boilerplate (@db.Text?), contactName?, contactEmail?, contactPhone?, datePublished?, location?, category?, keywords (String[]), imageUrl?, status (default "draft"), publishedAt?, distributedTo (String[]), createdAt, updatedAt + @@unique([orgId, slug]) + relations to Organization, User

Add back-relations to User model:
- `journalistContacts  JournalistContact[]`
- `prPitches           PRPitch[]`
- `mediaCoverage       MediaCoverage[]`
- `pressReleases       PressRelease[]`

Add back-relations to Organization model:
- `journalistContacts  JournalistContact[]`
- `prPitches           PRPitch[]`
- `mediaCoverage       MediaCoverage[]`
- `pressReleases       PressRelease[]`

Run: `npx prisma db push`

### Task 2: Create lib/pr/types.ts

Types for the PR service layer:
- `JournalistInput` — create/update journalist contact
- `PitchInput` — create pitch
- `PitchStatus` — union type of all valid statuses
- `CoverageResult` — enriched coverage with pitch link
- `PRDraftSuggestion` — structured pitch draft output
- `PressReleaseInput` — create/update press release

### Task 3: Create lib/pr/beat-classifier.ts

Pure TypeScript beat classification — no AI call.

```typescript
const BEAT_KEYWORDS: Record<string, string[]> = {
  technology: ['tech', 'software', 'startup', 'ai', 'app', 'digital', 'cyber', 'saas'],
  business: ['merger', 'acquisition', 'ipo', 'revenue', 'profit', 'ceo', 'founder'],
  // ... etc
}

export function classifyBeat(outletUrl: string, recentHeadlines: string[]): Promise<string>
```

Combines keyword scoring across all headlines to return the dominant beat string.

### Task 4: Create lib/pr/hunter-enricher.ts

Optional Hunter.io email lookup:

```typescript
export async function enrichJournalist(
  name: string,
  domain: string,
  apiKey: string
): Promise<{ email?: string; confidence?: number }>
```

Calls `https://api.hunter.io/v2/email-finder` — graceful empty return if HUNTER_API_KEY not set.

### Task 5: Create lib/pr/coverage-linker.ts

Link raw coverage results to open pitches:

```typescript
export function linkCoverage(
  coverage: RawCoverage,
  pitches: PRPitch[]
): string | null
```

Domain-matches coverage URL against journalist outlet domains. Returns pitchId if match found.

### Task 6: Create lib/pr/pitch-drafter.ts

Pure template-based pitch draft generation (no AI call):

```typescript
export function generatePitchDraft(
  journalist: JournalistContact,
  brandIdentity: { canonicalName: string; description: string },
  angle: string
): PRDraftSuggestion
```

Returns structured draft with subject, body, and personalisation note.

### Task 7: Create lib/pr/press-release-builder.ts
