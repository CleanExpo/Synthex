1. Add ScheduledAuditTarget model to prisma/schema.prisma:
   ```prisma
   model ScheduledAuditTarget {
     id              String    @id @default(cuid())
     userId          String
     user            User      @relation(fields: [userId], references: [id], onDelete: Cascade)

     url             String
     name            String    // Friendly name like "Main Site" or "Blog"
     frequency       String    // "daily", "weekly", "monthly"
     alertThreshold  Int       @default(10) // Score drop percentage to trigger alert
     enabled         Boolean   @default(true)

     lastRunAt       DateTime?
     lastScore       Float?

     createdAt       DateTime  @default(now())
     updatedAt       DateTime  @updatedAt

     @@index([userId])
     @@index([enabled, frequency])
     @@map("scheduled_audit_targets")
   }
   ```
   Add relation to User model: `scheduledAuditTargets ScheduledAuditTarget[]`

2. Create POST/GET /api/seo/scheduled-audits/route.ts:
   - POST: Create new scheduled audit target (Zod validation: url, name, frequency, alertThreshold)
   - GET: List user's scheduled audit targets with latest scores
   - Use APISecurityChecker with AUTHENTICATED_WRITE/READ policies
   - Check subscription (require professional+ plan)

3. Create GET/PATCH/DELETE /api/seo/scheduled-audits/[id]/route.ts:
   - GET: Single target details
   - PATCH: Update target settings (frequency, alertThreshold, enabled)
   - DELETE: Remove target
   - Verify ownership (target.userId === userId)

Run `npx prisma db push` after schema update.

1. Create lib/seo/audit-scheduler.ts with:
   - `getTargetsDueForAudit(frequency: string)`: Query targets where enabled=true and (lastRunAt is null OR lastRunAt + frequency interval < now)
   - `runScheduledAudit(target)`: Call existing audit logic from app/api/seo/audit/route.ts's performSEOAudit function — extract it to a shared lib/seo/audit-service.ts if needed, or inline the PageSpeed API call
   - `detectRegression(target, newScore)`: Compare newScore with lastScore, return { regressed: boolean, dropPercent: number }
   - `buildAlertEmail(target, oldScore, newScore, auditResult)`: HTML email template (follow weekly-digest pattern)

2. Create app/api/cron/seo-audits/route.ts:
   - GET handler with CRON_SECRET auth check (same pattern as weekly-digest)
   - Set `runtime = 'nodejs'`, `maxDuration = 300`
   - Query all enabled targets due for audit based on frequency
   - For each target:
     a. Run audit via performSEOAudit (reuse existing logic)
     b. Store result in SEOAudit model
     c. Update target.lastRunAt and target.lastScore
     d. If regression detected (score dropped by alertThreshold%), send email alert
   - Return summary: { auditsRun, regressionsDetected, emailsSent, errors }

3. Add to vercel.json crons array:
   ```json
   {
     "path": "/api/cron/seo-audits",
     "schedule": "0 3 * * *"
   }
   ```

- Score change (e.g., "85 → 72 (-13 points)")
