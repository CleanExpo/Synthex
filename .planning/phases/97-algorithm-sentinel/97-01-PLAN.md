# Phase 97-01 — Algorithm Sentinel: Service Layer, Prisma, Cron, API Routes

## Objective
Build the backend engine for the Algorithm Sentinel: Prisma models, lib/sentinel/ services, API routes, and cron job.

## Prerequisites
- GSC integration exists in `lib/google/search-console.ts` (getSearchAnalytics, getSitemapStatus)
- Cron pattern established in `app/api/cron/health-score/route.ts`
- APISecurityChecker and verifyToken available
- Existing PSI integration in `lib/seo/pagespeed-service.ts`

## Prisma Models

Three new models added to `prisma/schema.prisma`:
- `AlgorithmUpdate` — known Google algorithm updates (static seed + future)
- `SiteHealthSnapshot` — periodic snapshot of GSC + CWV metrics per user site
- `SentinelAlert` — generated alerts with severity, type, and metric deltas

User back-relations added:
- `siteHealthSnapshots SiteHealthSnapshot[]`
- `sentinelAlerts SentinelAlert[]`

Run: `npx prisma db push`

## Service Files

### lib/sentinel/types.ts
All shared TypeScript types for the sentinel domain.

### lib/sentinel/algorithm-feed.ts
- Static list of ~15 known Google algorithm updates (2024–2026)
- `seedAlgorithmUpdates()` — upsert static list to DB on first call
- `getRecentUpdates(days)` — fetch updates from DB within N days

### lib/sentinel/health-checker.ts
- `checkSiteHealth(siteUrl, userId, orgId)` — call GSC via `getSearchAnalytics`, get CWV from PSI if available, save snapshot to DB

### lib/sentinel/alert-engine.ts
- `runAlertEngine(current, previous, recentUpdates)` — compare snapshots, emit SentinelAlert rows for:
  - Ranking drop >20% (warning) / >50% (critical)
  - Traffic drop >30% (warning) / >60% (critical)
  - Crawl error spike >50% (warning)
  - CWV regression (LCP>3s, INP>300ms)
  - Algorithm update active + traffic drop (critical correlation)

### lib/sentinel/sentinel-agent.ts
- `runSentinelCheck(userId, orgId)` — orchestrates: seed updates → fetch site URL from user profile → checkSiteHealth → runAlertEngine → persist alerts

## API Routes

| Route | Method | Purpose |
|-------|--------|---------|
| `/api/sentinel/status` | GET | Current health snapshot for user's site |
| `/api/sentinel/alerts` | GET | Paginated alert list with severity filter |
| `/api/sentinel/alerts/[id]/acknowledge` | POST | Mark alert acknowledged |
| `/api/sentinel/updates` | GET | Algorithm update feed (last N days) |
| `/api/sentinel/check` | POST | Manual trigger of health check |
| `/api/cron/sentinel` | POST | Cron endpoint (CRON_SECRET auth) |
