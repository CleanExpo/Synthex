# Phase 97-02 — Algorithm Sentinel: Dashboard UI + Components + Navigation

## Objective
Build the frontend for the Algorithm Sentinel: 4 React components, a 3-tab dashboard page, sidebar entry, and command palette commands.

## Prerequisites
- Plan 97-01 complete (API routes and DB models in place)
- `ShieldExclamationIcon` added to `components/icons/index.ts`
- SWR data fetching pattern (no raw fetch in client components)

## Components

### components/sentinel/AlertFeed.tsx
- Alert list with severity icon + colour (critical=red, warning=amber, info=blue)
- Alert type badge: ranking-drop, traffic-drop, crawl-error-spike, cwv-regression, algorithm-update
- Metric change display: "Average Position: 8.2 → 12.4 (+51%)" in red
- "Acknowledge" button → POST /api/sentinel/alerts/[id]/acknowledge
- Empty state: "No alerts — your site is healthy"

### components/sentinel/SiteHealthCard.tsx
- Health score 0–100 (computed: GSC metrics + CWV + alert penalty)
- Metric row: Avg Position | Total Clicks | Impressions | Crawl Errors
- CWV mini-display: LCP/INP/CLS with pass/fail dots
- Last checked timestamp + "Check Now" button → POST /api/sentinel/check

### components/sentinel/AlgorithmUpdateTimeline.tsx
- Vertical timeline: date + update name + impact badge (high=red, medium=amber, low=grey)
- "Active rollout" indicator if rolloutEnd is null or in the future
- Correlation chip if update date correlates with user's snapshot traffic drop

### components/sentinel/MetricTrendChart.tsx
- Pure SVG chart (no chart library — keeps bundle small)
- X-axis: snapshot dates, Y-axis: metric value
- Supports: clicks, impressions, avgPosition
- Props: `metric`, `data: { date, value }[]`, `label`

## Page

### app/dashboard/sentinel/page.tsx
Three tabs (URL param: `?tab=health|alerts|algorithm`):

1. **Health tab** — SiteHealthCard + MetricTrendChart (clicks + position)
2. **Alerts tab** — AlertFeed with severity filter + "Mark All Read"
3. **Algorithm tab** — AlgorithmUpdateTimeline + date range filter
