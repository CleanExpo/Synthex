## Goal

Build the dashboard page at `/dashboard/workflows` that shows active workflow executions,
step-by-step progress, and approval actions. Users can start a workflow, monitor its progress,
approve/reject waiting steps, and view execution history.

---

## Acceptance Criteria

- [ ] `/dashboard/workflows` page renders (SSR + client hydration)
- [ ] Active executions listed with status badges and step progress indicators
- [ ] Clicking an execution opens detail view with all steps and their statuses
- [ ] Steps in `waiting_approval` status show Approve/Reject buttons
- [ ] Approve action calls `POST /api/workflows/executions/[id]/approve`
- [ ] Cancel action calls `POST /api/workflows/executions/[id]/cancel` with confirm dialog
- [ ] "New Workflow" button opens creation dialog (template selection or ad-hoc)
- [ ] Loading + empty + error states implemented
- [ ] Navigation: sidebar link + command palette entry
- [ ] `npm run type-check` passes (0 errors)

## Components

### `components/workflows/ExecutionList.tsx` (client)
- Renders list of `WorkflowExecution` objects
- Status badge per execution: `pending` (grey), `running` (blue pulse), `waiting_approval`
  (amber), `completed` (green), `failed` (red), `cancelled` (grey)
- Progress bar: `currentStepIndex / totalSteps`
- Click row → opens `ExecutionDetail`

### `components/workflows/ExecutionDetail.tsx` (client)
- Full execution detail view (sheet or expanded row)
- Shows execution metadata: title, triggerType, startedAt, completedAt
- Step timeline: vertical list of all `StepExecution` records in order
- Each step shows: stepName, stepType icon, status, completedAt, confidenceScore badge
- For `waiting_approval` steps: action buttons inline
- Cancel button (with confirmation) if execution is running or waiting_approval

### `components/workflows/StepTimeline.tsx`
- Visual timeline component for step execution sequence
- Each node: icon (step type) + label + status colour + confidence badge
- Running step: animated pulsing indicator
- Failed step: red with errorMessage tooltip

### `components/workflows/ApprovalActions.tsx` (client)
- Rendered for steps in `waiting_approval` status
- "Approve" button: calls approve API, optimistic update
- "Reject" button: opens reason textarea, calls cancel API
- Loading states on both buttons during API call

### `components/workflows/NewWorkflowDialog.tsx` (client)
- Dialog for starting a new workflow execution
- Step 1: Select template from `GET /api/workflows/templates` OR choose "Ad-hoc"
- Step 2: Enter title + optional input data (JSON or key-value editor)
- Submit: calls `POST /api/workflows/executions`
- On success: optimistic add to execution list + close dialog

### `components/workflows/ConfidenceBadge.tsx`
- Small badge showing confidence score
- Colour: green (≥ 0.85), amber (0.6–0.84), red (< 0.6)
- Text: "95%" format
- Tooltip: "Auto-approved" or "Queued for review"

## Design Patterns

Following Synthex glassmorphic design system:
- Cards: `bg-white/5 border border-white/10 rounded-xl`
- Status badges: coloured `bg-[colour]/20 text-[colour] border border-[colour]/30`
- Progress bar: `bg-gradient-to-r from-cyan-500 to-blue-500`
- Step timeline: vertical connector line with node circles
- Confidence badge: small inline chip with colour coding
- Loading skeletons: `animate-pulse bg-white/5 rounded`
