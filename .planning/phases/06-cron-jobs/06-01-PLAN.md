  <name>Task 1: Create competitor metrics fetcher using platform services</name>
  <files>lib/social/competitor-fetcher.ts</files>
  <action>
Create lib/social/competitor-fetcher.ts — a utility that fetches public profile metrics for tracked competitors using platform APIs.

**Architecture:**
The key challenge is that platform services are designed for authenticated user operations (the user's own account). For competitor tracking, we need to look up OTHER users' public data. Each platform handles this differently:

1. **Twitter** — GET `https://api.twitter.com/2/users/by/username/{username}?user.fields=public_metrics,profile_image_url,description` using the authenticated user's Bearer token. Returns follower_count, following_count, tweet_count, listed_count.

2. **Instagram** — Business Discovery API: GET `https://graph.facebook.com/v18.0/{user_ig_id}?fields=business_discovery.fields(username,name,biography,followers_count,follows_count,media_count,profile_picture_url).username({competitor_username})`. Requires the requesting user to have a Business/Creator account.

3. **LinkedIn** — Public profile lookup is restricted. LinkedIn API does not allow looking up arbitrary profiles. Skip with warning — store null metrics.

4. **YouTube** — GET `https://www.googleapis.com/youtube/v3/channels?part=statistics,snippet&forHandle={handle}` (or forUsername). Returns subscriberCount, viewCount, videoCount. Works with any valid API key.

5. **TikTok** — Public profile lookup not supported by Content API. Skip with warning.

6. **Facebook** — Page lookup: GET `https://graph.facebook.com/v18.0/{page_id_or_username}?fields=followers_count,fan_count,name,about`. Requires Page Public Content Access.

7. **Pinterest** — No public profile API for other users. Skip with warning.

8. **Reddit** — GET `https://www.reddit.com/user/{username}/about.json` (public, no auth needed). Returns link_karma, comment_karma, created_utc.

9. **Threads** — GET `https://graph.threads.net/v1.0/{threads_user_id}?fields=username,threads_profile_picture_url,threads_biography`. Requires knowing the user's Threads ID.

**Implementation:**

```typescript
export interface CompetitorMetrics {
  platform: string;
  handle: string;
  followersCount: number | null;
  followingCount: number | null;
  postsCount: number | null;
  engagementRate: number | null;
  fetchedAt: Date;
  success: boolean;
  error?: string;
}

export async function fetchCompetitorMetrics(
  platform: string,
  handle: string,
  accessToken: string | null
): Promise<CompetitorMetrics>
```

For each platform, implement a focused fetch function that:
1. Makes one API call to get public profile metrics
2. Returns CompetitorMetrics with whatever data is available
3. Returns `success: false` with error message if platform doesn't support it or API fails
4. Never throws — always returns a result

**For unsupported platforms** (LinkedIn, TikTok, Pinterest, Threads without ID): Return `{ success: false, error: 'Platform does not support public profile lookup' }` with null metrics.

**For Reddit**: Use public endpoint (no auth needed), add User-Agent header.

**Token source**: The cron will pass the tracked competitor's owner's (the Synthex user's) access token for the relevant platform. Look up PlatformConnection for the user+platform to get the token.

Don't: Import or instantiate full platform services — this is lightweight public profile lookup, not the full sync flow.
Don't: Throw errors — always return CompetitorMetrics with success flag.
Don't: Install any new packages.

   - If engagementRate significantly changed: create informational alert
