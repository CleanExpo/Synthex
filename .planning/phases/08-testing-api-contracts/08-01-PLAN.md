  <action>
Create tests for the content generation and CRUD API routes.

**Setup:**
- Mock `@/lib/prisma` with mockPrisma object
- Mock `@/lib/services/content-generator` with mockContentGenerator
- Mock `@/lib/auth/jwt-utils` (getUserIdFromRequestOrCookies → returns 'test-user-id')
- Mock `@/lib/middleware/withAuth` to call the handler directly with userId attached
- Mock `@/lib/supabase-client`

**Test categories:**

1. **POST /api/content/generate — Input validation:**
   - Valid request with all required fields → 200 with content
   - Missing platform → 400 with validation error
   - Missing topic → 400 with validation error
   - Invalid platform name → 400 with validation error
   - Topic too long (>500 chars) → 400
   - Optional fields (personaId, tone, hookType, targetLength) accepted

2. **POST /api/content/generate — Auth:**
   - No auth token → 401 unauthorized
   - Valid auth → proceeds to generate

3. **POST /api/content/generate — Business logic:**
   - Persona lookup when personaId provided → uses persona data
   - PersonaId not found → 404 or fallback
   - Content generator returns primary + alternatives → 200 with shaped response
   - Content generator throws → 500 error response

4. **GET /api/content/[id] — CRUD:**
   - Valid id → returns content object
   - Not found → 404
   - Unauthorized (different user's content) → 403

5. **Response shape verification:**
   - Generate response has: success, content (primary, alternatives), userId, timestamp
   - Error responses have: error string, optional details

**Test count target:** 15-18 test cases.

Don't: Test the content generator internals — only the route contract.
Don't: Test AI generation quality — mock returns.
Reference: Read app/api/content/generate/route.ts for exact validation schema and response shapes.
