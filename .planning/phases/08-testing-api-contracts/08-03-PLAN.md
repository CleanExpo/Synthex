   - Valid priceId → creates Stripe checkout session, returns { sessionId, url }

   - Invalid plan name → 400 error
   - Stripe not configured (stripe = null) → 503 with bypass message
   - Unauthenticated → 401
   - Security check fails → 403
   - Stripe API error → 500
   - Verify 14-day trial period passed to Stripe
   - Verify success/cancel URL construction

   - Valid user → returns { url } for billing portal
   - User without Stripe customer → creates customer first, then portal
   - Unauthenticated → 401
   - Stripe error → 500

3. **POST /api/stripe/change-plan:**
   - Valid plan change → returns updated subscription
   - Invalid plan → 400
   - No existing subscription → 400
   - Unauthenticated → 401

4. **lib/stripe/config.ts:**
   - PRODUCTS has 3 tiers (professional, business, custom)
   - getProductByPriceId returns correct product
   - getProductByPriceId with unknown ID returns undefined
   - getProductByName returns correct product (case insensitive)
   - stripe is null when STRIPE_SECRET_KEY not set

Create tests for lib/stripe/webhook-handlers.ts and the webhook route.

- Mock `@/lib/security/audit-logger` (auditLogger.log)
- Mock `@/lib/logger`
- Create Stripe event fixtures for each event type

**Test categories:**

1. **Webhook route (app/api/webhooks/stripe/route.ts):**
   - Valid signature → 200 with { received: true, eventId }
   - Missing stripe-signature header → 400 error
   - Invalid signature (webhookHandler.receive fails) → 400 error
   - Handler throws → 500 error

2. **handleSubscriptionCreated:**
   - Valid subscription event → calls subscriptionService.updateFromStripeSubscription
   - Logs audit event
   - Returns success

3. **handleSubscriptionUpdated:**
   - Valid update → calls subscriptionService.updateFromStripeSubscription
   - Status change logged to audit

4. **handleSubscriptionCancelled:**
   - Cancellation event → calls subscriptionService.updateFromStripeSubscription with cancelled status
   - Logs cancellation audit event

5. **handlePaymentSucceeded:**
   - Invoice with subscription → extracts subscription ID, updates subscription
   - Invoice without subscription → handles gracefully (no error)
   - Tests getSubscriptionIdFromInvoice with old API format (direct .subscription)
   - Tests getSubscriptionIdFromInvoice with new API format (parent.subscription_details)

6. **handlePaymentFailed:**
   - Failed payment → logs failure, updates subscription status
   - No subscription on invoice → handles gracefully

7. **Handler registration:**
   - registerStripeWebhookHandlers() registers all 5 handlers with webhookHandler
   - Each handler registered for correct event type

**Test count target:** 18-22 test cases.
