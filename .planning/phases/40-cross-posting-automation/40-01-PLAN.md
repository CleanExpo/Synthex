**lib/ai/cross-post-service.ts:**

Create CrossPostService class that orchestrates the full cross-posting workflow:

1. Import MultiFormatAdapter, createPlatformService, prisma, decryptField
2. Define CrossPostParams interface:
   - sourceContent: string (the content to cross-post)
   - platforms: string[] (target platforms)
   - tone?: string (optional tone override)
   - goal?: string (content goal)
   - personaId?: string (brand voice persona)
   - scheduledAt?: Date (optional scheduling)
   - mediaUrls?: string[] (optional media)
   - campaignId?: string (optional campaign association)

3. Define CrossPostResult interface:
   - adaptedContent: AdaptedContent (from MultiFormatAdapter)
   - results: { platform: string; success: boolean; postId?: string; url?: string; error?: string }[]
   - scheduledCount: number (if scheduled instead of immediate)

4. Implement crossPost method:
   a. Call multiFormatAdapter.adaptContent() to get platform-specific variants
   b. For each platform:
      - Get user's PlatformConnection from database
      - Decrypt access token using decryptField
      - Create platform service via createPlatformService
      - Call service.createPost() with variant content + mediaUrls
      - Track success/failure per platform
   c. Save Post records to database (status: scheduled or published)
   d. Return combined results

5. Implement previewCrossPost method (adapt only, no posting):
   - Just calls multiFormatAdapter.adaptContent()
   - Returns AdaptedContent for UI preview

Export singleton: crossPostService

**app/api/content/cross-post/route.ts:**

POST endpoint for cross-posting:
1. withAuth wrapper for authentication
2. Zod schema:
   - content: z.string().min(1).max(50000)
   - platforms: z.array(z.enum([...SUPPORTED_PLATFORMS])).min(1)
   - tone: z.string().optional()
   - goal: z.string().optional()
   - personaId: z.string().optional()
   - scheduledAt: z.string().datetime().optional()
   - mediaUrls: z.array(z.string()).optional()
   - mode: z.enum(['preview', 'publish']).default('publish')

3. For mode='preview': Call crossPostService.previewCrossPost(), return variants only
4. For mode='publish': Call crossPostService.crossPost(), return full results

Handle errors gracefully — partial success is OK (some platforms succeed, some fail).

**WHY this approach:**
- Reuses existing MultiFormatAdapter (no duplicate adaptation logic)
- Reuses existing platform services (no duplicate posting logic)
- Clean separation: preview mode for UI preview, publish mode for actual posting
- Follows established patterns from /api/social/post and /api/content/multi-format

**hooks/use-cross-post.ts:**

1. State: content, platforms (Set), tone, goal, personaId, scheduledAt, variants, results, isLoading, error
