# Plan 114-01: Environment Variable Audit & Configuration

## Objective

Audit all production environment variables against the required list in `.env.example`, identify gaps, and configure every missing variable in Vercel's production environment before deploying.

## Execution Context

- Phase: 114 ‚Äî Vercel Production Deployment
- Plan: 1 of 2
- Model: Sonnet
- Estimated context: ~40%
- Strategy: B (segmented ‚Äî has human-action checkpoint)

## Context

**Required production env vars (from `.env.example`):**

Core (all envs):
- `DATABASE_URL` ‚Äî Supabase PostgreSQL connection string
- `DIRECT_URL` ‚Äî Direct Supabase connection (for Prisma migrations)
- `JWT_SECRET` ‚Äî Token signing (must be 32+ chars, random)
- `FIELD_ENCRYPTION_KEY` ‚Äî Field-level encryption (must be 32+ chars)
- `OAUTH_STATE_SECRET` ‚Äî OAuth CSRF protection
- `ENCRYPTION_KEY_V1` ‚Äî Legacy encryption key
- `NEXT_PUBLIC_SUPABASE_URL` ‚Äî Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` ‚Äî Supabase anon key
- `OPENROUTER_API_KEY` ‚Äî AI provider (required for any AI features)

Billing (already set in test mode from Phase 68 ‚Äî need live mode swap in Phase 115):
- `STRIPE_SECRET_KEY` ‚Äî currently `sk_test_*` ‚Üí Phase 115 swaps to `sk_live_*`
- `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` ‚Äî currently `pk_test_*`
- `STRIPE_WEBHOOK_SECRET` ‚Äî live-mode webhook secret (Phase 115)
- `STRIPE_PROFESSIONAL_PRICE_ID` ‚Äî live price ID (Phase 115)
- `STRIPE_BUSINESS_PRICE_ID` ‚Äî live price ID (Phase 115)

Additional production:
- `SENTRY_DSN` ‚Äî Error monitoring
- `RESEND_API_KEY` ‚Äî Email delivery (must be domain-verified for synthex.social)
- `CRON_SECRET` ‚Äî Cron job authentication
- `NEXT_PUBLIC_APP_URL` ‚Äî Must be `https://synthex.social`

OAuth (at least one platform required for social connections):
- `TWITTER_CLIENT_ID` + `TWITTER_CLIENT_SECRET`
- `INSTAGRAM_CLIENT_ID` + `INSTAGRAM_CLIENT_SECRET`
- `LINKEDIN_CLIENT_ID` + `LINKEDIN_CLIENT_SECRET`
- Other platforms as available

**Note:** Stripe vars stay as test keys for now. Phase 115 handles live mode swap.

**Vercel CLI is already linked** to the Synthex project (confirmed Phase 68). Run `vercel env ls --environment=production` to see current state.

## Tasks

### Task 1: Audit current production vars (auto)

Run `vercel env ls --environment=production` and compare against the required list above. Produce a clear report of:
- Present ‚úì
- Missing ‚úó

Output the audit result as a formatted checklist.

### [CHECKPOINT: human-action]

**Action required:** The audit report above shows which vars are missing. Before proceeding, the user must supply values for any missing variables.

For secrets that need to be generated fresh:
- `JWT_SECRET`: Run `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` to generate
- `FIELD_ENCRYPTION_KEY`: Same as above
- `OAUTH_STATE_SECRET`: Same as above
- `ENCRYPTION_KEY_V1`: Same as above

For values from external services:
- `DATABASE_URL` / `DIRECT_URL`: Supabase dashboard ‚Üí Project Settings ‚Üí Database ‚Üí Connection string
- `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Supabase dashboard ‚Üí Project Settings ‚Üí API
- `OPENROUTER_API_KEY`: https://openrouter.ai ‚Üí API Keys
- `SENTRY_DSN`: Sentry dashboard ‚Üí Project ‚Üí Settings ‚Üí Client Keys
- `RESEND_API_KEY`: Resend dashboard ‚Üí API Keys (verify domain first)
- `CRON_SECRET`: Generate a random string

**User confirms values are ready before plan continues.**

### Task 2: Add missing vars to Vercel + pre-deploy validation (auto)

For each missing var identified in Task 1 (with values supplied by user):
1. Run `vercel env add <VAR_NAME> production` for each
2. Set `NEXT_PUBLIC_APP_URL=https://synthex.social` if not present

Then run pre-deploy validation:
```bash
npm run type-check
npm run lint
npm test
```

All three must pass with zero errors. If any fail, fix before proceeding to Plan 114-02.
