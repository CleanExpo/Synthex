## Objective

Build the backend infrastructure for Prompt Intelligence: Prisma models to track prompts and results, a pure-TypeScript service layer for prompt generation, testing (via existing Claude API), and gap analysis, plus 4 API routes.

## Prisma Models

### PromptTracker
Track user-defined prompts for their entity.

Fields:
- `id` — cuid PK
- `userId` / `orgId` — ownership
- `entityName` — e.g. "Synthex"
- `entityType` — brand/product/service/person/location
- `promptText` — the natural language query
- `promptCategory` — brand-awareness/competitor-comparison/local-discovery/use-case/how-to/product-feature
- `targetModel` — default claude-3-5-haiku-20241022
- `status` — pending/tested/scheduled
- `brandMentioned` — null=not tested, true/false=result
- `brandPosition` — position of first mention
- `competitorsMentioned` — JSON array
- `lastTestedAt`

### PromptResult
Individual test result per tracker run.

Fields:
- `id` — cuid PK
- `trackerId` — FK to PromptTracker (cascade delete)
- `aiResponse` — full text response
- `brandMentioned` — boolean
- `brandPosition` — nullable int
- `mentionContext` — sentence containing brand mention
- `competitorsFound` — JSON array of strings
- `responseQuality` — 0-1 float
- `testedAt`

Back-relations added to User model: `promptTrackers PromptTracker[]`

## Service Layer: lib/prompts/

### types.ts
- `PromptCategory` union type (6 values)
- `PromptTemplate` interface
- `PromptGapAnalysis` interface
- `CompetitorVisibility` interface

### prompt-generator.ts
`generatePrompts(entityName, entityType, topic, location?)` → `PromptTemplate[]`
- Generates 20-30 prompts across all 6 categories using template substitution
- Pure TypeScript — no external API calls

### prompt-tester.ts
`testPrompt(prompt, entityName)` → result object
- Uses existing OpenRouter client (claude-3-5-haiku-20241022)
- Parses response for entity mention, position, context, competitors
- Returns brandMentioned, brandPosition, mentionContext, competitorsFound, responseQuality

### gap-analyzer.ts
`analyzeGaps(trackers, results)` → `PromptGapAnalysis`
- Pure computation — no DB calls
- Calculates coverage rate per category
- Identifies gap categories with lowest mention rates
- Generates recommendations

## API Routes

### GET/POST /api/prompts/trackers
- GET: list user's trackers with optional filters
- POST: create new tracker

### POST /api/prompts/generate
- Accept: entityName, entityType, topic, location?
- Return: generated prompt templates (not saved to DB)
- No rate limit (pure computation)

### POST /api/prompts/test
- Accept: trackerId
- Run prompt through Claude, parse result, save PromptResult, update tracker
- Rate limit: 10/hour using existing RateLimiter
- Feature gate: promptTests

### GET /api/prompts/gaps
- Fetch user's tested trackers + results
- Run gap analyzer
- Return PromptGapAnalysis
