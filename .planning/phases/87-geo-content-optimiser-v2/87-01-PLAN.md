Create `lib/geo/tactic-scorer.ts` with 9 pure scoring functions + orchestrator.

Each function receives `(text: string): number` returning 0–100.

**Function implementations (from 87-RESEARCH.md algorithms):**

```typescript
// 1. Authoritative Citations (+40%): named references per 200 words
// Patterns: "According to [Source]", "(Author, YYYY)", "per the [Org]"
// 4 citations per 200 words = 100. Scale linearly.

// 2. Statistics (+37%): specific numbers with context
// Match: percentages, numbers with units (million/billion/thousand)
// 3 stats per 200 words = 100.

// 3. Quotations (+30%): attributed direct quotes >20 chars
// Patterns: "quoted" — Name or > blockquote marker
// 1 quote per 300 words = 100.

// 4. Fluency (+25%): inverted hedge density
// Hedge words: might, perhaps, could possibly, seems to, appears to,
//              it seems, maybe, possibly, arguably, somewhat, rather, perhaps
// 0 hedges/100 words = 100. 5+ hedges/100 words = 0. Linear interpolation.

// 5. Readability (+20%): sentence length proxy
// Split on [.!?]+. Average words per sentence.
// 15-18 words avg = 100. <10 or >25 = 50. >35 = 20. Linear between bands.

// 6. Technical Vocabulary (+18%): defined terms and domain abbreviations
// Patterns: "Term (ABBR)", "ABBR — definition", standalone 2-5 char ALLCAPS
// Weighted: defined term = 20pts, abbreviation usage = 5pts each. Cap at 100.

// 7. Uniqueness (+15%): type-token ratio
// words = text.toLowerCase().split(/\s+/)
// TTR = uniqueWords.size / words.length
// TTR > 0.65 = 100, TTR < 0.35 = 10. Linear between.

// 8. Information Flow (+15-30%): transition phrase density
// Transitions: furthermore, moreover, however, therefore, consequently, in addition,
//              building on, beyond that, as a result, this means, specifically,
//              for example, notably, additionally, in contrast
// 1 transition per paragraph = 100. 0 = 20. Cap at 100.

// 9. Persuasion (+/-): inverted keyword density penalty
// Count 3-gram repetitions. Max repetitions found:
// 0-2 = 100, 3-4 = 70, 5-6 = 40, 7+ = 20.
```

The main export is:
```typescript
export function scoreTactics(text: string): TacticScoreResult
```

Which runs all 9 scorers and:
1. Returns a `TacticScore[]` array (all 9 tactics)
2. Calculates `compositeGEOScore` as weighted average:
   - citations: 20%, statistics: 18%, quotations: 15%, fluency: 12%,
   - readability: 10%, technical-vocabulary: 9%, uniqueness: 8%,
   - information-flow: 5%, persuasion: 3%
3. Sets `status` based on score: green ≥70, amber 40–69, red <40
4. Sets `explanation` and `suggestions` based on score and specific findings

Each tactic must have hardcoded label, explanation templates, and suggestions.
Example for citations:
- Green: "Good citation density. Continue adding named sources."
- Amber: "Add 1-2 more named citations per section."
- Red: "Add named citations (e.g., 'According to [Source], [Year]') to every major point."

Suggestions should be specific and actionable (1-3 items per tactic).

Guard: if `text.length < 50`, return all scores at 50 (neutral) with explanation "Add more content to score".
