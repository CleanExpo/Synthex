# Phase 28 Plan 01: Comments & Shares API + Hooks

## Objective

Create CRUD API routes for ContentComment and ContentShare models, plus React hooks for state management.

## Execution Context

**Models already exist in Prisma schema:**
- ContentComment (id, contentType, contentId, content, parentId, authorId, sentiment, sentimentScore, emotions, resolved, resolvedBy, resolvedAt, mentions, attachments, createdAt, updatedAt)
- ContentShare (id, contentType, contentId, sharedWithUserId, sharedWithTeamId, sharedWithEmail, permission, canDownload, canReshare, expiresAt, accessLink, accessLinkExpiry, message, sharedById, createdAt, revokedAt, revokedReason)

**Hook pattern:** Follow `hooks/use-webhooks.ts` pattern (raw fetch + useState, AbortController, TypeScript types)

**API pattern:** Follow `app/api/webhooks/user/route.ts` and `app/api/webhooks/user/[id]/route.ts` pattern

## Tasks

### Task 1: Create Comments CRUD API

**Files to create:**
- `app/api/comments/route.ts` - GET (list by content), POST (create)
- `app/api/comments/[id]/route.ts` - GET, PATCH, DELETE

**GET /api/comments query params:**
- `contentType` (required): campaign, post, calendar_post, project
- `contentId` (required): ID of the content
- `parentId` (optional): For fetching thread replies

**POST /api/comments body:**
- `contentType`, `contentId`, `content` (required)
- `parentId` (optional): For threaded replies
- `mentions` (optional): Array of user IDs

**PATCH /api/comments/[id] body:**
- `content` (optional): Edit comment text
- `resolved` (optional): Mark as resolved
- `resolvedBy` (optional): User who resolved

**Auth:** Require session, extract userId from session
**Validation:** Zod schemas for request bodies
**Error handling:** Standard pattern with try/catch, 401/400/404/500 responses

### Task 2: Create Shares CRUD API

**Files to create:**
- `app/api/shares/route.ts` - GET (list by content or user), POST (create)
- `app/api/shares/[id]/route.ts` - GET, PATCH (update permission), DELETE (revoke)

**GET /api/shares query params:**
- `contentType` + `contentId`: List shares for specific content
- OR `sharedWithMe=true`: List content shared with current user
- OR `sharedByMe=true`: List content current user shared

**POST /api/shares body:**
- `contentType`, `contentId` (required)
- `sharedWithUserId` OR `sharedWithTeamId` OR `sharedWithEmail` (one required)
- `permission`: view, comment, edit, admin (default: view)
- `canDownload`, `canReshare`, `expiresAt`, `message` (optional)

**PATCH /api/shares/[id] body:**
- `permission`, `canDownload`, `canReshare`, `expiresAt` (all optional)

**DELETE /api/shares/[id]:**
- Sets `revokedAt` and optionally `revokedReason` from query param

**Auth:** Require session
**Access control:** Only share owner or admin can modify

### Task 3: Create useComments Hook

**File:** `hooks/use-comments.ts`

**Exports:**
- `ContentComment` type (matches API response)
- `CreateCommentData`, `UpdateCommentData` types
- `useComments(contentType: string, contentId: string)` hook

**Hook state:**
- `comments: ContentComment[]`
- `loading: boolean`
- `error: string | null`

**Hook actions:**
- `refresh()`: Fetch comments for content
- `create(data: CreateCommentData): Promise<ContentComment>`
- `update(id: string, data: UpdateCommentData): Promise<void>`
- `remove(id: string): Promise<void>`
- `resolve(id: string): Promise<void>`
- `unresolve(id: string): Promise<void>`

**Pattern:** Follow use-webhooks.ts (AbortController, useEffect fetch, callbacks)

### Task 4: Create useShares Hook

**File:** `hooks/use-shares.ts`

**Exports:**
- `ContentShare` type (matches API response)
- `CreateShareData`, `UpdateShareData` types
- `ShareFilter`: { contentType?: string; contentId?: string; sharedWithMe?: boolean; sharedByMe?: boolean }
- `useShares(filter: ShareFilter)` hook

**Hook state:**
- `shares: ContentShare[]`
- `loading: boolean`
- `error: string | null`

**Hook actions:**
- `refresh()`: Fetch shares based on filter
- `create(data: CreateShareData): Promise<ContentShare>`
- `update(id: string, data: UpdateShareData): Promise<void>`
- `revoke(id: string, reason?: string): Promise<void>`

**Pattern:** Follow use-webhooks.ts
