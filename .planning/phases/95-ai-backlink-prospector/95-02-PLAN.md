## Overview

Build the Link Prospector dashboard UI, four reusable components, and add navigation entries to the sidebar and command palette.

## Components

### components/backlinks/BacklinkProspectCard.tsx

Card showing prospect details with action buttons.

Fields displayed:
- Domain name (bold, large)
- Opportunity type badge (colour-coded):
  - resource-page → blue
  - guest-post → green
  - broken-link → amber
  - competitor-link → purple
  - journalist-mention → cyan
- Domain authority score (0-100) with colour coding:
  - 70+ → green (high authority)
  - 40-69 → amber (medium)
  - < 40 → red/grey (low)
- Status badge: identified / contacted / published / rejected
- Target URL (truncated, linked)
- Discovered date

Action buttons:
- "View Details" — expand/modal
- "Create Outreach" — opens OutreachPanel
- "Mark Published" — PATCH status to 'published'

Props: `{ prospect: BacklinkProspect, onOutreach: (id: string) => void, onMarkPublished: (id: string) => void }`

### components/backlinks/OpportunityMatrix.tsx

Matrix grid showing prospects by type × domain authority tier.

X-axis: opportunity types (resource-page, guest-post, broken-link, competitor-link, journalist-mention)
Y-axis: DA tiers:
- High: DA ≥ 70
- Medium: DA 40–69
- Low: DA < 40

Each cell shows prospect count with click-to-filter behaviour.
Highlighted cells (> 3 prospects) use coloured backgrounds.
Click → emits `onFilterChange({ type, tier })` to parent.

Props: `{ prospects: BacklinkProspect[], onFilterChange: (filter: MatrixFilter) => void, activeFilter: MatrixFilter | null }`

### components/backlinks/OutreachPanel.tsx

Outreach email editor and tracker.

Features:
- Template selector (4 types → dropdown)
- Pre-filled email fields: To (outreachEmail), Subject, Body
- Body is pre-filled from outreach-template on the client
- "Copy to clipboard" button (copies formatted email)
- "Mark as Contacted" button — POST /api/backlinks/outreach to update status
- Shows confirmation on success

State:
- selectedType: BacklinkOpportunityType
- emailBody: string (editable textarea)
- subject: string (editable input)
- copied: boolean (feedback state)
- saving: boolean

Props: `{ prospect: BacklinkProspect | null, brandName: string, onClose: () => void, onOutreachSaved: () => void }`

### components/backlinks/BacklinkAnalysisSummary.tsx

Summary card for a completed analysis.

Displays:
- Topic/domain used for the search
- Total opportunities found
- By-type breakdown (horizontal bar segments or simple list with counts)
- High-value count (DA ≥ 40) with green highlight
- Date run
- "View Prospects" button → navigates to prospects tab with filter

Props: `{ analysis: BacklinkAnalysis }`

## Dashboard Page

### app/dashboard/backlinks/page.tsx

3-tab layout using Radix UI Tabs.

#### Tab 1: Prospects

Top section:
- Form: topic input + optional userDomain input + optional competitorDomain input
- "Find Opportunities" button → POST /api/backlinks/analyze (shows loading spinner)
- Analysis result summary appears after run

Middle section:
- Filter bar: status dropdown, type dropdown, DA tier dropdown
- OpportunityMatrix (compact, above prospect list)

Main section:
- Grid of BacklinkProspectCards (fetched via useSWR from GET /api/backlinks/prospects)
- Pagination (limit 20, offset-based)
- Empty state with prompt to run first analysis

#### Tab 2: Outreach

- Prospect selector (dropdown of all identified/contacted prospects)
- OutreachPanel for selected prospect
- Outreach history table below (all contacted prospects with date + status)

#### Tab 3: Analysis

- List of past BacklinkAnalysis records (fetched via useSWR from GET /api/backlinks/analysis)
- Each record rendered as BacklinkAnalysisSummary card
- Empty state with prompt to run first analysis
