### SYN-2: Unite-Hub Connector

**Connector pattern** (`lib/unite-hub-connector.ts`):
- Fire-and-forget: `void pushEvent(event)` — never throws, logs errors only
- Uses `UNITE_HUB_API_KEY` + `UNITE_HUB_API_URL` env vars
- If env vars missing: no-op (safe for local dev)

**Events to push:**
```ts
type UniteHubEvent =
  | { type: 'user.signup'; userId: string; plan: string; email: string }
  | { type: 'user.upgrade'; userId: string; fromPlan: string; toPlan: string }
  | { type: 'user.churn'; userId: string; plan: string }
  | { type: 'payment.received'; userId: string; amount: number; currency: string }
  | { type: 'content.published'; userId: string; platform: string; postId: string }
  | { type: 'campaign.started'; userId: string; campaignId: string }
  | { type: 'campaign.completed'; userId: string; campaignId: string }
```

**Pull endpoint** (`app/api/unite-hub/route.ts`):
- GET — authenticated via `x-unite-hub-api-key` header
- Returns:
```json
{
  "health": {
    "stripeConnected": true,
    "activeUsers": 42,
    "mrr": 12450,
    "postsPublishedToday": 87,
    "apiResponseTimeMs": 45,
    "timestamp": "2026-03-10T..."
  }
}
```
- `activeUsers`: count users with subscription status active in last 30 days
- `mrr`: sum of Stripe subscription amounts (from Prisma Subscription model)
- `postsPublishedToday`: count PlatformPost where publishedAt >= today

**Daily revenue cron** (`app/api/cron/unite-hub-revenue/route.ts`):
- Auth: `CRON_SECRET` header (same pattern as other crons — check `app/api/cron/health-score/route.ts`)
- Pushes: `{ type: 'revenue.daily', mrr: N, customers: N, newCustomers: N, churned: N, byTier: {...} }`
- Schedule: `0 6 * * *` (6am daily) — add to `vercel.json`

**Event integration points** (add fire-and-forget calls):
- `app/api/webhooks/stripe/route.ts` — on `checkout.session.completed` → `user.signup` or `user.upgrade`; on `customer.subscription.deleted` → `user.churn`; on `invoice.payment_succeeded` → `payment.received`
- `app/api/posts/[id]/publish/route.ts` OR check existing publish flow → `content.published`
- Campaign start/complete → find campaign state transitions in API routes

### Task 2: Unite-Hub connector library (SYN-2)

Create `lib/unite-hub-connector.ts`:

```ts
const UNITE_HUB_URL = process.env.UNITE_HUB_API_URL;
const UNITE_HUB_KEY = process.env.UNITE_HUB_API_KEY;

export type UniteHubEvent = { type: string; [key: string]: unknown };

export async function pushUniteHubEvent(event: UniteHubEvent): Promise<void> {
  if (!UNITE_HUB_URL || !UNITE_HUB_KEY) return; // no-op in local dev
  try {
    await fetch(`${UNITE_HUB_URL}/api/events`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': UNITE_HUB_KEY,
      },
      body: JSON.stringify({ ...event, source: 'synthex', timestamp: new Date().toISOString() }),
    });
  } catch {
    // fire-and-forget: never throw, only log
    console.error('[unite-hub] Failed to push event:', event.type);
  }
}
```

Add to `.env.example`:
```
UNITE_HUB_API_URL=https://unite-hub.unite-group.com.au
UNITE_HUB_API_KEY=your-unite-hub-api-key
```
