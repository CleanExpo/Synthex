# Plan 80-02: Brand Profile UI

## objective

Build a "Brand Profile" settings page under `/dashboard/settings/brand-profile` that lets users view and edit their organisation's brand identity after onboarding. Currently there is no post-onboarding UI for updating organisation fields (name, logo, primary colour, website, industry, team size, ABN, description, social handles). The existing `BrandingTab` in Settings is enterprise-only white-label config — this is a separate, always-available brand identity page for all users.

Linear: SYN-55
Phase dir: `.planning/phases/80-brand-profile-setup/`

---

## execution_context

- New API from 80-01: `GET /api/brand-profile` and `PATCH /api/brand-profile`
- Settings page: `app/dashboard/settings/page.tsx` — Tabs-based layout (profile, notifications, integrations, privacy, billing, branding, advanced)
- Media upload: `app/api/media/upload/route.ts` (POST, multipart/form-data) — returns `{ data: { url } }` (from Phase 77)
- Supabase storage helper: `lib/storage/supabase-storage.ts` — used by media upload route, 10MB image limit
- Component patterns: `components/settings/branding-tab.tsx` — glassmorphic Card layout, Zod-style inline validation, `credentials: 'include'` fetch, fire-and-forget save pattern
- Icon barrel: `@/components/icons` — `Building2, Globe, Palette, Users, Upload, Save, Loader2, X, AlertCircle`
- Toast: `sonner` — `toast.success()` / `toast.error()`
- SWR rule: client components use `useSWR` with `credentials: 'include'` fetcher — not raw `fetch()` in component body
- Form submission: use `fetch` in `handleSave` callback (mutation), `useSWR` for initial load only

## context

### Discovery Findings

**Gap 1 — No dedicated brand profile page**: After onboarding, the only way to see brand data is in the sidebar org switcher name. No UI exists to update `logo`, `primaryColor`, `website`, `industry`, `description`, `socialHandles`, `abn`, or `teamSize`.

**Gap 2 — Settings tabs are already crowded**: Adding a new "Brand Profile" tab to the existing settings tabs list would push the row past comfortable width on mobile. The correct approach is a standalone `/dashboard/settings/brand-profile` page accessible from a link in the Settings nav, matching the `settings/accounts` pattern.

**Gap 3 — Logo upload needs integration**: The `Organization.logo` is a URL field. Users should be able to type a URL or click-to-upload from their device. Upload goes through `POST /api/media/upload` (existing) which returns a public Supabase Storage URL. The brand profile form then PATCHes that URL into the org.

### Architecture Decision

- **Page**: `app/dashboard/settings/brand-profile/page.tsx` — server wrapper, exports metadata
- **Client component**: `components/settings/brand-profile-tab.tsx` — all state and fetch logic
- **Hook**: `hooks/use-brand-profile.ts` — SWR data fetching + mutation helper
- **Navigation**: Add "Brand Profile" link to Settings page sidebar nav or as a card in the profile tab
- **Form pattern**: Match `components/settings/branding-tab.tsx` — local state from SWR data, explicit save button, inline validation errors, toast feedback
- **Logo upload**: Drag-and-drop not required — a simple file input button that calls `POST /api/media/upload` and populates the logo URL field
- **Live preview**: Show a small brand card preview (org name + logo + primary colour) that updates as the user types — same pattern as `BrandingTab` live preview

## tasks

### Task 1: SWR hook — `hooks/use-brand-profile.ts`

**File**: `hooks/use-brand-profile.ts`

```ts
import useSWR from 'swr';
import type { BrandProfileResponse, BrandProfileUpdatePayload } from '@/app/api/brand-profile/types';

const fetchJson = (url: string) =>
  fetch(url, { credentials: 'include' }).then((r) => r.json());

export function useBrandProfile() {
  const { data, error, isLoading, mutate } = useSWR<{ data: BrandProfileResponse }>(
    '/api/brand-profile',
    fetchJson,
    { revalidateOnFocus: false }
  );

  const updateBrandProfile = async (payload: BrandProfileUpdatePayload): Promise<BrandProfileResponse> => {
    const res = await fetch('/api/brand-profile', {
      method: 'PATCH',
      credentials: 'include',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload),
    });
    const json = await res.json();
    if (!res.ok) throw new Error(json.error || 'Failed to update brand profile');
    await mutate();
    return json.data as BrandProfileResponse;
  };

  return {
    profile: data?.data ?? null,
    isLoading,
    error,
    mutate,
    updateBrandProfile,
  };
}
```

**Note:** The SWR fetcher uses `credentials: 'include'` per project rule. The `updateBrandProfile` mutation is a plain `async` function (not SWR) because mutations use `fetch()` in callbacks per established codebase pattern.

**Logo upload flow:**

```tsx
const handleLogoUpload = async (file: File) => {
  setIsUploadingLogo(true);
  try {
    const formData = new FormData();
    formData.append('file', file);
    const res = await fetch('/api/media/upload', {
      method: 'POST',
      credentials: 'include',
      body: formData,
    });
    const json = await res.json();
    if (!res.ok) throw new Error(json.error || 'Upload failed');
    handleChange('logo', json.data.url);
    toast.success('Logo uploaded successfully');
  } catch (error) {
    toast.error(error instanceof Error ? error.message : 'Failed to upload logo');
  } finally {
    setIsUploadingLogo(false);
  }
};
```

**Save handler:**

```tsx
const handleSave = async () => {
  if (!validate()) return;
  setIsSaving(true);
  try {
    await updateBrandProfile({
      name: localProfile.name,
      description: localProfile.description,
      website: localProfile.website,
      industry: localProfile.industry,
      teamSize: localProfile.teamSize,
      abn: localProfile.abn,
      logo: localProfile.logo,
      favicon: localProfile.favicon,
      primaryColor: localProfile.primaryColor,
      socialHandles: localProfile.socialHandles,
    });
    toast.success('Brand profile saved');
  } catch (error) {
    toast.error(error instanceof Error ? error.message : 'Failed to save brand profile');
  } finally {
    setIsSaving(false);
  }
};
```

**Validation rules:**
- `name`: required, max 100 chars
- `primaryColor`: optional, must match `/^#[0-9A-Fa-f]{6}$/` if provided
- `website`: optional, must be valid URL if provided (`new URL()` try/catch)
- `abn`: optional, max 20 chars
- `description`: optional, max 500 chars

**Loading state:** Render `<Loader2>` spinner card (same as `BrandingTab`) while SWR is loading.
