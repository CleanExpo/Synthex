**ContentRepurposer service (lib/ai/content-repurposer.ts):**

Create a class following MultiFormatAdapter pattern:

1. **Input types:**
   - sourceContent: string (the long-form content)
   - sourceType: 'blog' | 'article' | 'video_transcript' | 'podcast' | 'newsletter'
   - outputFormats: string[] (formats to generate)

2. **Output formats (6 total):**
   - `thread` - Twitter/X thread (numbered tweets with key points)
   - `video_script` - Short-form video script (TikTok/Reels/Shorts format with hooks, body, CTA)
   - `carousel_outline` - Instagram/LinkedIn carousel (slide-by-slide outline)
   - `key_takeaways` - Bulleted list of main insights (5-7 points)
   - `summary` - Condensed summary paragraph (150-250 words)
   - `quote_graphics` - 3-5 quotable statements for graphics

3. **Implementation:**
   - Use getAIProvider() for AI calls (same as multi-format)
   - Use ai.models.balanced for generation
   - Create system prompts per output format
   - Use Promise.allSettled for parallel generation
   - Score each output with contentScorer
   - Return array of RepurposedContent objects

4. **Types:**
   ```typescript
   interface RepurposeContentParams {
     sourceContent: string;
     sourceType: 'blog' | 'article' | 'video_transcript' | 'podcast' | 'newsletter';
     outputFormats: string[];
   }

   interface RepurposedContent {
     format: string;
     content: string;
     metadata: {
       wordCount: number;
       characterCount: number;
       itemCount?: number; // for threads, carousels, takeaways
       formatDetails?: Record<string, unknown>;
     };
     score?: number;
   }
   ```

**API endpoint (app/api/content/repurpose/route.ts):**

1. POST handler with:
   - withAuth wrapper (from lib/auth)
   - Zod validation for request body:
     - sourceContent: z.string().min(100).max(50000)
     - sourceType: z.enum(['blog', 'article', 'video_transcript', 'podcast', 'newsletter'])
     - outputFormats: z.array(z.enum(['thread', 'video_script', 'carousel_outline', 'key_takeaways', 'summary', 'quote_graphics'])).min(1).max(6)
   - Call contentRepurposer.repurpose(params)
   - Return JSON with results array

2. Error handling: Try-catch with appropriate status codes (400 for validation, 500 for AI errors)

**What to avoid:**
- Don't use jsonwebtoken (use jose if JWT needed - CommonJS issues)
- Don't duplicate AI provider instantiation (use getAIProvider singleton)
- Don't skip contentScorer integration (quality scores are valuable)
