<task id="2" name="Create lib/quality/humanness-scorer.ts">
Composite humanness score engine. Combines slop scan + fingerprint (if sufficient text).

Export interface HumannessResult:
```typescript
export interface HumannessResult {
  score: number;           // 0-100, higher = more human
  grade: 'A' | 'B' | 'C' | 'D' | 'F';
  passThreshold: number;   // configurable, default 60
  passes: boolean;         // score >= passThreshold
  components: {
    slopPenalty: number;   // 0-40 penalty from slop density
    ttrBonus: number;      // 0-20 bonus from vocabulary diversity
    rhythmBonus: number;   // 0-20 bonus from varied sentence lengths
    readabilityBonus: number; // 0-20 bonus from good Flesch score
  };
  slopScan: SlopScanResult;
  fingerprint?: VoiceFingerprint;
  wordCount: number;
  issues: string[];         // Human-readable issues list
  suggestions: string[];    // Actionable improvements
}
```

Scoring algorithm:
1. Start at 100
2. Slop penalty: -10 per error match, -4 per warning match, capped at -40
3. If fingerprint available (≥200 words):
   - TTR bonus: if ttr > 0.55 +20, if ttr > 0.45 +10, else +0
   - Rhythm bonus: if sentenceLengths.stdDev > 4 +20, if > 2 +10, else +0
   - Readability bonus: if fleschReadingEase 30-70 +20, if 20-80 +10, else +0
4. Floor at 0, ceiling at 100

Grade mapping:
- 90-100: A
- 75-89: B
- 60-74: C
- 40-59: D
- 0-39: F

Issues list (auto-generated from results):
- If errorCount > 0: "Found X high-confidence AI phrases (delve, robust, leverage...)"
- If slopDensity > 3: "Slop density X.X/100 words is above the 3.0 error threshold"
- If ttr < 0.40: "Low vocabulary diversity (TTR X.XX) — suggests repetitive word choice"
- If stdDev < 2: "Very uniform sentence lengths (stdDev X.X) — suggests machine generation"
- If fleschReadingEase > 80: "Readability score unusually high — may indicate oversimplified AI output"

Export function: scoreHumanness(text: string, threshold?: number): HumannessResult
</task>
