<task id="1" name="Create components/quality/HumannessScoreCard.tsx">
Displays a HumannessResult with large score display and component breakdown.

Layout:
- Large centre score: number (0-100) + grade letter (A/B/C/D/F) + pass/fail badge
- Pass badge: green "Passes Quality Gate" or red "Below Threshold ({score}/{threshold})"
- Score components bar (4 items, horizontal):
  - Slop Penalty: shows -X reduction in red
  - Vocabulary Bonus: shows +X in green if ttr bonus
  - Rhythm Bonus: shows +X in green if rhythm bonus
  - Readability Bonus: shows +X in green if readability bonus
- Issues list: each issue as a red-tinted alert row
- Suggestions list: each suggestion as an amber-tinted tip row

Colour coding:
- A (90-100): emerald
- B (75-89): cyan
- C (60-74): amber
- D (40-59): orange
- F (0-39): red

Props: { result: HumannessResult; className?: string }
</task>

<task id="2" name="Create components/quality/QualityGateBadge.tsx">
Small inline badge for use in content editors and publishing flows.
Shows a compact pass/fail status with score.

Variants:
- 'inline' (default): small coloured badge "Quality: A (94)" or "Quality: F (28)"
- 'button': clickable badge that opens a tooltip with top 2 issues + "Run Full Audit" link

Props: { score: number; grade: string; passes: boolean; variant?: 'inline' | 'button' }

This component is small but important — it's the integration point other parts of the
dashboard can drop in next to publish buttons.
</task>

<task id="3" name="Create app/dashboard/quality/page.tsx">
Content Quality Gate dashboard. Simpler than voice — single workflow, two tabs.

Tabs: "Quality Audit" | "Audit History"

Quality Audit Tab:
- Full-width textarea for content input
- Word count display (live)
- Threshold slider: 40-80 (default 60) — "Pass threshold"
- "Run Quality Audit" button
- Results below: HumannessScoreCard + SlopScanResults (reused from Phase 88)
- If content score available, also show writingQuality dimension from content-scorer

Audit History Tab:
- List of saved audits from /api/quality/audit (GET with userId filter)
- Each row: date, word count, score, grade, pass/fail
- Click to expand: shows full audit result
- Note: Only audits where save=true were persisted

State: useApi for POST to /api/quality/audit
For history: useSWR on a GET endpoint if it exists, otherwise show "No saved audits yet"

Note: Create a GET /api/quality/audit/route.ts that fetches user's ContentQualityAudit records
as part of this task (simple read endpoint, no new Prisma model needed).
</task>
