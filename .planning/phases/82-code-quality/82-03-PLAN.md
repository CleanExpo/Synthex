## context

### `any` Type Strategy

**Approach by category:**

| Pattern | Replacement |
|---------|-------------|
| `prisma.$transaction` callback with `any[]` args | `Parameters<typeof prisma.$transaction>[0]` or specific Prisma types |
| `catch (error)` with `: any` annotation | Remove annotation — catch variable is `unknown` in strict mode |
| `JSON.parse(x) as any` | Use `unknown` then narrow with type guard or Zod |
| `record[key] as any` | Use `Record<string, unknown>` then cast to known type |
| Third-party library with poor types | Add `// eslint-disable-next-line @typescript-eslint/no-explicit-any` with justification comment |

**Files with existing ESLint suppression** (review, keep if justified):
- `app/api/auth/oauth/google/callback/route.ts` — line 24
- `app/api/cron/publish-scheduled/route.ts` — line 63
- `app/api/library/content/[contentId]/route.ts` — line 171
- `app/api/webhooks/social/route.ts` — lines 408, 544

These may be legitimate suppressions where third-party types are genuinely weak. Document the justification in the inline comment.
