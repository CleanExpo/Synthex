    **Step 1 — lib/performance.ts: Fix collectWebVitals**

    The current implementation creates a fresh zeroed `metrics` object on every call, sets one field, and sends it — meaning each report has 5 zeroes and 1 real value.

    Fix by removing the internal metrics object construction and instead sending the single metric directly. The `sendMetricsToAnalytics` function should accept a single named metric rather than the full PerformanceMetrics shape:

    Replace the `collectWebVitals` function:
    ```typescript
    export function collectWebVitals(metric: WebVitalsMetric) {
      if (process.env.NODE_ENV === 'production') {
        sendMetricToAnalytics(metric);
      }
    }

    async function sendMetricToAnalytics(metric: WebVitalsMetric) {
      try {
        const body = {
          metrics: {
            [metric.name.toLowerCase()]: metric.value,
          },
          url: window.location.href,
          timestamp: new Date().toISOString(),
          userAgent: navigator.userAgent,
          rating: metric.rating,
          metricId: metric.id,
        };

        if (navigator.sendBeacon) {
          // sendBeacon is fire-and-forget, works during page unload
          navigator.sendBeacon(
            '/api/analytics/performance',
            new Blob([JSON.stringify(body)], { type: 'application/json' })
          );
        } else {
          await fetch('/api/analytics/performance', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(body),
            keepalive: true,
          });
        }
      } catch {
        // Never let vitals reporting throw — it's best-effort
      }
    }
    ```

    Remove the old `sendMetricsToAnalytics` function (the one with the broken fresh-object pattern).
    Keep all other exports (analyzeResourceTiming, monitorMemoryUsage, VITALS_THRESHOLDS, etc.) unchanged.

    **Step 2 — next.config.mjs: Add resource hints**

    In the `async headers()` return array, add a new entry for the root path to include a `Link` preload header for the Google Fonts preconnect (supplements the `<link rel="dns-prefetch">` already in the HTML):

    Add to the headers array in next.config.mjs:
    ```javascript
    {
      source: '/:path*',
      headers: [
        {
          key: 'Link',
          value: '<https://fonts.googleapis.com>; rel=preconnect, <https://fonts.gstatic.com>; rel=preconnect; crossorigin',
        },
      ],
    },
    ```

    This ensures preconnect hints are sent as HTTP headers (browser can act on them before parsing HTML).
