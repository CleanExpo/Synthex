## context

### Existing GEO Architecture

The GEO engine has 5 scoring dimensions with weights summing to exactly 1.0:
- Citability 25%, Structure 20%, Multi-Modal 15%, Authority 20%, Technical 20%

`entityCoherence` is added as a **standalone metric** (not part of the overall weighted score — preserves backward compatibility with existing stored scores and client expectations). It is displayed separately in the UI as a diagnostic metric that informs authority and citability.

### Entity Extraction Strategy

No NLP library available — implement pattern-based extraction (same approach as the rest of `lib/geo/`):

**Entity patterns** (regex-based classification):
- **PERSON**: Capitalised two-word sequences matching `[A-Z][a-z]+ [A-Z][a-z]+` not starting sentences
- **ORGANISATION**: Known org suffixes (Inc, Ltd, LLC, Corp, Foundation, Institute, University, Association) + all-caps acronyms 3+ chars
- **LOCATION**: Country/city patterns (requires a seed list of top-50 countries/cities), plus `in [Capitalised]`, `at [Capitalised]`
- **CONCEPT**: Capitalised single nouns that repeat 2+ times (indicates intentional entity use)

**Entity density calculation**:
- Proper noun density = (entities found / total words) × 100
- Target: 15-25 entities, ~20.6% proper noun density
- Coherence penalty: entities with inconsistent naming (e.g., "Apple" vs "Apple Inc" vs "the company" referring to same entity)

**Coherence scoring** (0-100):
- Base: min(entities.length / 15, 1) × 50 (density component)
- Diversity: min(uniqueEntityTypes / 3, 1) × 25 (type variety)
- Consistency: (1 - ambiguityRatio) × 25 (naming consistency)

### Prisma Changes (backward-compatible only)

GEOAnalysis: Add `entityCoherenceScore Float? @map("entity_coherence_score")` — optional so existing records remain valid.

New `EntityAnalysis` model (separate table for detailed entity data):
- Linked to `GEOAnalysis` by id
- Stores full entity extraction result as JSON
- Used by Phase 85-02 dashboard

### Key file patterns to follow
- All `lib/geo/*.ts` modules export a single pure function
- Types defined in `lib/geo/types.ts`, imported by other modules

### Task 2: Create `lib/geo/entity-analyzer.ts`

New file — pure function, no imports from external NLP libraries. Follow exact same module structure as `structure-analyzer.ts`.

```typescript
/**
 * GEO Entity Coherence Analyzer
 *
 * Extracts named entities from content and scores entity density + coherence.
 * Research basis: Princeton KDD 2024 — 15+ entities = 4.8x higher citation rate.
 * Optimal proper noun density: ~20.6% of total words.
 *
 * @module lib/geo/entity-analyzer
 */

import type { EntityAnalysisResult, EntityMention, EntityType } from './types';

// --- Constants ---
const ORG_SUFFIXES = /\b(Inc|Ltd|LLC|Corp|Corporation|Foundation|Institute|University|Association|Agency|Group|Holdings|Technologies|Solutions|Services|Systems|Labs|Studio|Studios|Media|Partners|Ventures|Capital|Fund)\b/;
const ACRONYM_PATTERN = /\b[A-Z]{3,}\b/g;
const PERSON_PATTERN = /\b[A-Z][a-z]+\s+[A-Z][a-z]+(?:\s+[A-Z][a-z]+)?\b/g;  // "John Smith" or "John D Smith"
const LOCATION_SEEDS = new Set([/* top 50 countries + major cities — see below */]);

// Export the main function
export function analyzeEntities(text: string): EntityAnalysisResult { ... }

// Private helpers
function extractPersons(text: string): EntityMention[] { ... }
function extractOrganisations(text: string): EntityMention[] { ... }
function extractLocations(text: string): EntityMention[] { ... }
function extractConcepts(text: string, allEntities: EntityMention[]): EntityMention[] { ... }
function detectCoherenceIssues(entities: EntityMention[], text: string): string[] { ... }
function calculateEntityScore(entities: EntityMention[], wordCount: number, issues: string[]): number { ... }
```

**Implementation details for `analyzeEntities(text)`**:

1. Strip HTML tags from text before processing: `text.replace(/<[^>]+>/g, ' ')`
2. Count total words: `text.split(/\s+/).filter(Boolean).length`
3. Extract persons via `PERSON_PATTERN` — filter out sentence-starting capitals (where preceding char is `.` or start of string), deduplicate, count occurrences
4. Extract organisations via `ORG_SUFFIXES` — find all words immediately before the suffix, plus `ACRONYM_PATTERN` matches
5. Extract locations via `LOCATION_SEEDS` intersection — find capitalised words/phrases that match seed list
6. Extract concepts — capitalised single nouns (not persons/orgs/locations) that appear 2+ times
7. Merge all entities, sort by count DESC
8. Detect coherence issues:
   - Check if any entity appears with 2+ different variants (e.g., both "Apple" and "Apple Inc")
   - Check for pronouns near entity mentions that suggest ambiguous reference ("it", "they", "the company" within 100 chars of an org entity)
9. Score calculation:
   - `densityScore = Math.min(entities.length / 15, 1) * 50`
   - `diversityScore = Math.min(uniqueTypes / 3, 1) * 25`
   - `consistencyScore = Math.max(0, 1 - issues.length * 0.15) * 25`
   - `total = Math.round(densityScore + diversityScore + consistencyScore)`
10. Return full `EntityAnalysisResult`

**LOCATION_SEEDS** — include at minimum:
```
Australia, United States, United Kingdom, Canada, Germany, France, Japan, China, India, Brazil,
Sydney, Melbourne, London, New York, San Francisco, Tokyo, Berlin, Paris, Singapore, Toronto,
Google, Microsoft, Apple, Meta, Amazon, OpenAI, Anthropic, Tesla, Netflix, Spotify
```
(Include major tech companies here too as they're often treated as known entities)
