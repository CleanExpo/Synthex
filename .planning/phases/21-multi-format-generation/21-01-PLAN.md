  <name>Task 1: Create multi-format adapter service</name>
  <files>lib/ai/multi-format-adapter.ts</files>
  <action>
Create a `MultiFormatAdapter` class with a singleton export pattern (matching content-scorer.ts pattern).

**Core method:** `async adaptContent(params: AdaptContentParams): Promise<AdaptedContent>`

**AdaptContentParams interface:**
- `sourceContent: string` — original content
- `sourcePlatform?: string` — platform the content was originally written for (optional)
- `targetPlatforms: string[]` — array of platform names to generate for
- `tone?: string` — optional tone override (professional, casual, playful, etc.)
- `goal?: string` — engagement, reach, conversions, brand_awareness, traffic

**AdaptedContent interface:**
- `source: { content: string; platform?: string }`
- `variants: PlatformVariant[]`

**PlatformVariant interface:**
- `platform: string`
- `content: string` — the adapted text
- `format: 'standard' | 'thread' | 'carousel' | 'short-form' | 'long-form' | 'script'`
- `metadata: { characterCount: number; characterLimit: number; hashtags: string[]; mentions: string[]; wordCount: number; formatDetails?: Record<string, unknown> }`
- `score?: number` — optional quality score from ContentScorer

**Platform format rules (CRITICAL — these define how content is adapted):**

| Platform | Char Limit | Default Format | Adaptation Rules |
|----------|-----------|----------------|------------------|
| twitter | 280 | standard (or thread if >280) | Concise, punchy. If source >280 chars, split into numbered thread (1/N format). Add relevant hashtags (max 2). |
| linkedin | 3000 | long-form | Professional tone. Add line breaks for readability. Hook line first. End with question or CTA. Hashtags at end (max 5). |
| instagram | 2200 | standard | Visual-friendly. Emoji-rich. Hashtags in comment block format (separated by dots). Strong hook. |
| tiktok | 2200 | short-form | Casual, trendy. Short sentences. Trending hashtag style. Hook in first line. |
| facebook | 63206 | standard | Conversational. Questions to drive comments. Moderate length. |
| youtube | 5000 | long-form | Description format. Timestamps if long. Links and CTAs. Chapter markers if applicable. |
| pinterest | 500 | short-form | SEO-rich description. Keywords first. Action-oriented. |
| reddit | 40000 | long-form | Authentic, no marketing-speak. Provide value. Subreddit-appropriate tone. |
| threads | 500 | short-form | Casual, conversational. Similar to Twitter but slightly longer. |

**Implementation approach:**
1. For each target platform, build a platform-specific system prompt that instructs the AI to adapt the content
2. Use `getAIProvider()` from `lib/ai/providers` for AI calls (do NOT call OpenRouter directly like content-variations.ts does)
3. Make AI calls in parallel using `Promise.allSettled()` — one call per platform
4. After AI generates each variant, run it through `contentScorer.score(variant, platform)` to get quality score
5. Parse hashtags/mentions from generated content for metadata
6. If AI call fails for a platform, return a rule-based fallback (truncate/pad source content to fit platform limits)

**Thread splitting logic (for Twitter when content > 280 chars):**
- Split on sentence boundaries
- Each tweet max 270 chars (reserve 10 for "1/N" numbering)
- Return as single string with `\n---\n` delimiter between tweets
- Set `format: 'thread'` and `formatDetails: { tweetCount: N }`

**Do NOT:**
- Import from `src/services/` (legacy, may have different patterns)
- Add rate limiting (withAuth handles it)
- Make this a pure function — AI calls require async
- Block on a single platform failure — use Promise.allSettled

Export singleton: `export const multiFormatAdapter = new MultiFormatAdapter();`
Export types: `AdaptContentParams`, `AdaptedContent`, `PlatformVariant`

  <name>Task 2: Create multi-format API endpoint</name>

  <action>
Create `POST /api/content/multi-format` endpoint following the exact pattern of `app/api/content/score/route.ts`.

**Request body schema (Zod):**
```
MultiFormatRequestSchema = z.object({
  content: z.string().min(1).max(10000),
  sourcePlatform: z.string().optional(),
  targetPlatforms: z.array(z.enum([all 9 platforms])).min(1).max(9),
  tone: z.enum(['professional', 'casual', 'playful', 'authoritative', 'friendly']).optional(),
  goal: z.enum(['engagement', 'reach', 'conversions', 'brand_awareness', 'traffic']).default('engagement'),
})
```

**Handler:**
1. Parse and validate request body with Zod
2. Call `multiFormatAdapter.adaptContent()` with validated params
3. Return `{ success: true, source: {...}, variants: [...] }`

**Error handling:**
- 400 for validation errors (Zod flatten)
- 400 for invalid JSON body
- 500 for unexpected errors (log with console.error, return generic message)

**Pattern references:**
- Auth: `withAuth` wrapper from `lib/middleware/withAuth.ts` (same as score route)
- Export: `export const POST = withAuth(handlePost);`
- Runtime: `export const runtime = 'nodejs';` (not edge — needs AI provider)

**Do NOT:**
- Add a second rate limiter on top of withAuth
- Use edge runtime (Prisma and AI providers need Node.js)
- Return raw error objects to client (leak internal details)
