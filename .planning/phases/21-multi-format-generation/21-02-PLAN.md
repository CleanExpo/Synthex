  <action>
Create a 'use client' page at `/dashboard/content/multi-format` with the following layout:

**Page structure — Two-phase UI (input → results):**

**Phase 1: Input (shown initially)**

Use PageHeader with title "Multi-format Generator" and description "Transform your content into platform-optimized versions for every channel."

Input card (full width) with glassmorphic styling (bg-[#0f172a]/80, border-cyan-500/10):
1. **Source content textarea** — Full width, 6 rows minimum, placeholder "Paste or type your content here..."
2. **Platform multi-select** — Grid of 9 platform checkboxes (3 columns on desktop, 2 on tablet, 1 on mobile). Each checkbox shows platform name. All 9 platforms: Twitter/X, LinkedIn, Instagram, TikTok, Facebook, YouTube, Pinterest, Reddit, Threads. Default: none selected. "Select All" / "Clear" buttons above the grid.
3. **Options row** — Tone selector (dropdown: professional, casual, playful, authoritative, friendly) and Goal selector (dropdown: engagement, reach, conversions, brand_awareness, traffic). Default tone: professional, default goal: engagement.
4. **Generate button** — Full width on mobile, right-aligned on desktop. Text: "Generate Variants" with a Sparkles icon. Disabled when content is empty or no platforms selected. Shows "Generating..." with spinner when loading.

**Phase 2: Results (shown after generation completes)**

"Back to Editor" button at top to return to input phase.

Platform variant cards in a responsive grid (2 columns on desktop, 1 on mobile):
- Each card has:
  - Platform name as card title with platform icon
  - Format badge (e.g., "thread", "long-form", "short-form") as small colored pill
  - Content display (the generated variant text) in a pre-wrap div
  - Character count: "X / Y characters" with color coding (green if under limit, red if over)
  - Quality score badge if score exists (colored circle like Phase 20 ScoreCircle but smaller)
  - "Copy" button (copies variant text to clipboard, shows "Copied!" briefly)
  - Hashtags displayed as small pills below content (if any)

**State management:**
- `content: string` — source content
- `selectedPlatforms: string[]` — selected target platforms
- `tone: string` — selected tone
- `goal: string` — selected goal
- `variants: PlatformVariant[] | null` — results from API
- `isGenerating: boolean` — loading state
- `error: string | null` — error state
- `phase: 'input' | 'results'` — UI phase

**API call:**
- On generate click, POST to `/api/content/multi-format` with `{ content, targetPlatforms: selectedPlatforms, tone, goal }`
- Use `credentials: 'include'` for auth
- On success, set variants and switch to results phase
- On error, show toast or inline error message

**loading.tsx:** Skeleton matching the input phase layout — textarea skeleton, grid of 9 checkbox skeletons, two dropdown skeletons, button skeleton. Use animate-pulse + bg-white/5 blocks. Mirror the glassmorphic card styling.

**error.tsx:** Use DashboardError component:
```tsx
import { DashboardError } from '@/components/dashboard/error-fallback';
export default function MultiFormatError({ error, reset }) {
  return <DashboardError error={error} reset={reset} title="Multi-format Generator Error" description="Failed to load the multi-format generator." />;
}
```

**Icons:** Use Layers icon from components/icons for this feature. Use Sparkles for the generate button.

**Do NOT:**
- Create a custom hook for this (the API call is simple enough for inline fetch)
- Add real-time scoring (variants already come pre-scored from the API)
- Import from src/ (legacy directory)
- Use platform social icons from react-icons (just use text labels in checkboxes)
