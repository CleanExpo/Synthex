**Google Search Console API endpoints:**
- Search Analytics: `POST https://www.googleapis.com/webmasters/v3/sites/{siteUrl}/searchAnalytics/query`
- Sitemaps: `GET https://www.googleapis.com/webmasters/v3/sites/{siteUrl}/sitemaps`
- URL Inspection: `POST https://searchconsole.googleapis.com/v1/urlInspection/index:inspect`

Create Search Console service and API routes following the lib/google/indexing.ts pattern exactly.

**1. lib/google/search-console.ts:**

Reuse the Service Account auth pattern from lib/google/indexing.ts (createJWT, getAccessToken, loadCredentials). Extract shared auth into helper or duplicate — keeping it self-contained is fine since the indexing.ts file is small.

Three exported functions:

- `getSearchAnalytics(siteUrl, options)`: POST to Search Console API searchAnalytics.query endpoint. Options: startDate, endDate, dimensions (query/page/country/device), rowLimit (default 25). Returns rows with keys, clicks, impressions, ctr, position. Fall back to demo data if credentials not configured (log warning).

- `getIndexingStatus(siteUrl, inspectionUrl)`: POST to URL Inspection API. Returns indexCoverage status (SUBMITTED_AND_INDEXED, CRAWLED_NOT_INDEXED, etc.), lastCrawlTime, crawlState, robotsTxtState, indexingState. Fall back to informative error if credentials missing.

- `getSitemapStatus(siteUrl)`: GET from Sitemaps API. Returns list of sitemaps with path, lastSubmitted, isPending, isSitemapsIndex, lastDownloaded, warnings, errors. Fall back to empty array with warning if credentials missing.

Use the `webmasters.readonly` scope: `https://www.googleapis.com/auth/webmasters.readonly`
For URL Inspection, also need: `https://www.googleapis.com/auth/webmasters`

Important: The siteUrl parameter must be URL-encoded when used in API paths (e.g., `encodeURIComponent('https://synthex.social')`).

**2. app/api/seo/search-console/analytics/route.ts:**
- POST endpoint accepting { siteUrl: string, startDate?: string, endDate?: string, dimensions?: string[], rowLimit?: number }
- Defaults: last 28 days, dimensions=['query'], rowLimit=25
- Use APISecurityChecker with AUTHENTICATED_READ policy
- Zod validation for request body
- Return { success, analytics: { rows, totals } }

**3. app/api/seo/search-console/indexing-status/route.ts:**
- POST endpoint accepting { siteUrl: string, inspectionUrl: string }
- Use APISecurityChecker with AUTHENTICATED_READ policy
- Zod validation
- Return { success, inspection: { indexingState, crawlState, lastCrawlTime, ... } }

**4. app/api/seo/search-console/sitemaps/route.ts:**
- GET endpoint with siteUrl as query param
- Use APISecurityChecker with AUTHENTICATED_READ policy
- Return { success, sitemaps: [...] }

Follow app/api/seo/audit/route.ts error handling pattern exactly.
