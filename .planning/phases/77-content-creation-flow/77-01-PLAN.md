# Plan 77-01: Media Upload & Attachment Pipeline

## objective

Build a media upload pipeline that lets users attach images/videos to posts before publishing. Currently, content generation is text-only -- the `PostContent.mediaUrls` and `PostContent.mediaFiles` fields on the social service interface exist but are never populated from the UI. The publish cron also sends `mediaUrls: []` to `PlatformPost`. This plan wires up Supabase Storage for file hosting, adds an upload API route, and creates a reusable `MediaAttacher` component that slots into the content creation and schedule flows.

Linear: SYN-45
Phase dir: `.planning/phases/77-content-creation-flow/`

## context

### Discovery Findings

**Gap 1 -- No upload API**: There is no `/api/upload` or `/api/media/upload` route for direct file uploads. The media library API at `/api/media/library` accepts `base64Data` or `url` but not multipart form data. Users need a simple drag-and-drop upload that returns a public URL.

**Gap 2 -- Publish cron ignores media**: `publish-scheduled/route.ts` line 261 sends `{ text: post.content }` only. The `Post.metadata.images` array (if present) is never passed to `service.createPost()` as `mediaUrls`.

**Gap 3 -- No UI for media attachment**: The `GeneratedContent` component shows text only. No way to attach images, preview them, or remove them before scheduling.

### Architecture Decision

- **Storage**: Use Supabase Storage (already provisioned) with a `post-media` bucket
- **Upload route**: New `POST /api/media/upload` -- accepts `multipart/form-data`, validates file type/size, uploads to Supabase Storage, returns public URL
- **Size limits**: 10MB per file, max 4 files per post (platform limits: Twitter 4, Instagram 10, LinkedIn 9, others 1-4)
- **File types**: `image/jpeg`, `image/png`, `image/gif`, `image/webp`, `video/mp4` (max 100MB for video)
- **Component**: `MediaAttacher` -- drag-and-drop zone, preview grid, remove button, upload progress
- **Publish fix**: Wire `metadata.images` into `createPost({ text, mediaUrls })` in the cron

### Task 2: Upload API route -- `app/api/media/upload/route.ts`

Create a multipart file upload endpoint.

**File**: `app/api/media/upload/route.ts`

- Auth: `getUserIdFromRequestOrCookies(request)` -- same as other API routes
- Accept `multipart/form-data` with a `file` field
- Validate file type and size using `validateFile()`
- Upload to Supabase Storage using `uploadToStorage()`
- Return `{ url, path, size, mimeType }` with 201 status
- Max request body: set `export const config = { api: { bodyParser: false } }` -- for Next.js App Router use `request.formData()`

```ts
export async function POST(request: NextRequest) {
  const userId = await getUserIdFromRequestOrCookies(request);
  if (!userId) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });

  const formData = await request.formData();
  const file = formData.get('file') as File | null;
  if (!file) return NextResponse.json({ error: 'No file provided' }, { status: 400 });

  const validationError = validateFile({ size: file.size, type: file.type });
  if (validationError) return NextResponse.json({ error: validationError }, { status: 400 });

  const buffer = Buffer.from(await file.arrayBuffer());
  const result = await uploadToStorage(userId, file.name, buffer, file.type);

  return NextResponse.json({ data: result }, { status: 201 });
}
```

**Verification**: `curl -X POST /api/media/upload -F "file=@test.jpg"` returns `{ data: { url: "..." } }`.
