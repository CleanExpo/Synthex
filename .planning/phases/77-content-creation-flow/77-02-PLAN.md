# Plan 77-02: Platform Preview Cards & Content Composer Upgrade

## objective

Add platform-specific preview cards that show users exactly how their content will appear on each social platform before publishing. Currently, generated content is displayed as plain text in a `<pre>` block with no visual indication of how it will render on Twitter, Instagram, LinkedIn, etc. This plan adds a `PlatformPreview` component that simulates each platform's post UI, and upgrades the content creation flow with a unified composer that combines generation, editing, media, and preview into a single streamlined experience.

Linear: SYN-45
Phase dir: `.planning/phases/77-content-creation-flow/`

## context

### Discovery Findings

**Gap 1 -- No platform preview**: Users generate content and see raw text. They cannot visualise how a tweet will look with its character count, how an Instagram caption will render below an image, or how a LinkedIn post will appear in the feed. This is the single biggest UX gap in the content creation flow.

**Gap 2 -- Disjointed flow**: The current page has generation settings on the left and results on the right, but media attachment (from 77-01), preview, and publishing actions are separate concerns. A unified composer pattern would reduce friction.

**Gap 3 -- No character count warnings**: Platform character limits are documented in `platform_master_config.json` but not enforced or displayed in the UI. Users may generate content that exceeds Twitter's 280 characters or LinkedIn's 3000 characters without realising.

### Architecture Decision

- **Platform preview component**: `PlatformPreview` renders a mock of each platform's post UI (avatar, username, content, media grid, engagement icons)
- **Character limit display**: Real-time character counter with colour coding (green/amber/red)
- **Preview data**: Uses content text + attached media URLs + platform selection to render appropriate preview
- **No new API calls**: Preview is entirely client-side rendering -- no API dependency
- **Platform configs**: Hard-code essential limits in a `platform-limits.ts` config (subset of master config)

### Task 2: PlatformPreview component -- `components/content/platform-preview.tsx`

Create a component that renders a simulated platform post.

**File**: `components/content/platform-preview.tsx`

**Props**:
```ts
interface PlatformPreviewProps {
  platform: string;
  content: string;
  mediaUrls?: string[];
  hashtags?: string[];
  username?: string; // defaults to 'your_handle'
}
```

**Rendering**:
1. Platform header bar with platform icon + display name + brand colour
2. Mock post card:
   - Avatar circle (initials or placeholder)
   - Username + "just now" timestamp
   - Content text (truncated if over platform limit, with "[...]" indicator)
   - Media grid (if mediaUrls provided): 1 image = full width, 2 = side-by-side, 3+ = grid
   - Engagement icons row (heart, comment, share, bookmark -- all static/decorative)
3. Character count bar below the card:
   - `{current}/{max}` with colour: green (<80%), amber (80-100%), red (>100%)
   - If over limit, show "Over by X characters" warning text
4. Platform-specific tweaks:
   - Twitter: show thread indicator if content > 280 chars
   - Instagram: show image placeholder if no media (Instagram posts need an image)
   - LinkedIn: show "Read more" link after 200 chars preview
   - Pinterest: show vertical pin aspect ratio

**Design**: Glassmorphic card (`bg-zinc-900/50 border-zinc-800/50 rounded-xl`). Use icons from `@/components/icons`.

Export from `components/content/index.ts`.

**Verification**: Renders correctly for each platform, character count updates live.
