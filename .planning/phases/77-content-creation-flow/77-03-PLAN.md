# Plan 77-03: Draft-to-Publish Pipeline & Publish Confirmation

## objective

Close the gap between saving a draft and actually publishing content. Currently, the drafts page (`/dashboard/content/drafts`) shows drafts with copy/delete actions but has no way to edit, schedule, or publish a draft. Users must mentally track which drafts they want to publish, navigate to the content page, re-generate or manually re-enter the content, then schedule. This plan adds edit-in-place on drafts, a "Schedule" action that moves a draft into the scheduler, a publish confirmation modal with platform account selection, and status tracking from draft through to published.

Linear: SYN-45
Phase dir: `.planning/phases/77-content-creation-flow/`

## context

### Discovery Findings

**Gap 1 -- Drafts are dead ends**: The drafts page has no edit action. The `DraftCard` component shows copy and delete but no way to modify content, change platform, or schedule the draft. The `PATCH /api/content-drafts/[id]` endpoint exists and accepts content/platform/status updates, but the UI never calls it.

**Gap 2 -- No draft-to-schedule bridge**: When a user saves generated content as a draft, there is no path from the draft back to the scheduler. The `ContentDraft` model has a `scheduledPostId` field designed for this linkage, but it is never populated.

**Gap 3 -- No platform account verification before publish**: The publish cron (`publish-scheduled/route.ts`) checks for a connected platform account at publish time and fails the post if none exists. But there is no upfront check during scheduling to warn users they do not have a connected account for the selected platform.

**Gap 4 -- No publish confirmation**: Clicking "Schedule" on the content page immediately creates a scheduled post with a hardcoded "1 hour from now" time. There is no confirmation modal, no date/time picker, and no platform account selection.

### Architecture Decision

- **Draft edit**: Add inline edit capability to `DraftCard` -- expand the card to show a textarea + save button
- **Schedule from draft**: Add a "Schedule" button to `DraftCard` that opens a `PublishConfirmModal`
- **PublishConfirmModal**: Reusable modal used by both drafts page and content page. Shows: date/time picker, platform account selector (from connected accounts), preview summary, confirm button
- **Platform account check**: On modal open, fetch `/api/platforms/connections` to verify the user has a connected account for the target platform. Show warning if not connected, with link to `/dashboard/platforms`
- **Draft status sync**: When a draft is scheduled, update its `status` to 'scheduled' and store the `scheduledPostId` for cross-reference

### Task 1: PublishConfirmModal -- `components/content/publish-confirm-modal.tsx`

Create a confirmation modal for scheduling/publishing content.

**Props**:
```ts
interface PublishConfirmModalProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  content: string;
  platform: string;
  mediaUrls?: string[];
  hashtags?: string[];
  onConfirm: (options: PublishOptions) => Promise<void>;
}

interface PublishOptions {
  scheduledAt: string; // ISO datetime
  platform: string;
  connectionId?: string; // specific platform connection to use
}
```

**Features**:
1. Modal overlay (use existing Dialog from Radix UI / `@/components/ui/dialog`)
2. Content summary: truncated text preview (first 100 chars) + platform badge + media count
3. Date/time picker: use native `<input type="datetime-local" />` styled with Tailwind
   - Default: 1 hour from now (pre-filled)
   - Minimum: current time (prevent scheduling in the past)
4. Platform account selector: fetch connected accounts for the selected platform
   - If 1 account: auto-select, show account name
   - If 0 accounts: show warning "No connected {platform} account" + link to connect
   - If multiple: dropdown selector
5. "Schedule" confirm button with loading state
6. "Cancel" button

**Data fetching**: Use SWR to fetch platform connections:
```ts
const { data } = useSWR(
  open ? `/api/platforms?platform=${platform}` : null,
  (url) => fetch(url, { credentials: 'include' }).then(r => r.json())
);
```

First check what API exists for listing platform connections -- search for routes under `app/api/platforms/`. If no suitable endpoint exists, create a simple GET that returns connections for the current user filtered by platform.

Export from `components/content/index.ts`.

**Verification**: Modal opens, shows date picker and platform account, confirm creates scheduled post.

### Task 3: Add edit and schedule actions to DraftCard

**File**: `app/dashboard/content/drafts/page.tsx`

Upgrade the `DraftCard` component:

**3a. Edit mode**:
1. Add `isEditing` state to `DraftCard`
2. When edit icon clicked, expand card to show:
   - `<Textarea>` with current `draft.content`, pre-filled
   - Platform selector dropdown (pre-filled with `draft.platform`)
   - "Save" and "Cancel" buttons
3. On save, call `PATCH /api/content-drafts/${draft.id}` with updated content/platform via `fetchWithCSRF`
4. On success, update local state optimistically + toast

**3b. Schedule action**:
1. Add "Schedule" button to `DraftCard` actions (alongside Copy, Delete)
2. On click, open `PublishConfirmModal` with the draft's content and platform
3. On confirm, create a scheduled post via `/api/scheduler/posts` and then update the draft's status to 'scheduled' + set `scheduledPostId`:
   ```ts
   // 1. Create scheduled post
   const scheduleRes = await fetchWithCSRF('/api/scheduler/posts', { method: 'POST', ... });
   const { data: post } = await scheduleRes.json();

   // 2. Update draft status + link
   await fetchWithCSRF(`/api/content-drafts/${draft.id}`, {
     method: 'PATCH',
     body: JSON.stringify({ status: 'scheduled', scheduledPostId: post.id }),
   });
   ```

**3c. Update DraftCard action buttons layout**:
- Current: Copy | Delete
- New: Edit | Schedule | Copy | Delete
- Use icon-only buttons to save space, with tooltips

**Verification**: Click edit on a draft -- textarea appears. Save changes -- draft content updates. Click schedule -- modal opens. Confirm -- draft status changes to "scheduled".
