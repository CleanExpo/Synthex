# Plan 78-03: Bulk Scheduling & Auto-Fill Queue

## objective

Add the ability to schedule multiple posts at once across different time slots and platforms. Currently, creating 10 scheduled posts requires 10 individual schedule operations through the content page or calendar modal. This plan adds: (1) a bulk schedule upload/paste flow where users provide multiple pieces of content and the system auto-distributes them across ML-optimal time slots, (2) a CSV import for scheduling pre-written content in bulk, and (3) auto-fill capabilities that spread N posts across the next 7 days using optimal slots. This is the "fill my queue for the week" workflow.

## context

### Discovery Findings

**Gap 1 -- One post at a time**: Every scheduling surface (content page, calendar, drafts) creates posts individually. There is no batch creation flow.

**Gap 2 -- Auto-schedule API exists but is unwired**: The auto-schedule API has an `AutoSchedulePostsSchema` that accepts `postIds` and `platforms` and can distribute posts across optimal times. However, no UI calls this action -- it is only accessible via direct API call.

**Gap 3 -- No content import**: Users who have pre-written content (e.g., in a spreadsheet) have no way to import it as scheduled posts. They must manually enter each one.

**Gap 4 -- No "fill my week" workflow**: The most common scheduling pattern for social media managers is to batch-create a week's content in one sitting. There is no workflow that supports: "here are 15 posts, spread them optimally across the next 7 days."

### Architecture Decision

- **BulkScheduleWizard**: Multi-step modal flow: (1) add content items, (2) select platforms, (3) auto-distribute or manually assign times, (4) review and confirm
- **CSV import**: Parse CSV with columns `content,platform,scheduledAt(optional),hashtags(optional)`. Auto-schedule if `scheduledAt` is blank.
- **Auto-fill engine**: Client-side algorithm using `useOptimalTimes` predictions + `useScheduleConflicts` to distribute N posts across K days without conflicts
- **Integration**: Accessible from queue page ("Bulk Schedule" button) and schedule header

## tasks

### Task 1: Auto-fill scheduling algorithm -- `lib/scheduling/auto-fill.ts`

Pure function that distributes N content items across optimal time slots.

```ts
interface ContentItem {
  content: string;
  platform: string;
  hashtags?: string[];
  mediaUrls?: string[];
  preferredDate?: Date; // optional user preference
}

interface OptimalSlot {
  day: string;
  hour: number;
  score: number;
  platform: string;
}

interface ExistingPost {
  platform: string;
  scheduledAt: Date;
}

interface ScheduleSlot {
  contentItem: ContentItem;
  scheduledAt: Date;
  score: number; // ML confidence score for this slot
  conflict: boolean; // whether this slot had a near-conflict that was resolved
}

function autoFillSchedule(
  items: ContentItem[],
  optimalSlots: OptimalSlot[],
  existingPosts: ExistingPost[],
  options: {
    startDate: Date;
    endDate: Date;
    minIntervalHours: number; // minimum gap between posts on same platform
    timezone: string;
    maxPostsPerDay: number; // prevent overloading (default 5)
  }
): ScheduleSlot[]
```

**Algorithm**:
1. For each content item, get the optimal slots for its platform (sorted by score descending)
2. Map abstract day+hour slots to concrete dates within the start-end range
3. Greedily assign items to the highest-scoring available slot:
   - A slot is "available" if no existing post and no already-assigned item is within `minIntervalHours` on the same platform
   - Do not exceed `maxPostsPerDay` total across all platforms
4. If an item's platform has no free optimal slots, fall back to the next-best non-conflicting slot
5. Return the assignments sorted by `scheduledAt`

This is a library function, not a React component -- no hooks, no fetch calls. Takes pre-fetched data as arguments. This makes it testable.

**Commit:** `feat(78-03): add auto-fill scheduling algorithm (SYN-44)`

### Task 2: CSV parser for content import -- `lib/scheduling/csv-parser.ts`

Parse CSV content into `ContentItem[]` for the bulk schedule wizard.

```ts
interface ParsedCSVRow {
  content: string;
  platform: string;
  scheduledAt?: string; // ISO datetime, optional
  hashtags?: string[]; // comma-separated in CSV, parsed to array
}

interface CSVParseResult {
  items: ParsedCSVRow[];
  errors: Array<{ row: number; message: string }>;
  warnings: Array<{ row: number; message: string }>;
}

function parseScheduleCSV(csvText: string): CSVParseResult
```

**Rules**:
1. Required columns: `content`, `platform`
2. Optional columns: `scheduledAt`, `hashtags`
3. Validate platform values against supported list (twitter, instagram, linkedin, tiktok, facebook, youtube, pinterest, reddit, threads)
4. Validate scheduledAt is a valid ISO datetime if present
5. Skip empty rows
6. Report per-row errors (missing content, invalid platform) and warnings (scheduledAt in the past)
7. Max 100 rows per import

Provide a downloadable CSV template string for users.

### Task 3: BulkScheduleWizard component -- `components/scheduling/bulk-schedule-wizard.tsx`

Multi-step modal for batch scheduling.

**Steps**:

**Step 1 -- Add Content**:
- Option A: "Paste" -- multiple textareas with platform selector per item. "Add another" button.
- Option B: "Import CSV" -- file upload zone (drag-drop or click-to-browse). Show parsed preview table with validation errors highlighted in red.
- Option C: "Use drafts" -- show checkboxes for existing drafts (fetch from `/api/content-drafts?status=draft`). Select drafts to include.
- Bottom: item count summary "12 items ready"

**Step 2 -- Platform & Settings**:
- If items already have platforms set (from CSV or drafts), show summary
- If items need platforms assigned, show a default platform selector + per-item override
- Settings:
  - Date range: start date + end date (default: today + 7 days)
  - Min interval between posts: 2h / 4h / 6h / 8h dropdown (default 4h)
  - Max posts per day: 3 / 5 / 7 / 10 dropdown (default 5)

**Step 3 -- Schedule Preview**:
- Show the auto-fill result as a visual timeline or table
- Each item: content preview | platform badge | assigned date/time | ML score badge
- Conflict warnings highlighted
- Drag-to-reorder capability (swap time assignments between items)
- "Regenerate Schedule" button to re-run auto-fill with different settings
- "Auto-fill" button runs `autoFillSchedule()` with the current settings

**Step 4 -- Confirm**:
- Summary: "Schedule N posts across M platforms from [startDate] to [endDate]"
- "Schedule All" button with loading state
- Creates posts sequentially via `/api/scheduler/posts` (one per item)
- Shows progress: "Creating 3/12..."
- On completion: success toast + navigate to queue page

**Modal**: Use existing Dialog from Radix UI. Full-width (max-w-4xl) for the wizard.

**Commit:** `feat(78-03): add BulkScheduleWizard for batch content scheduling (SYN-44)`
