# Plan 78-02: Time-Slot Picker & Smart Scheduling

## objective

Build a reusable time-slot picker component that suggests ML-predicted optimal posting times and allows manual slot selection, then integrate it across all scheduling surfaces (content page, drafts page, calendar modal, queue reschedule). Currently, scheduling uses a basic `<input type="datetime-local" />` in the PublishConfirmModal (77-03) and a date + hour-dropdown in the calendar page. There is no visual representation of when slots are free vs occupied, no integration with the ML best-time predictions from `/api/optimize/auto-schedule`, and no conflict detection (two posts to the same platform at the same time). This plan upgrades the time selection experience to be smart, visual, and conflict-aware.

Linear: SYN-44
Phase dir: `.planning/phases/78-post-queue-scheduling/`

## context

### Discovery Findings

**Gap 1 -- No visual slot picker**: The PublishConfirmModal (77-03) uses a native HTML datetime-local input. It works but provides no context about: (a) when optimal times are, (b) whether other posts are already scheduled at that time, (c) which slots are free.

**Gap 2 -- ML predictions exist but are disconnected**: The `OptimalTimes` component on the schedule page fetches predictions from `/api/optimize/auto-schedule` and displays them as static badges. These predictions are not available in the scheduling flow when the user is actually choosing a time.

**Gap 3 -- No conflict detection**: Two posts can be scheduled to the same platform at the same minute. The publish cron will process them sequentially, but some platforms rate-limit rapid sequential posts (e.g., Twitter). The user gets no warning.

**Gap 4 -- No "auto-pick best time" shortcut**: Users must manually choose a date and time. There should be a one-click "Use recommended time" option that fills in the ML-predicted best next slot.

### Architecture Decision

- **TimeSlotPicker component**: `components/scheduling/time-slot-picker.tsx` -- visual 7-day x 24-hour grid showing free/busy/optimal slots
- **useOptimalTimes hook**: `hooks/use-optimal-times.ts` -- SWR wrapper around `/api/optimize/auto-schedule` for ML predictions
- **useScheduleConflicts hook**: `hooks/use-schedule-conflicts.ts` -- fetches existing posts for a date range and detects same-platform overlaps
- **Smart suggestion**: "Suggest Best Time" button that auto-fills the next available ML-optimal slot without conflicts
- **Integration**: Wire into PublishConfirmModal, calendar schedule modal, and queue reschedule popover

## tasks

### Task 1: useOptimalTimes hook -- `hooks/use-optimal-times.ts`

SWR hook wrapping the auto-schedule API for consumption by the time-slot picker.

```ts
interface UseOptimalTimesOptions {
  platforms: string[];
  timezone?: string;
  enabled?: boolean;
}

interface OptimalTimeSlot {
  day: string;  // 'Monday', 'Tuesday', etc.
  hour: number; // 0-23
  score: number; // 0-100
  confidence: number; // 0-1
  platform: string;
}

interface UseOptimalTimesResult {
  slots: OptimalTimeSlot[];
  isLoading: boolean;
  bestNextSlot: (platform: string, afterDate: Date) => Date | null;
}
```

**`bestNextSlot` logic**: Given a platform and a "not before" date, find the highest-scoring ML slot that falls after the given date. Map the day name + hour to a concrete `Date` in the next 7 days. If no ML predictions available, fall back to industry defaults from `schedule-config.ts`.

Use SWR with `revalidateOnFocus: false`, cache for 5 minutes.

**Commit:** `feat(78-02): add useOptimalTimes hook for ML time predictions (SYN-44)`

### Task 2: useScheduleConflicts hook -- `hooks/use-schedule-conflicts.ts`

Detects scheduling conflicts by fetching existing posts and comparing against a proposed time.

```ts
interface UseScheduleConflictsOptions {
  startDate: Date;
  endDate: Date;
  enabled?: boolean;
}

interface ScheduleConflict {
  existingPostId: string;
  platform: string;
  scheduledAt: Date;
  content: string; // first 60 chars
}

interface UseScheduleConflictsResult {
  existingPosts: Array<{ id: string; platform: string; scheduledAt: Date; content: string }>;
  checkConflict: (platform: string, proposedTime: Date, bufferMinutes?: number) => ScheduleConflict | null;
  isLoading: boolean;
}
```

**`checkConflict` logic**: Given a platform and proposed time, check if any existing post on the same platform falls within `bufferMinutes` (default 30 minutes) of the proposed time. Return the conflicting post or null.

Fetch from `/api/scheduler/posts?status=scheduled&startDate=X&endDate=Y` via SWR. Cover a 7-day window around the selected date.

**Commit:** `feat(78-02): add useScheduleConflicts hook for overlap detection (SYN-44)`

### Task 3: TimeSlotPicker component -- `components/scheduling/time-slot-picker.tsx`

Visual time selection with ML-optimal slot highlighting and conflict markers.

**Props**:
```ts
interface TimeSlotPickerProps {
  value: Date | null;
  onChange: (date: Date) => void;
  platform: string; // primary platform for conflict + optimal checks
  platforms?: string[]; // all platforms being scheduled (for multi-platform)
  minDate?: Date; // prevent scheduling in the past
  showOptimalSlots?: boolean; // default true
  showConflicts?: boolean; // default true
  compact?: boolean; // compact mode for popovers vs full-page
}
```

**Rendering (full mode)**:

1. **Date selector**: 7-day horizontal strip (today + 6 days). Each day shows abbreviated day name + date. Active day highlighted in cyan.

2. **Hour grid**: For the selected day, show 24 hourly slots in a vertical list or 4x6 grid.
   - Each slot shows the hour (e.g., "9:00 AM")
   - Background colour based on ML score:
     - Score >= 80: green tint (bg-green-500/10, border-green-500/30)
     - Score 50-79: amber tint (bg-amber-500/10)
     - Score < 50 or no data: neutral (bg-gray-800/50)
   - Conflict marker: if another post is scheduled within 30 min on the same platform, show a red dot + "Post scheduled at X:XX" tooltip
   - Selected slot: cyan highlight ring (ring-2 ring-cyan-500)

3. **"Suggest Best Time" button**: One-click auto-fill. Calls `bestNextSlot()` from `useOptimalTimes`, checks for conflicts via `checkConflict()`, and picks the first conflict-free optimal slot.

4. **Minute selector**: After selecting an hour, show a quick 0/15/30/45 minute picker (4 buttons).

**Compact mode**: For use inside popovers (queue reschedule). Shows only the date strip + hour grid without the surrounding card chrome.

**Styling**: Glassmorphic dark theme. Use existing `Card` component for the outer container.

Export from `components/scheduling/index.ts`.

**Commit:** `feat(78-02): add TimeSlotPicker with ML-optimal slots and conflict detection (SYN-44)`
