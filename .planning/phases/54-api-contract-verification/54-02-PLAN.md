  <action>
Create contract tests for two user-facing routes.

**Onboarding** (`app/api/onboarding/route.ts`):
- POST: Accepts the full onboarding wizard payload. Creates Organization, BusinessOwnership, Persona, marks onboarding complete, fires webhooks, returns new JWT.
- Uses `getUserIdFromRequestOrCookies` for auth.
- `onboardingSchema` requires: organizationName (min 1), industry (min 1), teamSize (min 1). Many fields optional.

Mock strategy:
```ts
jest.mock('@/lib/prisma', () => ({ default: { organization: { create: jest.fn() }, ... } }))
jest.mock('@/lib/auth/jwt-utils', () => ({
  getUserIdFromRequestOrCookies: jest.fn(),
  generateToken: jest.fn().mockReturnValue('new-jwt-token')
}))
jest.mock('@/lib/webhooks', () => ({ webhookHandler: { trigger: jest.fn() } }))
jest.mock('@/lib/logger', () => ({ logger: { info: jest.fn(), error: jest.fn() } }))
```

Test cases (onboarding):
1. Unauthenticated POST → 401
2. Missing required fields (no organizationName) → 400 with Zod validation error
3. Missing industry → 400
4. Valid minimal payload `{ organizationName, industry, teamSize }` → 200 with `{ success: true, token: string }`
5. Response includes `token` field (new JWT issued after onboarding complete)

**Referrals** (`app/api/referrals/route.ts`):
- GET: Returns `{ referralCode, stats: { totalReferrals, successfulReferrals, pendingReferrals }, referrals: [...] }`
- POST: Sends invite email, accepts `{ email }` (uses inline `z.object({ email: z.string().email() })`)
- Uses `APISecurityChecker` for auth.

Test cases (referrals):
6. GET unauthenticated → 401
7. GET authenticated → 200 with `{ referralCode, stats, referrals }` shape
8. POST unauthenticated → 401
9. POST invalid email → 400 validation error
10. POST valid email → 200 `{ success: true, message: string }`

Read both route files fully before writing to confirm exact response shapes and Prisma model names used.

Run: `npm test -- --testPathPattern="onboarding-referrals.contract" --passWithNoTests`
