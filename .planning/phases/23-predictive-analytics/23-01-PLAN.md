**API response shapes (from existing backend):**

POST /api/analytics/predict-engagement:
```typescript
{
  prediction: { id, likes, comments, shares, reach, engagementRate, confidence },
  factors: Array<{ factor, impact: 'positive'|'neutral'|'negative', weight }>,
  recommendations: string[],
  predictedAt: string
}
```

GET /api/analytics/predict-engagement:
```typescript
{
  predictions: Array<EngagementPredictionRecord>,
  total: number,
  hasMore: boolean,
  stats: { totalPredictions, withResults, avgAccuracy }
}
```

POST /api/predict/trends?action=forecast:
```typescript
{
  forecast: {
    platform, metric, currentValue,
    predictions: Array<{ date, predicted, lowerBound, upperBound }>,
    trend: 'rising'|'stable'|'declining'|'volatile',
    growthRate, confidence
  }
}
```

GET /api/optimize/auto-schedule?platform=X&timezone=Y:
```typescript
{
  optimalTimes: {
    platform, timezone,
    slots: Array<{ day: 0-6, hour: 0-23, score: 0-100, confidence: 0-1 }>,
    topSlot, nextOptimalTime, basedOnDataPoints,
    methodology: 'historical'|'industry'|'hybrid'
  }
}
```

1. Create `components/predictions/types.ts` with interfaces matching API responses:
   - `PredictionInput`: { text, platform, contentType, hasMedia, mediaType?, scheduledTime?, audienceSize? }
   - `PredictionResult`: { prediction: { id, likes, comments, shares, reach, engagementRate, confidence }, factors: Array<{ factor, impact, weight }>, recommendations: string[], predictedAt: string }
   - `PredictionHistoryItem`: { id, platform, contentType, predictedLikes, predictedComments, predictedShares, predictedReach, predictedEngRate, actualLikes?, actualComments?, actualShares?, actualReach?, accuracy?, confidenceLevel, predictedAt }
   - `PredictionStats`: { totalPredictions, withResults, avgAccuracy }
   - `OptimalTimeSlot`: { day: number, hour: number, score: number, confidence: number }
   - `OptimalTimesResult`: { platform, timezone, slots: OptimalTimeSlot[], topSlot: OptimalTimeSlot, nextOptimalTime: string, basedOnDataPoints: number, methodology: 'historical'|'industry'|'hybrid' }
   - `ForecastPoint`: { date: string, predicted: number, lowerBound: number, upperBound: number }
   - `EngagementForecast`: { platform, metric, currentValue, predictions: ForecastPoint[], trend, growthRate, confidence }

2. In `hooks/use-dashboard.ts`, add three hooks:

   a. `usePredictionHistory(params?: { platform?: string, withActuals?: boolean, limit?: number })`:
      - Build URL: `/api/analytics/predict-engagement?platform=X&withActuals=true&limit=20`
      - useApi with staleTime 60s, cacheTime 5min
      - Return typed `{ predictions: PredictionHistoryItem[], stats: PredictionStats, total: number }`

   b. `useOptimalTimes(params: { platform: string, timezone?: string })`:
      - Build URL: `/api/optimize/auto-schedule?platform=X&timezone=Y`
      - useApi with staleTime 300s (5min), cacheTime 30min (these change infrequently)
      - Return typed OptimalTimesResult

   c. `useEngagementForecast(params: { platform: string, metric?: string, days?: number })`:
      - Build URL: `/api/predict/trends?action=forecast` (POST with body)
      - Since useApi is GET-based, use `useApi` with a GET endpoint. Instead, build a custom fetch wrapper:
        - Actually, the GET endpoint for trends is `GET /api/predict/trends?action=trending&platform=X`
        - The forecast is POST-only. So create a simple `useForecast` that does a one-time fetch via useEffect + useState pattern, similar to old analytics pattern but simpler.
        - OR: Build the forecast URL as GET if the auto-schedule endpoint provides what we need. The auto-schedule GET returns optimal times, not forecasts.
        - Decision: Use a `fetchForecast` async function (not a hook) that POSTs to `/api/predict/trends?action=forecast`. The page will call this on demand when user changes platform. Export it from use-dashboard.ts.

   d. `fetchEngagementPrediction(input: PredictionInput)`: Async function that POSTs to `/api/analytics/predict-engagement`. Returns PredictionResult. Export from use-dashboard.ts.

Follow the pattern of `usePerformanceAnalytics` for GET-based hooks. For POST-based operations, export plain async functions.
