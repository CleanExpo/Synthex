<objective>
Make AI chat conversations survive page reload and browser close by moving
conversation selection from ephemeral React state into the URL.

Purpose: Currently `selectedConversationId` lives in `useState` on the page component
and is lost on every reload. Moving it to the URL (`/dashboard/ai-chat/[conversationId]`)
means browser back/forward works, reload preserves the active conversation, and users
can bookmark or share specific conversations.

Output:
- `app/dashboard/ai-chat/[conversationId]/page.tsx` — conversation detail page (new)
- `app/dashboard/ai-chat/page.tsx` — refactored to redirect to most recent conversation
  or show empty state if none exist
- Conversation sidebar uses `<Link>` for navigation (replaces `setSelectedConversationId`)
</objective>

orchestrator.

Keep the loading state (spinner) while conversations are fetching.
Keep the upgrade-required state.
Keep the error state.

New chat creation logic:
```tsx
const handleCreateConversation = async () => {
  setIsCreating(true);
  try {
    const newConversation = await createConversation();
    if (newConversation) {
      router.push(`/dashboard/ai-chat/${newConversation.id}`);
    }
  } finally {
    setIsCreating(false);
  }
};
```

On mount, once `isLoading === false` and `conversations.length > 0`, call:
`router.replace('/dashboard/ai-chat/' + conversations[0].id)`

This means visiting `/dashboard/ai-chat` always lands the user on their most recent chat.
