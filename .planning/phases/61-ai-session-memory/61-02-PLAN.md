  <name>Task 1: Auto-title API endpoint</name>
  <files>app/api/ai/chat/conversations/[conversationId]/auto-title/route.ts</files>
  <action>
Create POST `/api/ai/chat/conversations/[conversationId]/auto-title`.

This endpoint is called after the first assistant response in a new conversation.
It:
1. Authenticates the request (Supabase JWT via APISecurityChecker)
2. Verifies ownership: `prisma.aIConversation.findFirst({ where: { id, userId } })`
3. Checks the conversation still has a default title ("New Chat") — skip if already renamed
4. Fetches the first user message from the conversation:
   `prisma.aIMessage.findFirst({ where: { conversationId, role: 'user' }, orderBy: { createdAt: 'asc' } })`
5. Generates a title from the message content: trim to 60 chars, strip newlines
6. Updates the conversation title via `prisma.aIConversation.update`
7. Returns `{ success: true, title }`

Zod validation:
```ts
const schema = z.object({}); // no body needed — all data from DB
```

Use `DEFAULT_POLICIES.AUTHENTICATED_WRITE`. No subscription check needed (same user,
same conversation — this is an internal maintenance call, not a feature gate).

Add `export const runtime = 'nodejs'` at end.

- Delete button: existing delete

Add `archiveConversation` to `useAIChat`:
```ts
const archiveConversation = useCallback(async (id: string) => {
  await fetchWithCSRF(`/api/ai/chat/conversations/${id}`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ status: 'archived' }),
  });
  setConversations(prev => prev.filter(c => c.id !== id));
}, []);
```
