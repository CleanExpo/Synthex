## Summary

Wire BayesianOptimisation (BO) into the two primary GEO scoring surfaces by
replacing the fixed weight constants in `geo-analyzer.ts` and `tactic-scorer.ts`
with dynamic BO-optimised weights per organisation. This is wiring work — all
infrastructure was built in Phase 101. Zero new systems are introduced.

**Fallback guarantee**: if the BO service is unconfigured or unavailable, both
scorers return the existing fixed weights unchanged. No user-visible impact.

## Tasks

### Task 1 — Surface adapter: geo_score_weights

**File**: `lib/bayesian/surfaces/geo-weights.ts`

Create a typed surface adapter with:

- `GEO_SCORE_DEFAULTS` — the existing 5 fixed weights as the heuristic fallback
  ```
  citability: 0.25, structure: 0.20, multiModal: 0.15, authority: 0.20, technical: 0.20
  ```
- `GEO_SCORE_BOUNDS` — parameter bounds for BO (each dimension 0.05 to 0.50)
- `ensureGeoScoreSpace(orgId, client)` — creates the BO space on first use if it
  does not exist; uses `sumEquals: 1` constraint so weights always sum to 1.0
- `getGeoScoreWeights(orgId)` — calls `getOptimisedWeightsOrFallback` from
  `lib/bayesian/fallback.ts` and returns a typed `GeoScoreWeights` object.
  Returns the heuristic weights (source: 'heuristic') when BO is unavailable.

Exported types:
```typescript
export interface GeoScoreWeights {
  citability: number;
  structure: number;
  multiModal: number;
  authority: number;
  technical: number;
}
```

**Note**: The `sumEquals: 1` constraint is enforced at the BO service level.
No normalisation is needed in TypeScript unless `source === 'heuristic'`.

### Task 3 — Integrate geo-weights into geo-analyzer.ts

**File**: `lib/geo/geo-analyzer.ts` (lines 26-32)

Replace:
```typescript
// Score weights
const WEIGHTS = {
  citability: 0.25,
  structure: 0.20,
  multiModal: 0.15,
  authority: 0.20,
  technical: 0.20,
} as const;
```

With a dynamic lookup inside `analyzeGEO()`. The function signature gains an
optional `orgId?: string` parameter (non-breaking — existing callers pass no
org context, they receive heuristic weights).

Pattern:
```typescript
import { getGeoScoreWeights } from '@/lib/bayesian/surfaces/geo-weights';

export async function analyzeGEO(
  input: GEOAnalysisInput,
  orgId?: string,
): Promise<GEOAnalysisResult> {
  const weights = orgId
    ? (await getGeoScoreWeights(orgId)).weights
    : GEO_SCORE_DEFAULTS;
  // ...rest of function uses `weights` instead of `WEIGHTS`
}
```

All five weight references inside the function body (`WEIGHTS.citability`, etc.)
are updated to use the dynamic `weights` object.

**Breaking-change check**: `analyzeGEO` is called in:
- `app/api/geo/analyze/route.ts` — must pass `orgId` once fetched from DB
- Verify no other callers in `lib/` or `app/` before modifying

### Task 5 — Auto-observation in geo/analyze/route.ts

**File**: `app/api/geo/analyze/route.ts`

After the successful GEO analysis and DB write, add a fire-and-forget call:

```typescript
import { registerObservationSilently } from '@/lib/bayesian/fallback';

// After: return NextResponse.json({ id: analysis.id, ...result });
// Before the return, add:
const userOrg = await prisma.organizationMember.findFirst({
  where: { userId },
  select: { organizationId: true },
  orderBy: { createdAt: 'desc' },
});
if (userOrg) {
  void registerObservationSilently(
    'geo_score_weights',
    userOrg.organizationId,
    {
      citability: result.score.citability / 100,
      structure: result.score.structure / 100,
      multiModal: result.score.multiModal / 100,
      authority: result.score.authority / 100,
      technical: result.score.technical / 100,
    },
    result.score.overall,
    { contentLength: input.contentText.length, platform: input.platform },
  );
}
```

**Notes**:
- Use `void` — this must never block the response
- The observation target is the overall GEO score (0-100, already in the DB)
- Parameters are the 5 weight values used (normalised to 0-1 for BO)
- Verify the Prisma model name for org membership — it may be `OrganizationMember`
  or `organizationMember` — check schema before writing

### Task 6 — API route: spaces (GET/POST)

**File**: `app/api/bayesian/spaces/route.ts`

**GET** — List all BO spaces for the authenticated org.
- Auth: `getUserIdFromRequest` (jwt-utils)
- Fetch org: look up the user's organisation from DB
- Query: `prisma.bOSpace.findMany({ where: { orgId } })`
- Response: `{ data: spaces[] }`

**POST** — Create a new BO space for the org.
- Zod schema:
  ```typescript
  const createSpaceSchema = z.object({
    surface: z.enum([/* all BOSurface values */]),
    acquisitionFunction: z.enum(['ucb', 'ei', 'poi']).optional(),
  });
  ```
- Feature-limit check: use `isSurfaceAvailable(plan, surface)` and
  `isWithinBOLimit(plan, 'optimisationSpaces', currentCount)` — return 403 if exceeded
- If BO service available: call `client.createSpace(...)` then persist to `BOSpace`
- If BO service unavailable: create a `BOSpace` record with status `'pending'` for
  later activation — do not error
- Response: `{ data: space }`

**Pattern reference**: `app/api/campaigns/route.ts` for response format and auth.

### Task 9 — API route: run (POST)

**File**: `app/api/bayesian/run/route.ts`

**POST** — Trigger an async maximise optimisation run.
- Auth: JWT + org-scope
- Zod schema:
  ```typescript
  const runSchema = z.object({
    spaceId: z.string().min(1),
    initPoints: z.number().int().min(3).max(20).optional(),
    nIterations: z.number().int().min(5).max(100).optional(),
  });
  ```
- Verify space ownership
- Feature-limit check: `isWithinBOLimit(plan, 'monthlyOptimisations', monthlyRunCount)`
- Enqueue to BullMQ `BAYESIAN_OPTIMISATION` queue with job type `'bo:run-optimisation'`
  ```typescript
  import { getQueue, QUEUE_NAMES } from '@/lib/queue/bull-queue';
  const queue = getQueue(QUEUE_NAMES.BAYESIAN_OPTIMISATION);
  const job = await queue.add('bo:run-optimisation', {
    type: 'bo:run-optimisation',
    spaceId,
    orgId: userOrgId,
    initPoints,
    nIterations,
    callbackUrl: `${process.env.NEXT_PUBLIC_APP_URL}/api/internal/bo-callback`,
  });
  ```
- Create `BOOptimisationRun` record with `externalJobId: job.id`, status `'running'`
- Response: `{ data: { jobId: job.id, runId: run.id } }`

### Task 11 — API route: internal/bo-callback (POST)

- Auth: verify `X-Service-Key` header matches `BAYESIAN_SERVICE_API_KEY` env var

  ```typescript
  const callbackSchema = z.object({
    jobId: z.string(),
    spaceId: z.string(),
    status: z.enum(['completed', 'failed', 'cancelled']),
    bestParameters: z.record(z.string(), z.number()).nullable(),
    bestTarget: z.number().nullable(),

  ```
- Update `BOOptimisationRun` record: set `status`, `bestParameters`, `bestTarget`,
  `completedAt`
- If `status === 'completed'` and `bestParameters` non-null: update `BOSpace`
  with the new best parameters (store as `latestSuggestion` or equivalent field)
- Response: `{ ok: true }`
- Rate limiting: not needed — authenticated by service key
