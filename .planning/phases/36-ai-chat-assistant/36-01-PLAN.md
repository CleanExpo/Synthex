<task type="auto">
  <name>Task 3: Create Chat Streaming Messages API</name>
  <files>app/api/ai/chat/conversations/[conversationId]/messages/route.ts</files>
  <action>
Create streaming chat endpoint at /api/ai/chat/conversations/[conversationId]/messages:

**POST endpoint:**
- runtime='nodejs', dynamic='force-dynamic' (required for SSE on Vercel)
- Use APISecurityChecker with DEFAULT_POLICIES.AUTHENTICATED_WRITE
- Subscription gate: 'creator' OR 'business' OR 'custom'
- Validate conversationId ownership
- Parse body with Zod: { message: string.min(1).max(3000) } — shorter limit than PM

**Streaming flow:**
1. Store user message immediately (AIMessage with role='user')
2. Call generateChatResponse(userId, conversationId, message)
3. Create ReadableStream for SSE:
   - event: message_start — send userMessageId
   - data: { token } — stream each token
   - event: message_complete — send assistantMessageId, latencyMs
4. Store assistant message after stream completes (AIMessage with role='assistant')
5. Update conversation: messageCount += 2, lastMessageAt, auto-title if 'New Chat'

**SSE headers:**
- Content-Type: text/event-stream
- Cache-Control: no-cache, no-transform
- Connection: keep-alive
- X-Accel-Buffering: no

Reference: app/api/ai/pm/conversations/[conversationId]/messages/route.ts (exact pattern)
Don't: Call extractStructuredData — chat doesn't need action items extraction (simpler)
Don't: Store contextSnapshot — chat context is lightweight, rebuild each time
