<task type="auto">
  <name>Task 1: Create content scoring service</name>
  <files>lib/ai/content-scorer.ts</files>
  <action>
Create a ContentScorer class that analyzes content and returns detailed quality metrics WITHOUT modifying it.

Score dimensions (0-100 each):
- **readability**: Flesch-Kincaid approximation for social content (sentence length, word length, paragraph breaks, list usage)
- **engagement**: Hook presence, question usage, emoji usage, CTA strength, conversation starters
- **platformFit**: Character count vs platform limits, formatting patterns (LinkedIn line breaks, Twitter thread potential, Instagram hashtag density)
- **clarity**: Jargon detection, passive voice ratio, specificity (vague words like "things", "stuff", "nice")
- **emotional**: Sentiment strength, power words, urgency cues, storytelling elements

Each dimension returns: score (0-100), issues (string[]), suggestions (string[])

Overall score = weighted average (engagement 30%, readability 25%, platformFit 20%, clarity 15%, emotional 10%)

Reference `calculateOptimizationScore` in app/api/ai-content/optimize/route.ts for the existing basic scoring — this service should be MORE granular and comprehensive while reusing helper functions (hasEmojis, hasCTA, hasHashtags, startsWithHook).

Export a singleton `contentScorer` instance. Use pure functions internally — NO AI calls in the scorer (keep it fast for real-time use). AI-enhanced scoring is a future enhancement.

Do NOT duplicate the optimization logic from the optimize endpoint. This is scoring only.
  </action>

<task type="auto">
  <name>Task 2: Create content score API endpoint</name>
  <files>app/api/content/score/route.ts</files>
  <action>
Create POST /api/content/score endpoint using withAuth wrapper.

Request body (Zod validated):
- content: string (required, 1-10000 chars)
- platform: enum of supported platforms (twitter, linkedin, instagram, tiktok, facebook, youtube, pinterest, reddit, threads)
- goal: enum (engagement, reach, conversions, brand_awareness, traffic) — default "engagement"
- templateId: string (optional) — if provided, score against template structure expectations

Response shape:
```json
{
  "success": true,
  "score": {
    "overall": 72,
    "dimensions": {
      "readability": { "score": 85, "issues": [...], "suggestions": [...] },
      "engagement": { "score": 65, "issues": [...], "suggestions": [...] },
      "platformFit": { "score": 80, "issues": [...], "suggestions": [...] },
      "clarity": { "score": 70, "issues": [...], "suggestions": [...] },
      "emotional": { "score": 60, "issues": [...], "suggestions": [...] }
    },
    "topSuggestions": ["Add a hook...", "Include a question..."],
    "templateMatch": null | { "score": 85, "missingElements": [...] }
  }
}
```

When templateId is provided, fetch the PromptTemplate from database, compare content against template.structure (does it have hook, body, cta, hashtags per template), and include templateMatch in response.

Use the established withAuth pattern from app/api/templates/route.ts. Rate limit with readDefault from lib/middleware/api-rate-limit.ts.
