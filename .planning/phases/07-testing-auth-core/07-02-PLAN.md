Create tests for lib/prisma.ts covering connection management, health checks, retry logic, and transactions.

**Setup:**
- Mock the PrismaClient constructor and its methods
- Mock the @prisma/adapter-pg and pg Pool (if used in config)
- Use jest.useFakeTimers() for retry delay testing

**Test categories:**

1. **Singleton pattern:**
   - getPrismaClient() returns same instance on multiple calls
   - In development, handles hot-reload (global.__prisma caching pattern)

2. **Health check** (checkDatabaseHealth):
   - Returns { healthy: true, latency: N } when DB responds
   - Returns { healthy: false, error: message } when DB is down
   - Measures latency accurately (mock Date.now or performance.now)
   - Handles timeout gracefully

3. **Connection pool metrics** (getPoolMetrics):
   - Returns current connection counts
   - Tracks error count
   - Reports last health check timestamp

4. **Retry logic** (executeWithRetry):
   - Succeeds on first try → returns result, no retries
   - Fails then succeeds → retries and returns result
   - Exceeds max retries → throws final error
   - Uses exponential backoff (verify delays increase)
   - Only retries on connection errors (not business logic errors)

5. **Transaction wrapper** (withTransaction):
   - Successful transaction → returns result
   - Failed transaction → rolls back, throws
   - Timeout → aborts transaction
   - Nested operations execute within transaction context

6. **Disconnect/reconnect:**
   - disconnectPrisma() calls $disconnect
   - reconnectPrisma() creates new client after disconnect

**Test count target:** 15-20 test cases.

Don't: Make real database connections — mock everything.
Don't: Test Prisma query building — test our wrapper functions.
Reference: Read lib/prisma.ts to understand exact function signatures and error types.

<task type="auto">
  <name>Task 2: Create competitor-fetcher test suite</name>
  <files>tests/unit/lib/social/competitor-fetcher.test.ts</files>
  <action>
Create tests for lib/social/competitor-fetcher.ts covering all platform fetch paths.

**Setup:**
- Mock global fetch() with jest.fn()
- Import fetchCompetitorMetrics from lib/social/competitor-fetcher

**Test categories:**

1. **Twitter fetch:**
   - Valid response → returns followers, following, postsCount from public_metrics
   - 401 unauthorized → returns { success: false, error: message }
   - Network error → returns { success: false }, never throws
   - No access token → returns failure

2. **Instagram fetch (Business Discovery API):**
   - Valid response → returns followers_count, media_count
   - Invalid business account → returns failure
   - API rate limited → returns failure gracefully

3. **YouTube fetch:**
   - Valid response with channel stats → returns subscriberCount, videoCount
   - Supports API key fallback (no OAuth needed)
   - Channel not found → returns failure

4. **Facebook fetch:**
   - Valid page response → returns followers_count
   - Page not found → returns failure

5. **Reddit fetch (public endpoint):**
   - Valid response → returns link_karma, comment_karma
   - No auth needed — verifies fetch called without Authorization header
   - User-Agent header present (Synthex/1.0)
   - 404 user not found → returns failure

6. **Unsupported platforms:**
   - LinkedIn → { success: false, error: 'Platform does not support...' }
   - TikTok → same
   - Pinterest → same
   - Threads → same
   - Verify fetch() NOT called for unsupported platforms

7. **Result shape:**
   - All results have: platform, handle, followersCount, followingCount, postsCount, engagementRate, fetchedAt, success
   - Success results have fetchedAt as Date instance
   - Failure results have error string

**Mocking strategy:**
- Mock global.fetch = jest.fn() returning platform-specific JSON responses
- Different mock responses per platform URL pattern
- Use mockImplementation to vary behavior per test

**Test count target:** 18-22 test cases covering all platforms + edge cases.

Don't: Test full platform services — only the lightweight competitor-fetcher module.
Don't: Make real API calls.
Reference: Read lib/social/competitor-fetcher.ts for exact platform URL patterns and response parsing.
