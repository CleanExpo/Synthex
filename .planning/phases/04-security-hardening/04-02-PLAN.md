<task type="auto">
  <name>Task 1: Standardize canonical rate limiter with route helper</name>
  <files>lib/middleware/rate-limiter.ts, lib/middleware/api-rate-limit.ts</files>
  <action>
1. **Designate lib/middleware/rate-limiter.ts as canonical** — it uses Upstash REST API (works in serverless/Edge), has tier-based limits, and is already used by the most routes. Add a comment at the top: "Canonical rate limiter — use this for all new routes."

2. **Create lib/middleware/api-rate-limit.ts** — a thin helper that provides pre-configured rate limit wrappers by sensitivity category:

```
Categories:
- authStrict: 5 req/min (login, signup, password reset — brute force protection)
- authGeneral: 15 req/min (token refresh, verify — moderate protection)
- admin: 30 req/min (admin operations)
- billing: 20 req/min (checkout, subscription changes)
- aiGeneration: per-tier (free=5, pro=20, business=100 — already defined in rate-limiter.ts)
- mutation: 60 req/min (POST/PUT/DELETE on user data)
- readDefault: 120 req/min (GET endpoints)
```

Each category function should:
- Import and call withRateLimit from lib/middleware/rate-limiter.ts
- Pass the appropriate endpoint pattern or custom limits
- Return a function with signature: (req: NextRequest, handler: () => Promise<NextResponse>) => Promise<NextResponse>

3. **Do NOT delete or modify the other 3 rate limiter files** (rate-limiter-enhanced.ts, rate-limiter-redis.ts, rate-limiter-v2.ts) — they may be imported by existing routes. Just establish the canonical one. Consolidation/removal is a future cleanup task.

Don't: Create a new rate limiting algorithm — use the existing withRateLimit from rate-limiter.ts.
Don't: Add global middleware rate limiting — the matcher excludes /api/ and changing that has cascade effects.
Reference: lib/middleware/rate-limiter.ts for existing withRateLimit signature and tier definitions.

  <done>lib/middleware/api-rate-limit.ts exists with pre-configured category wrappers. lib/middleware/rate-limiter.ts marked as canonical. Type-check passes.</done>
