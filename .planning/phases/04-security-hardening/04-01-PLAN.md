<task type="auto">
  <name>Task 1: Wire EnvValidator to Next.js instrumentation hook</name>
  <files>instrumentation.ts</files>
  <action>
Create Next.js instrumentation.ts at project root (standard Next.js startup hook — runs once on server start).

In the `register()` export:
1. Import EnvValidator from lib/security/env-validator.ts
2. Call `EnvValidator.getInstance().validate()`
3. For CRITICAL vars missing (DATABASE_URL, JWT_SECRET, FIELD_ENCRYPTION_KEY, OAUTH_STATE_SECRET): log the error with the validator's error messages and throw to prevent startup
4. For SECRET/INTERNAL vars missing: log warnings but allow startup (graceful degradation)
5. Guard with `if (process.env.NEXT_RUNTIME === 'nodejs')` — don't validate in Edge Runtime (env vars may not all be available there)
6. Skip validation entirely when NODE_ENV === 'test' (test env uses .env.test with minimal vars)

Don't: Import anything that pulls in Prisma or heavy deps — instrumentation.ts runs before the app is ready, keep it lightweight.
Don't: Use console.error with color codes — Vercel logs don't support ANSI colors.
Reference: Next.js instrumentation docs — export async function register() is the standard pattern.
  </action>
  <verify>npm run build succeeds. Check that instrumentation.ts is picked up by Next.js (appears in build output).</verify>
  <done>instrumentation.ts exists at project root. App validates env vars at server startup. Missing CRITICAL vars cause startup failure with clear error messages. Missing optional vars produce warnings.</done>
</task>
