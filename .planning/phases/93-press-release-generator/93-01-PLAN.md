## Tasks

### Task 1 — Add PRDistribution to Prisma schema

**File**: `prisma/schema.prisma`

Add model after the PressRelease model:

```prisma
model PRDistribution {
  id          String    @id @default(cuid())
  releaseId   String
  channel     String    // 'self-hosted' | 'pr-com' | 'openpr' | 'prlog'
  channelUrl  String?   // URL where it was submitted/published
  status      String    @default("pending") // 'pending' | 'submitted' | 'published' | 'failed'
  submittedAt DateTime?
  publishedAt DateTime?
  notes       String?   @db.Text
  createdAt   DateTime  @default(now())

  release     PressRelease @relation(fields: [releaseId], references: [id], onDelete: Cascade)

  @@index([releaseId])
}
```

Also add `distributions PRDistribution[]` to the PressRelease model's relations block.

Run: `npx prisma db push`

### Task 2 — Create lib/pr/ai-generator.ts

AI-powered press release generation using the OpenRouter client pattern.

**Exports**:
- `PRGenerationInput` interface
- `PRGenerationResult` interface
- `generatePressRelease(input, apiKey?)` async function

**Behaviour**:
- Attempts OpenRouter API call with `anthropic/claude-3-haiku` model
- System prompt instructs the model to write PR copy optimised for AI search engine citation
- Returns structured JSON: `{ title, summary, body, suggestedSlug, jsonLd }`
- On failure: falls back to template from `buildPressRelease()` in press-release-builder.ts

---

### Task 3 — Create lib/pr/distribution-channels.ts

Static registry of free PR distribution channels.

**Exports**:
- `DistributionChannel` interface
- `DISTRIBUTION_CHANNELS` constant array
- `getChannel(id)` helper

**Channels**:
- `self-hosted` — Self-Hosted Newsroom (auto-published, no manual steps)
- `pr-com` — PR.com (manual, free tier)
- `openpr` — OpenPR (manual, free)
- `prlog` — PRLog (manual, free, Google-indexed)

Each channel has: id, name, url, submissionUrl, isFree, requiresManualSubmission, instructions.

### Task 4 — Create app/api/pr/press-releases/generate/route.ts

**Method**: POST
**Auth**: Required
**Body**: `{ angle, keyFacts, targetAudience, quoteName, quoteText, contactInfo?, brandName? }`
**Behaviour**:
- Validate with Zod
- Check pressReleases feature limit
- Call `generatePressRelease(input)`
- Return generated content (preview only — not saved to DB)

### Task 5 — Create app/api/pr/press-releases/[id]/distribute/route.ts

**Method**: POST
**Auth**: Required
**Body**: `{ channels: string[] }` — e.g. `['self-hosted', 'pr-com']`
**Behaviour**:
- Validate release belongs to authenticated user
- For each channel: upsert PRDistribution record with status `'submitted'`
- For `self-hosted`: set `release.status = 'published'`, set `publishedAt`
- Return `{ distributions, publicUrl }`
