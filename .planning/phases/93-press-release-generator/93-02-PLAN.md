# Plan 93-02: Distribution UI + Analytics

**Phase**: 93 — Press Release Generator & Distribution
**Status**: Pending
**Scope**: Frontend — AI generator form, distribution panel, analytics summary, enhanced PR dashboard tab

---

## Context

Plan 93-01 delivers the backend (Prisma model, AI generator service, distribution channel registry,
API routes). Plan 93-02 builds the UI layer on top of those routes. All components are React client
components using SWR for data fetching (following the SWR rule in CLAUDE.md).

---

## Tasks

### Task 1 — Create components/pr/PRGeneratorForm.tsx

**Purpose**: AI generation form — user fills in story details, clicks Generate, reviews output.

**Fields**:
- `angle` — textarea — e.g. "Series A funding round of $5M AUD"
- `keyFacts` — multi-line textarea (one fact per line)
- `targetAudience` — text input — e.g. "B2B SaaS founders"
- `quoteName` — text input — spokesperson name
- `quoteText` — textarea — the quote itself

**Behaviour**:
- Collapse/expand toggle ("Generate with AI" button reveals form)
- On submit: POST `/api/pr/press-releases/generate`
- Loading state: spinner + "Generating press release..."
- On success: show generated title, summary, and body in a preview panel
- "Save as Draft" button: POST `/api/pr/press-releases` with generated content
- On save: calls `onSaved(release)` callback prop, collapses form
- Error display: red alert banner

**Props**:
```typescript
interface PRGeneratorFormProps {
  onSaved?: (release: { id: string; slug: string; headline: string }) => void;
}
```

---

### Task 2 — Create components/pr/DistributionPanel.tsx

**Purpose**: Channel selection + distribution tracking for a selected press release.

**Data**:
- Channels: SWR fetch from `GET /api/pr/channels`
- Distributions: SWR fetch from `GET /api/pr/press-releases/{id}/distributions`

**UI**:
- Checkboxes for each channel with name and "Free" badge
- "Distribute" button: POST `/api/pr/press-releases/{id}/distribute` with selected channels
- After distribute: mutate SWR to refresh distribution list
- Distribution status list: table rows showing channel, status badge, submittedAt
- Status badges: `pending` (grey), `submitted` (yellow), `published` (green), `failed` (red)
- For `requiresManualSubmission` channels: show instruction text + external link button
- "Mark Published" button for manual channels (PATCH distribution record status)

**Props**:
```typescript
interface DistributionPanelProps {
  releaseId: string;
  orgSlug?: string; // for generating public newsroom URL
}
```

---

### Task 3 — Create components/pr/PRAnalyticsSummary.tsx

**Purpose**: High-level distribution overview stats at top of Press Releases tab.

**Data**: Derived client-side from the releases list + distributions (no new API needed).
Accepts releases as a prop and shows:
- Total press releases count
- Published count (status === 'published')
- Draft count (status === 'draft')
- Archived count (status === 'archived')

**Layout**: 4 stat cards in a row (matching the style of PROverviewStats in the Journalists tab).

**Props**:
```typescript
interface PRAnalyticsSummaryProps {
  releases: Array<{ id: string; status: string }>;
  isLoading?: boolean;
}
```

### Task 4 — Enhance app/dashboard/pr/page.tsx

**Current state**: Press Releases tab renders `<PressReleaseEditor />` in a single panel.

**Changes**:
1. Import `PRGeneratorForm`, `DistributionPanel`, `PRAnalyticsSummary`
2. Add state: `selectedRelease: { id: string; slug: string; headline: string } | null`
3. SWR fetch releases list inside the press-releases tab (or lift from PressReleaseEditor via callback)
4. In the Press Releases `TabsContent`:
   - Top: `<PRAnalyticsSummary />` (receives releases from SWR)
   - Below: `<PRGeneratorForm onSaved={r => setSelectedRelease(r)} />`
   - Below: existing `<PressReleaseEditor />` (pass `onSelectRelease` callback)
   - Below (conditional, when selectedRelease): `<DistributionPanel releaseId={selectedRelease.id} />`

The PressReleaseEditor already lists releases — wire its selection callback to `setSelectedRelease`
so that choosing a release from the list shows the DistributionPanel below it.

**Note**: Minimise changes to PressReleaseEditor.tsx — only add an `onSelectRelease` prop and call it
when the user clicks a release row. Do not refactor the internals.
