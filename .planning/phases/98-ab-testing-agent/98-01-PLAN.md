# Phase 98-01 Plan: Service Layer + Prisma + API Routes

## Phase
98 — Autonomous A/B Testing & Self-Healing Agent

## Intent Classification
Build — New service layer, Prisma models, and API routes for SEO experiment management, self-healing, and dog-fooding.

## Risk Level
MEDIUM — New Prisma models, new API routes. No destructive schema changes.

## Scope

### 1. Prisma Models
Add 3 models to `prisma/schema.prisma`:
- `SEOExperiment` — track SEO A/B experiments (title-tag, meta, H1, schema, etc.)
- `ExperimentObservation` — individual metric readings per experiment
- `HealingAction` — auto-detected healing opportunities and fix suggestions

Add User back-relations:
- `seoExperiments SEOExperiment[]`
- `healingActions HealingAction[]`

Run `npx prisma db push`.

### 2. Service Layer — lib/experiments/

**types.ts**
- ExperimentType union type
- MetricToTrack union type
- ExperimentResult type
- HealingIssue interface
- DogfoodReport interface
- ExperimentSuggestion interface

**experiment-designer.ts**
- `suggestExperiments(url, currentMetrics)` — rule-based suggestions
- If GEO < 70: suggest H1 entity name addition
- If title < 50 chars: suggest title expansion
- If metaDescription < 100 chars: suggest fuller description

**self-healer.ts**
- `analyzeForHealing(url, userId, orgId)` — detect and persist healing opportunities
- `generateFixSuggestion(issue)` — returns actionable fix string
- Checks: missing meta, schema issues, low GEO, low quality, missing entities

**dogfood-checker.ts**
- `runDogfoodCheck()` — run all analyzers against synthex.social
- Returns DogfoodReport with per-module scores
- Modules: GEO, Entity Coherence, Quality Gate, E-E-A-T, Authority

### 3. API Routes

| Method | Route | Purpose |
|--------|-------|---------|
| GET | /api/experiments/experiments | List user's experiments |
| POST | /api/experiments/experiments | Create experiment |
| POST | /api/experiments/experiments/[id]/start | Start experiment |
| POST | /api/experiments/experiments/[id]/record | Record observation |
| POST | /api/experiments/experiments/[id]/complete | Complete + pick winner |
| GET | /api/experiments/healing | List healing actions |
| POST | /api/experiments/healing/analyze | Analyze URL for healing |
| GET | /api/experiments/dogfood | Run dogfood check |

All routes use `APISecurityChecker.check()` with `DEFAULT_POLICIES.AUTHENTICATED_READ`.
