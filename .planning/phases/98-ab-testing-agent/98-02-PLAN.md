## Scope

### 1. Components — components/experiments/

**ExperimentCard.tsx**
- Experiment type badge (colour-coded by type)
- Status indicator (draft/running/paused/completed/cancelled)
- Hypothesis text display
- Original vs Variant side-by-side display
- Metric progress: baseline → variant score with arrow + %
- Winner badge: green (Variant Won) / amber (Original Won) / grey (Inconclusive)
- Action buttons: Start / Record / Complete / Cancel

**ExperimentWizard.tsx**
- Multi-step form (3 steps)
- Step 1: Select experiment type (6 option cards with icons)
- Step 2: Enter target URL and hypothesis
- Step 3: Enter original + variant values
- Submit → POST /api/experiments/experiments
- Suggestion panel powered by experiment-designer logic

**HealingPanel.tsx**
- URL input for analysis
- POST /api/experiments/healing/analyze → issues list
- Issue cards with severity badge (critical=red, warning=amber)
- Expandable "Suggested Fix" section per issue
- Copy Fix button for clipboard
- Healing action history list

**DogfoodScorecard.tsx**
- Overall dogfood score gauge
- Module cards: GEO Score, Entity Coherence, Quality Gate, E-E-A-T, Authority
- Each card: current score + benchmark + status pill
- "Run Full Check" button → GET /api/experiments/dogfood
- Last checked timestamp

### 2. Dashboard Page — app/dashboard/experiments/page.tsx

1. **Experiments** — ExperimentWizard + filter bar + ExperimentCard grid
2. **Self-Healing** — HealingPanel (URL input + history)

Header with page title and New Experiment button.
