<task type="auto">
  <name>Task 1: Create webhooks dashboard page</name>
  <files>app/dashboard/webhooks/page.tsx</files>
  <action>
Create the webhooks management page. Follow the same layout patterns as app/dashboard/integrations/page.tsx (glassmorphic cards, dark theme).

**Page structure:**
1. **PageHeader** — Title "Webhooks", subtitle "Send real-time event notifications to external systems"
2. **Create button** — Opens a creation dialog
3. **Webhook list** — Cards showing each subscription with:
   - URL (truncated to 50 chars)
   - Description (if set)
   - Event badges (cyan bg-cyan-500/20 text-cyan-300)
   - Active/inactive toggle (Radix Switch)
   - Last delivered timestamp (relative)
   - Failure count (if > 0, show in red)
   - Delete button (with confirmation)
4. **Empty state** — When no webhooks, show DashboardEmptyState with Webhook icon, message "No webhooks configured", CTA "Create Webhook"
5. **Create dialog** — Radix Dialog (glassmorphic styling) with:
   - URL input (required, validated as URL)
   - Description input (optional)
   - Event checkboxes (grouped by category: User, Content, Campaign, Team, Subscription, Integration, Analytics, System)
   - "Select All" / "Deselect All" buttons per category
   - Secret field (optional — auto-generated if empty, shown with copy button after creation)
   - Create button + Cancel
6. **Success toast** — After creation, show the generated secret with a "Copy" button and warning "Save this secret — it won't be shown again"

**Component organization:** Keep everything in the page file (consistent with other dashboard pages that don't extract components). Use Radix Dialog, Switch, Badge from components/ui/.

**Icons:** Use Webhook (lucide-react) as primary icon. If Webhook not available, use Link2 or Globe.

**Event categories for checkbox grouping:**
- User: user.created, user.updated, user.deleted
- Content: content.created, content.updated, content.published, content.scheduled, content.failed, content.deleted
- Campaign: campaign.created, campaign.started, campaign.completed, campaign.paused, campaign.deleted
- Team: team.member.added, team.member.removed
- Subscription: subscription.created, subscription.updated, subscription.cancelled, subscription.payment.succeeded, subscription.payment.failed
- Integration: integration.connected, integration.disconnected
- Analytics: analytics.report.ready, analytics.alert.triggered
- System: webhook.test, webhook.ping

**State management:** Use useWebhooks hook from hooks/use-webhooks.ts. All CRUD operations go through the hook.

AVOID: Don't add toast library — use a simple inline success message that auto-dismisses. Don't extract sub-components into separate files. Don't add loading skeletons beyond a simple Loader2 spinner.
