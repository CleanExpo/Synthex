```
model WebhookEndpoint {
  id              String   @id @default(cuid())
  url             String
  secret          String
  description     String?
  events          Json     // Array of WebhookEventType strings stored as JSON
  active          Boolean  @default(true)
  createdAt       DateTime @default(now()) @map("created_at")
  updatedAt       DateTime @updatedAt @map("updated_at")
  lastDeliveredAt DateTime? @map("last_delivered_at")
  failureCount    Int      @default(0) @map("failure_count")
  metadata        Json?    // Additional config (headers, timeout overrides)

  // Relations
  userId          String   @map("user_id")
  user            User     @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId])
  @@index([userId, active])
  @@map("webhook_endpoints")
}
```

IMPORTANT: The `events` field stores an array as JSON (not a Prisma enum) because the event types are defined in lib/webhooks/sender.ts and may evolve. This avoids migration churn.

<task type="auto">
  <name>Task 2: Rewrite webhook subscription API routes with Prisma persistence</name>
  <files>app/api/webhooks/user/route.ts</files>
  <action>
Rewrite the existing stub route to use real Prisma persistence. Keep the same URL structure (app/api/webhooks/user/) but replace all stub responses with actual database operations.

**Auth:** Replace the current `getUserFromRequest` (Bearer token) with `getUserIdFromCookies` from `@/lib/auth/jwt-utils` — consistent with all recent v1.2 API routes.

**GET /api/webhooks/user** — List user's webhook subscriptions:
- Auth via getUserIdFromCookies
- Query: `prisma.webhookEndpoint.findMany({ where: { userId }, orderBy: { createdAt: 'desc' } })`
- Return subscriptions with `availableEvents` list (import from lib/webhooks/sender.ts WebhookEventType)
- Strip `secret` field from response (only show last 4 chars as `secretPreview`)

**POST /api/webhooks/user** — Create webhook subscription:
- Auth via getUserIdFromCookies
- Validate body with existing webhookSubscriptionSchema (update events enum to match sender.ts WebhookEventType)
- Generate secret if not provided: `crypto.randomBytes(32).toString('hex')`
- Create via `prisma.webhookEndpoint.create()`
- Send test webhook using `sendTestWebhook` from lib/webhooks/sender.ts
- If test fails, still create but set `active: false` and return warning
- Return full subscription (including secret — only shown once at creation)

**PATCH /api/webhooks/user** (add new handler — for updates):
- Auth via getUserIdFromCookies
- Accept `id` in body along with optional `url`, `events`, `active`, `description`
- Update via `prisma.webhookEndpoint.update()`
- If url changed, send test webhook to new URL
- Return updated subscription (without secret)

**DELETE /api/webhooks/user** — Delete webhook subscription:
- Auth via getUserIdFromCookies
- Get `id` from searchParams
- Delete via `prisma.webhookEndpoint.delete({ where: { id, userId } })` — the userId check prevents deleting other users' webhooks
- Return success

**Zod schema update:** Expand the events enum to include all WebhookEventType values from sender.ts: user.created, user.updated, user.deleted, content.created, content.updated, content.published, content.scheduled, content.failed, content.deleted, campaign.created, campaign.started, campaign.completed, campaign.paused, campaign.deleted, team.member.added, team.member.removed, subscription.created, subscription.updated, subscription.cancelled, subscription.payment.succeeded, subscription.payment.failed, integration.connected, integration.disconnected, analytics.report.ready, analytics.alert.triggered, webhook.test, webhook.ping.

Keep the `deliverWebhook` and `triggerWebhooks` exports for backward compatibility but update `triggerWebhooks` to actually query subscriptions from Prisma and use `broadcastWebhook` from lib/webhooks/sender.ts.

AVOID: Don't import from lib/webhooks/types.ts (that's the inbound webhook types). Use lib/webhooks/sender.ts types for outbound.

<task type="auto">
  <name>Task 3: Create useWebhooks React hook</name>

  <action>
Create a new hook following the exact same fetch+useState pattern as hooks/use-third-party-integrations.ts and hooks/use-report-templates.ts.

**Hook interface:**
```typescript
export function useWebhooks(): {
  webhooks: WebhookEndpoint[];
  availableEvents: string[];
  loading: boolean;
  error: string | null;
  create: (data: CreateWebhookData) => Promise<WebhookEndpoint | null>;
  update: (id: string, data: UpdateWebhookData) => Promise<boolean>;
  remove: (id: string) => Promise<boolean>;
  refresh: () => Promise<void>;
}
```

**Types (define in same file):**
```typescript
export interface WebhookEndpoint {
  id: string;
  url: string;
  secretPreview: string; // last 4 chars
  description: string | null;
  events: string[];
  active: boolean;
  createdAt: string;
  updatedAt: string;
  lastDeliveredAt: string | null;
  failureCount: number;
}

export interface CreateWebhookData {
  url: string;
  events: string[];
  secret?: string;
  description?: string;
}

export interface UpdateWebhookData {
  url?: string;
  events?: string[];
  active?: boolean;
  description?: string;
}
```

**Implementation:**
- `useEffect` fetches GET /api/webhooks/user on mount (with mounted ref to prevent state updates after unmount)
- `create` POSTs to /api/webhooks/user, refreshes list on success
- `update` PATCHes /api/webhooks/user, refreshes list on success
- `remove` DELETEs /api/webhooks/user?id={id}, refreshes list on success
- All methods catch errors and set error state
- Include 'use client' directive at top

AVOID: Don't use SWR, TanStack Query, or any external state management. Raw fetch + useState only.
