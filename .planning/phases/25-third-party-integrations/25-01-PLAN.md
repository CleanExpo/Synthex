**Established patterns:** fetch + useState hooks (no SWR), webhook signature verification (HMAC-SHA256), integration CRUD via PlatformConnection model with metadata JSON
**Constraining decisions:**
- PlatformConnection model supports arbitrary platform strings + metadata JSON — use for Canva/Buffer/Zapier (no schema migration needed)
- All API routes use getUserIdFromCookies() or verifyToken() for auth
- Webhook verification uses lib/webhooks/signature-verifier.ts pattern
- fetch() directly for external APIs (no SDK bloat)

  <name>Task 2: Create third-party integration API routes</name>
  <files>app/api/integrations/third-party/route.ts, app/api/integrations/third-party/[provider]/route.ts, app/api/integrations/third-party/[provider]/config/route.ts</files>
  <action>
Create API routes for third-party integration management:

**app/api/integrations/third-party/route.ts:**
- GET: List all third-party integrations for current user
  - Auth via getUserIdFromCookies() from lib/auth/jwt-utils
  - Query PlatformConnection where platform IN ('canva', 'buffer', 'zapier') AND userId = current user
  - For each connected integration, include status from INTEGRATION_REGISTRY config
  - For disconnected providers, return config-only entries so UI can show "Connect" buttons
  - Return { integrations: IntegrationStatus[] }

**app/api/integrations/third-party/[provider]/route.ts:**
- Validate provider param against isValidProvider() type guard
- POST (connect): Accept credentials body (Zod-validated per provider), create PlatformConnection record with platform=provider, store credentials in accessToken + metadata JSON, return connection status
- GET (status): Fetch PlatformConnection for this user+provider, attempt validateCredentials() on the service, return { connected, provider, lastSync, tokenValid }
- DELETE (disconnect): Soft-delete by setting isActive=false and clearing accessToken/refreshToken, return success

**app/api/integrations/third-party/[provider]/config/route.ts:**
- GET: Return provider-specific configuration (from INTEGRATION_REGISTRY) + any user overrides from PlatformConnection.metadata
- PUT: Update provider-specific settings in PlatformConnection.metadata (Zod-validated), e.g., Zapier event subscriptions, Buffer default profile

Auth pattern: Use getUserIdFromCookies() + unauthorizedResponse() from lib/auth/jwt-utils (same as app/api/integrations/[integrationId]/status/route.ts).
Validation: Use Zod schemas for all request bodies.
Error handling: Use try/catch with structured { error, message } responses and appropriate HTTP status codes.
Do NOT use supabase client directly — use prisma from lib/prisma.

**app/api/webhooks/zapier/route.ts:**
- POST: Receive inbound webhooks from Zapier
  - Verify signature using HMAC-SHA256 with ZAPIER_WEBHOOK_SECRET env var
  - Parse Zapier payload: { event, data, zapier_hook_id, timestamp }
  - Map Zapier events to internal WebhookEventType from lib/webhooks/types.ts
  - Enqueue via webhookHandler.receive('internal', ...) or eventQueue.enqueue()
  - Return 200 { success: true } on valid, 401 on invalid signature, 400 on bad payload
- GET: Return 200 { status: 'ok' } for Zapier health checks
- Use Zod for payload validation
- Log all received webhooks via logger
