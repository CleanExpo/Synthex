  <files>

  </files>
  <action>
    **Update app/api/library/content/route.ts:**

    1. Import prisma from '@/lib/prisma'
    2. Add Zod schema for POST validation:
       ```
       const createContentSchema = z.object({
         title: z.string().min(1).max(200),
         content: z.string().min(1).max(50000),
         contentType: z.enum(['post', 'caption', 'story', 'thread', 'template', 'snippet']),
         platform: z.string().optional(),
         category: z.string().optional(),
         tags: z.array(z.string()).optional().default([]),
         metadata: z.record(z.unknown()).optional(),
       });
       ```

    3. GET handler - Replace empty state with real query:
       - Get userId from security.context.userId
       - Query ContentLibrary with filters: userId, status != 'deleted'
       - Support query params: contentType, platform, category, search (title/content), limit, offset
       - Return paginated results with proper count

    4. POST handler - Replace 501 with real creation:
       - Validate body with Zod schema
       - Create ContentLibrary record with userId from auth context
       - Return created item with 201 status

    **Update app/api/library/content/[contentId]/route.ts:**

    1. Import prisma from '@/lib/prisma'
    2. Add Zod schema for PATCH validation (all fields optional)

    3. GET handler - Replace 501:
       - Parse contentId from params
       - Query ContentLibrary by id AND userId (IDOR protection)
       - Return 404 if not found
       - Return item with 200

    4. PATCH handler - Replace 501:
       - Validate body with Zod schema
       - Update ContentLibrary by id AND userId
       - Increment usageCount if content was used
       - Return updated item

    5. DELETE handler - Replace 501:
       - Soft delete: Update status to 'deleted'
       - Return 204 no content

    **Security requirements:**
    - Always filter by userId from auth context
    - Never trust contentId alone (IDOR protection)
    - Validate all input with Zod
