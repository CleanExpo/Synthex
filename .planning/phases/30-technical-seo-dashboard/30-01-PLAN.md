Create technical SEO service and API routes:

**1. lib/seo/technical-seo-service.ts:**
```typescript
// Service with three main functions:
// - checkMobileParity(url): Fetch desktop vs mobile via PageSpeed API, compare content/structure
// - validateRobotsTxt(url): Fetch robots.txt, parse directives, check AI bot access (GPTBot, ClaudeBot, PerplexityBot)
// - getCwvHistory(userId): Query SEOAudit table for historical CWV data from rawData.categories.coreWebVitals
```

Key implementation details:
- Use PageSpeed API with strategy='mobile' and strategy='desktop' for parity check
- robots.txt parsing: check User-agent blocks, Disallow rules, Crawl-delay, AI bot access
- CWV history: aggregate from existing SEOAudit records (already stores CWV in rawData)

**2. app/api/seo/technical/mobile-parity/route.ts:**
- POST endpoint accepting { url: string }
- Use APISecurityChecker with AUTHENTICATED_READ policy
- Call technicalSeoService.checkMobileParity()
- Return { mobileParity: { contentMatch, structureMatch, issues[], recommendations[] } }

**3. app/api/seo/technical/robots-txt/route.ts:**
- POST endpoint accepting { url: string }
- Use APISecurityChecker with AUTHENTICATED_READ policy
- Call technicalSeoService.validateRobotsTxt()
- Return { robotsTxt: { valid, directives[], aiBotsBlocked[], issues[], rawContent } }

**4. app/api/seo/technical/cwv-history/route.ts:**
- GET endpoint (no body needed)
- Use APISecurityChecker with AUTHENTICATED_READ policy
- Query last 30 days of SEOAudit records with CWV data
- Return { history: [{ date, url, lcp, cls, inp, fid }] }

Use Zod for request validation. Follow existing /api/seo/audit/route.ts patterns exactly.
