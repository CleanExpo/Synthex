  <name>Task 2: Accessibility sweep — fix remaining WCAG 2.1 AA gaps</name>

  <action>
Fix the remaining WCAG 2.1 AA gaps identified in the Phase 83 aftermath:

1. **Fix images with empty alt** — Review each image and apply the correct fix:
   - `components/bio/BioPagePreview.tsx:105` — Cover image is content-bearing. Add
     meaningful alt like `alt={page.title ? \`Cover image for ${page.title}\` : "Bio page cover image"}`
   - `components/OptimizedImage.tsx:147, 365` — These are placeholder/fallback images.
     If decorative, add `aria-hidden="true"` and keep `alt=""`. If content-bearing, add
     contextual alt text. Read the component to determine which.
   - `components/marketing/SocialProof.tsx:291` — Avatar image. Add `alt` with the
     person's name if available, or `alt="Customer testimonial avatar"`.
   - `components/marketing/HeroSection.tsx:204` — Avatar image. Same pattern as SocialProof.

2. **Re-enable color-contrast in E2E tests** — In `tests/e2e/accessibility.spec.ts`, find
   where the `color-contrast` rule is disabled in axe-core config. Change the approach:
   instead of globally disabling it, keep it enabled but add a comment noting that failures
   should be investigated manually if they occur in CI (headless rendering may report false
   positives). If the rule is in a `disableRules` array, remove `'color-contrast'` from it.

3. **Add aria-invalid pattern to key form** — Pick ONE representative form component that
   handles validation errors (e.g., the login form or signup form in `app/auth/`). Add
   `aria-invalid={!!error}` to the input and `aria-describedby="field-error"` linking to
   the error message element (add `id="field-error"` to the error text). This establishes
   the pattern for future forms. Do NOT refactor every form — just establish the pattern
   in one place.

4. **Validate heading hierarchy on dashboard** — Check `app/dashboard/layout.tsx` and
   2-3 dashboard pages for proper heading nesting (single h1, h2 under h1, etc.). If
   heading hierarchy is already correct, document it. If there's a skip (e.g., h1 → h3),
   fix the most egregious instance.
  </action>

    - `tests/e2e/accessibility.spec.ts` does NOT disable color-contrast rule
    - At least one form component demonstrates aria-invalid + aria-describedby pattern
    - npm run type-check passes
  </verify>
  <done>
