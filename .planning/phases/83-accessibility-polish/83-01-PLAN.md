# Plan 83-01: Keyboard Accessibility — Clickable Rows, `title` → `aria-label`, Nav `aria-current`

## objective

Fix the three most impactful WCAG 2.1 AA keyboard/screen-reader violations found in the Phase 83 audit:

1. **Clickable table rows without keyboard support** — `<tr onClick=...>` in `audit-log-viewer.tsx` and `users-table.tsx` have no `tabIndex`, no `role`, and no `onKeyDown` handler. Keyboard-only users cannot activate them.

2. **`title` attribute used instead of `aria-label`** — Four interactive elements use `title="..."` for their accessible name. Screen readers do not reliably announce `title` on interactive elements. Affected files: `queue-table.tsx` (Retry, Delete buttons), `realtime/ConnectionStatus.tsx` (Reconnect button), `dashboard/ContentSuggestionsWidget.tsx` (Copy button), `admin/audit-log-drawer.tsx` (Copy button in MetadataItem).

3. **Sidebar nav missing `aria-current="page"`** — `SidebarGroup.tsx` calculates `isActive` but does not pass `aria-current="page"` to the active `<Link>`. Screen reader users have no programmatic way to know which nav item is current.

Linear: SYN-58
Phase dir: `.planning/phases/83-accessibility-polish/`

### Pattern: Keyboard-Accessible Clickable Rows

Clickable `<tr>` elements must have:
- `role="button"` or use a button/link inside the row for the primary action
- `tabIndex={0}` to be keyboard focusable
- `onKeyDown` to handle Enter/Space activation
- `aria-label` describing the row's action

**Preferred pattern** (row acts as button):
```tsx
<tr
  key={entry.id}
  role="button"
  tabIndex={0}
  aria-label={`View details for ${entry.action} by ${entry.user?.email ?? entry.userId.slice(0, 8)}`}
  className="border-b border-white/5 hover:bg-white/5 transition-colors cursor-pointer focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-cyan-500"
  onClick={() => setSelectedEntry(entry)}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      setSelectedEntry(entry);
    }
  }}
>
```

### Pattern: `aria-label` Instead of `title`

The `title` attribute is not reliably announced by screen readers on interactive elements. Replace with `aria-label`:

```tsx
// Before
<button title="Retry" onClick={...}>
  <RefreshCw className="h-3.5 w-3.5" />
</button>

// After
<button aria-label="Retry post" onClick={...}>
  <RefreshCw className="h-3.5 w-3.5" />
</button>
```

### Pattern: `aria-current="page"` on Active Nav Links

```tsx
// In SidebarGroup.tsx Link
<Link
  key={item.href}
  href={item.href}
  aria-current={isActive ? 'page' : undefined}
  className={`flex items-center gap-3 ...`}
>
```
