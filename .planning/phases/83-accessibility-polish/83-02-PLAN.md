# Plan 83-02: Form Accessibility ‚Äî `focus-visible`, Standalone Spinner `aria-label`, Input `placeholder` as Label Audit

## objective

Address form-level and feedback-element accessibility gaps across the codebase:

1. **`focus:outline-none` without `focus-visible` replacement** ‚Äî Several form inputs suppress the browser's default focus ring with `focus:outline-none` but do not add a `focus-visible:` variant replacement. This fails WCAG 2.1 AA criterion 2.4.7 (Focus Visible). The fix is to add `focus-visible:ring-2 focus-visible:ring-cyan-500 focus-visible:ring-offset-0` alongside the existing custom border change they use (`focus:border-cyan-500/50`).

2. **Standalone loading spinners without `aria-label`** ‚Äî `<Loader2 className="... animate-spin">` used as a standalone page/section loader (not inside a labelled button) has no accessible label. Screen readers announce it as an unlabelled graphic. Affected: `ai/chat-assistant.tsx`, `ai-pm/AIPMPanel.tsx`.

3. **`placeholder` used as the only accessible name on inputs** ‚Äî Several input elements in `affiliates/LinkForm.tsx` and `affiliates/NetworkForm.tsx` rely solely on `placeholder` text for their label association. While placeholder provides a hint, WCAG 1.3.5 (Label in Name) requires a programmatic label. The fix is to wrap these inputs with a `<label>` or add `aria-label`.

Linear: SYN-58
Phase dir: `.planning/phases/83-accessibility-polish/`

## context

### WCAG Criteria Addressed

| ID | Criterion | Level |
|----|-----------|-------|
| 2.4.7 | Focus Visible ‚Äî keyboard focus indicator must be visible | AA |
| 1.3.1 | Info and Relationships ‚Äî form controls must have programmatic labels | A |
| 1.1.1 | Non-text Content ‚Äî non-decorative UI graphics need alt text or aria-label | A |

### `focus:outline-none` Fix Strategy

The current pattern uses custom border colour on focus (`focus:border-cyan-500/50`) which provides a visual cue but is subtle. To satisfy WCAG 2.4.7 reliably, add `focus-visible:ring-2 focus-visible:ring-cyan-500/50 focus-visible:ring-offset-0` while keeping the existing `focus:outline-none` and `focus:border-*` in place. The `focus-visible:` prefix means the ring only appears for keyboard users (not mouse), which is the correct behaviour.

**Pattern to apply**:
```
Before:
  className="... focus:outline-none focus:border-cyan-500/50"

After:
  className="... focus:outline-none focus:border-cyan-500/50 focus-visible:ring-2 focus-visible:ring-cyan-500/50 focus-visible:ring-offset-0"
```

Where the existing class already has `focus:border-cyan-500` (full opacity), use `focus-visible:ring-cyan-500`:
```
className="... focus:outline-none focus:border-cyan-500 focus-visible:ring-2 focus-visible:ring-cyan-500 focus-visible:ring-offset-0"
```

### Standalone Spinner Fix

A standalone spinner (not inside a labelled button) needs `role="status"` and `aria-label`:

```tsx
// Before
<Loader2 className="h-6 w-6 animate-spin text-cyan-400" />

// After
<Loader2
  className="h-6 w-6 animate-spin text-cyan-400"
  role="status"
  aria-label="Loading"
/>
```

When the spinner is paired with visible text (like `<span>Loading conversation...</span>`), the spinner itself should be hidden from screen readers with `aria-hidden="true"` and the container should have `role="status"`:

```tsx
// When accompanied by visible text
<div className="flex flex-col items-center gap-3 text-gray-400" role="status">
  <Loader2 className="h-8 w-8 animate-spin text-cyan-400" aria-hidden="true" />
  <span>Loading conversation...</span>
</div>
```

### Input Label Strategy

For inputs currently relying only on `placeholder`:
- If the input has a visible sibling label element or parent context making its purpose clear, add `aria-label` matching the placeholder text
- If the input is inside a form with a `<Label>` from Radix UI that isn't properly associated via `htmlFor`/`id`, ensure `id` is set on the input and `htmlFor` matches on the label

For `affiliates/LinkForm.tsx` and `affiliates/NetworkForm.tsx`:
- Read each input in context
- If the input has no `id` and no `aria-label`, add `aria-label` matching the placeholder or visual label text
- Do not restructure the form layout ‚Äî only add missing attributes
