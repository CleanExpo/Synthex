# Plan 81-02: Social Account Onboarding UI

## objective

Build a guided social account onboarding experience with three components:

1. A **`SocialConnectBanner`** — a persistent top-of-dashboard banner shown to users who have zero social platforms connected. Dismissible per-session. Disappears once at least one platform is connected.
2. A **`/dashboard/platforms` page enhancement** — add an empty-state card and improve the connect flow so clicking "Connect" initiates the OAuth flow via `/api/auth/oauth/[platform]` and opens the returned `authorizationUrl` in the current tab, with a success redirect back to `/dashboard/platforms?connected=<platform>`.
3. A **`hooks/use-social-connections.ts`** SWR hook — shared hook consuming `GET /api/auth/connections` for connection status, used by both the banner and the platforms page.

This closes the gap between onboarding completion (Phase 70) and first social post — users will see a clear prompt to connect their first account as soon as they land on the dashboard.

Linear: SYN-56
Phase dir: `.planning/phases/81-social-account-onboarding/`

## context

### Discovery Findings

**Gap 1 — No empty-state onboarding prompt**: A brand-new user who completes onboarding sees the dashboard with blank charts and zero data. There is no prompt encouraging them to connect a platform. The `/dashboard/platforms` page exists but requires navigation discovery.

**Gap 2 — Platforms page connect flow is incomplete**: `app/dashboard/platforms/page.tsx` has a connect button that calls `integrationsAPI.connect(id)` (from `lib/api/settings.ts`). Reading that file: it calls `GET /api/auth/oauth/[platform]` to get the `authorizationUrl`, then does `window.location.href = authorizationUrl`. The callback route at `/api/auth/callback/[platform]` redirects to `/dashboard/settings?connected=<platform>` on success — not back to the platforms page. This means the connect flow takes the user to Settings after connecting, losing context.

**Gap 3 — No shared connection state hook**: The platforms page uses `useEffect` + `fetch` to load integrations. The settings integrations tab uses `useSettingsData()`. Neither is shareable. A single `useSocialConnections()` SWR hook avoids duplicate state.

### Architecture Decision

- **Hook**: `hooks/use-social-connections.ts` — SWR on `GET /api/auth/connections`, provides `connections` (full status), `summary` (connected/total), `connect(platform)` mutation, `disconnect(platform)` mutation, `isLoading`, `mutate`.
- **Banner**: `components/dashboard/SocialConnectBanner.tsx` — 'use client', uses `useSocialConnections`. Shows only when `summary.connected === 0`. Dismissible via localStorage key `socialBannerDismissed` (checked on mount). Renders a row of platform pills + "Connect your first account" CTA linking to `/dashboard/platforms`.
- **Platforms page**: Refactor `app/dashboard/platforms/page.tsx` to use `useSocialConnections` hook instead of bespoke fetch. Update callback redirect to come back to `/dashboard/platforms?connected=<platform>` by setting `?returnTo=platforms` in the OAuth state (or by updating the callback redirect logic). Show `toast.success('Connected to <platform>!')` when URL contains `?connected=<platform>`.
- **Dashboard layout**: Insert `<SocialConnectBanner />` in `app/dashboard/layout.tsx` after the top nav and before the `{children}` slot.

## tasks

### Task 1: SWR hook — `hooks/use-social-connections.ts`

**File**: `hooks/use-social-connections.ts`

```ts
'use client';

import useSWR from 'swr';

const fetchJson = (url: string) =>
  fetch(url, { credentials: 'include' }).then((r) => r.json());

interface ConnectionStatus {
  platform: string;
  connected: boolean;
  username?: string;
  avatar?: string;
  connectedAt?: string;
  expiresAt?: string;
  isExpired: boolean;
  needsRefresh: boolean;
}

interface ConnectionsSummary {
  total: number;
  connected: number;
  needsAttention: number;
}

interface UseConnectionsResult {
  connections: ConnectionStatus[];
  summary: ConnectionsSummary | null;
  isLoading: boolean;
  error: unknown;
  mutate: () => void;
  connect: (platform: string) => Promise<void>;
  disconnect: (platform: string) => Promise<void>;
}

export function useSocialConnections(): UseConnectionsResult {
  const { data, error, isLoading, mutate } = useSWR<{
    connections: ConnectionStatus[];
    summary: ConnectionsSummary;
  }>('/api/auth/connections', fetchJson, {
    revalidateOnFocus: true,
    dedupingInterval: 10_000,
  });

  // Initiate OAuth flow for a platform
  const connect = async (platform: string): Promise<void> => {
    const res = await fetch(`/api/auth/oauth/${platform}`, {
      credentials: 'include',
    });
    const json = await res.json();
    if (!res.ok) {
      throw new Error(json.error || json.message || `Failed to initiate ${platform} OAuth`);
    }
    if (json.authorizationUrl) {
      window.location.href = json.authorizationUrl;
    }
  };

  // Disconnect a platform
  const disconnect = async (platform: string): Promise<void> => {
    const res = await fetch(`/api/auth/connections?platform=${encodeURIComponent(platform)}`, {
      method: 'DELETE',
      credentials: 'include',
    });
    const json = await res.json();
    if (!res.ok) {
      throw new Error(json.error || `Failed to disconnect ${platform}`);
    }
    await mutate();
  };

  return {
    connections: data?.connections ?? [],
    summary: data?.summary ?? null,
    isLoading,
    error,
    mutate,
    connect,
    disconnect,
  };
}
```

**Note**: `connect()` uses plain `fetch()` in a mutation callback (not SWR) per project data fetching rules. The `window.location.href` redirect is intentional — OAuth requires a full browser navigation.

### Task 2: Social connect banner — `components/dashboard/SocialConnectBanner.tsx`

**File**: `components/dashboard/SocialConnectBanner.tsx`

```tsx
'use client';

import { useEffect, useState } from 'react';
import Link from 'next/link';
import { useSocialConnections } from '@/hooks/use-social-connections';
import { X, Zap } from '@/components/icons';

const DISMISS_KEY = 'socialBannerDismissed';

export function SocialConnectBanner() {
  const { summary, isLoading } = useSocialConnections();
  const [dismissed, setDismissed] = useState(true); // start dismissed to avoid flash

  useEffect(() => {
    // Check localStorage only after mount to avoid SSR mismatch
    const wasDismissed = localStorage.getItem(DISMISS_KEY) === 'true';
    setDismissed(wasDismissed);
  }, []);

  const handleDismiss = () => {
    localStorage.setItem(DISMISS_KEY, 'true');
    setDismissed(true);
  };

  // Show only when: loaded, not dismissed, zero connections
  if (isLoading || dismissed || (summary && summary.connected > 0)) {
    return null;
  }

  return (
    <div className="mx-4 mt-4 rounded-xl border border-cyan-500/30 bg-cyan-500/10 p-4 flex items-center justify-between gap-4">
      <div className="flex items-center gap-3 min-w-0">
        <div className="flex-shrink-0 p-2 rounded-lg bg-cyan-500/20">
          <Zap className="h-4 w-4 text-cyan-400" />
        </div>
        <div className="min-w-0">
          <p className="text-sm font-semibold text-white">Connect your first social account</p>
          <p className="text-xs text-slate-400 mt-0.5">
            Link a platform to start publishing, scheduling, and tracking performance.
          </p>
        </div>
      </div>
      <div className="flex items-center gap-2 flex-shrink-0">
        <Link
          href="/dashboard/platforms"
          className="inline-flex items-center px-3 py-1.5 rounded-lg bg-cyan-500 hover:bg-cyan-400 text-black text-xs font-semibold transition-colors"
        >
          Connect a platform
        </Link>
        <button
          onClick={handleDismiss}
          aria-label="Dismiss banner"
          className="p-1 rounded-md text-slate-500 hover:text-slate-300 hover:bg-white/5 transition-colors"
        >
          <X className="h-4 w-4" />
        </button>
      </div>
    </div>
  );
}
```

**Design rationale:**
- Cyan border/background matches the Synthex accent colour system — not red/amber (those are reserved for errors/warnings)
- Starts `dismissed=true` to avoid hydration flash then reads localStorage on mount
- Single CTA: deep link to the platforms hub, not a modal (keeps OAuth flow simple)
- Dismiss is session-persistent (localStorage) — re-appears after clearing browser data
