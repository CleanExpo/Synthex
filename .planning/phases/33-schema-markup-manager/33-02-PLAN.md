**1. Create hooks/useSchemaMarkup.ts** following usePageSpeed/useSearchConsole pattern:

Export useSchemaMarkup() hook with:
- State: validationResult, extractionResult, templates, richPreview, isValidating, isExtracting, isLoadingTemplates
- Functions:
  - validateSchema(schema: object) — POST to /api/seo/schema-markup/validate
  - extractFromUrl(url: string) — POST to /api/seo/schema-markup/extract
  - loadTemplates() — GET /api/seo/schema-markup/templates
  - generatePreview(schema: object) — Call generateRichPreview locally (import from service) or compute client-side from schema
- Use AbortController for cleanup in useEffect return
- Standard error handling: try/catch, set error state, toast on failure

**2. Rebuild app/dashboard/seo/schema/page.tsx** as tabbed Schema Markup Manager:

Replace existing single-panel schema generator with a tabbed interface using state-driven tabs (no Radix Tabs needed — use simple button tabs with active state like other dashboard pages).

**Header:** Change title from "Schema Generator" to "Schema Markup Manager" with Braces icon. Keep ArrowLeft back link to /dashboard/seo.

**Tab bar:** 5 tabs — Editor | Validator | Extractor | Templates | SERP Preview
Use glassmorphic tab buttons: active = bg-cyan-500/20 text-cyan-400 border-cyan-500/30, inactive = text-gray-400 hover:text-white

**Tab 1: Editor (default)** — Keep existing schema generator functionality:
- Left panel: TypeSelector + SchemaFields/FAQEditor + Generate button (existing behavior)
- Right panel: SchemaPreview with copy button (existing component)
- Add "Validate" button next to Generate that calls validateSchema on the generated output
- Show validation score badge if validation result exists

**Tab 2: Validator** — Paste or upload JSON-LD for validation:
- Textarea (monospace, bg-black/30) for pasting JSON-LD
- "Validate" button calls validateSchema
- Results panel: score gauge (circular, color-coded: green 80+, yellow 50-79, red <50), error list with severity icons (red X for errors, yellow triangle for warnings), field-level messages
- "Fix Issues" suggestions text

**Tab 3: Extractor** — Extract schema from any URL:
- URL input field + "Extract" button
- Results: list of found schemas with type badges, expandable JSON preview for each
- Each extracted schema has "Copy" and "Edit in Editor" buttons (Edit sets the schema in Editor tab and switches)
- Show isDemo badge if demo data returned

**Tab 4: Templates** — Browse and use predefined templates:
- Category filter buttons (All, Business, Content, Commerce, Media, Navigation)
- Grid of template cards: schema type icon, name, description, popularity stars, "Use Template" button
- "Use Template" loads template data into Editor tab and switches to it

**Tab 5: SERP Preview** — How schema renders in Google:
- Only active when a schema has been generated/validated
- Call generateRichPreview to get preview data
- Render SERP-like preview card based on previewType:
  - Product: blue title link, green URL, price, rating stars, availability
  - Article: title, date, author, description snippet, thumbnail
  - FAQ: expandable Q&A dropdowns below search result
  - Event: date, location, title in event card format
  - Generic: title + description
- Use bg-white text-black rounded-lg shadow for Google-like appearance (inside the dark glassmorphic container)

All styling follows glassmorphic pattern from existing SEO pages. Reuse existing SchemaPreview, TypeSelector, SchemaFields, FAQEditor components — don't rebuild them.
