<task type="auto">
  <name>Task 1: Create Schema Markup Service</name>
  <files>lib/seo/schema-markup-service.ts</files>
  <action>
Create lib/seo/schema-markup-service.ts with 4 exported functions and types:

**1. validateSchema(schema: object): SchemaValidationResult**
- Check JSON-LD structure: must have @context (https://schema.org) and @type
- Validate required fields per schema type (Organization needs name+url, Product needs name+offers, Article needs headline+author+datePublished, FAQ needs mainEntity array, etc.)
- Check field types (urls should be valid URLs, dates should be ISO format, numbers should be numeric)
- Return { isValid, errors: [{field, message, severity: 'error'|'warning'}], warnings, score (0-100) }
- Use a VALIDATION_RULES record keyed by @type with required fields and type checks
- Score = 100 - (errors * 15) - (warnings * 5), minimum 0

**2. extractSchemaFromUrl(url: string): SchemaExtractionResult**
- Fetch the URL with a 10-second timeout using fetch()
- Parse the HTML response as text
- Extract all <script type="application/ld+json"> tags using regex: /<script[^>]*type=["']application\/ld\+json["'][^>]*>([\s\S]*?)<\/script>/gi
- Parse each JSON block, validate it has @context and @type
- Return { url, schemas: ParsedSchema[], totalFound, extractedAt, isDemo }
- If fetch fails or URL is invalid, return demo data with isDemo: true showing example Organization + FAQ schemas (consistent with search-console/pagespeed fallback pattern)
- ParsedSchema = { type: string, data: object, isValid: boolean, validationResult: SchemaValidationResult }

**3. getSchemaTemplates(): SchemaTemplate[]**
- Return predefined templates for all 14 supported schema types
- Each template has: id, type, name, description, category ('business'|'content'|'commerce'|'media'|'navigation'), fields (pre-filled example data), popularity (1-5 stars)
- Group by category for UI display
- Templates should have realistic example data (not empty strings)
- Include 2-3 templates per category covering the most common use cases

**4. generateRichPreview(schema: object): RichPreviewResult**
- Based on @type, generate a SERP-like preview showing how Google would display the rich result
- Organization: knowledge panel preview (name, logo, description, social links)
- Product: product card (name, price, rating, availability, image)
- Article/BlogPosting: article preview (headline, date, author, thumbnail, breadcrumb)
- FAQ: expandable Q&A preview
- Event: event card (name, date, location, ticket price)
- VideoObject: video result (thumbnail, title, duration, upload date)
- Recipe: recipe card (name, rating, time, calories)
- Review: review snippet (rating stars, author, excerpt)
- Return { type, previewType: 'knowledge-panel'|'product-card'|'article'|'faq'|'event'|'video'|'recipe'|'review'|'generic', previewData: Record<string, string|number|boolean|string[]> }
- For unsupported types, return previewType: 'generic' with basic title/description

**Types to export:**
- SchemaValidationResult, ValidationError, SchemaExtractionResult, ParsedSchema, SchemaTemplate, RichPreviewResult

Do NOT import from or modify the existing /api/seo/schema/route.ts — this is a separate service.

  <action>
Create 3 API routes under app/api/seo/schema-markup/ following the established pattern (APISecurityChecker + Zod + subscription check):

**1. POST /api/seo/schema-markup/validate**
- Zod schema: { schema: z.record(z.unknown()) } — accepts any JSON-LD object
- Call validateSchema() from schema-markup-service.ts
- Return { success, validation: SchemaValidationResult }
- Security: AUTHENTICATED_WRITE policy, professional plan required

**2. POST /api/seo/schema-markup/extract**
- Zod schema: { url: z.string().url() }
- Call extractSchemaFromUrl() from schema-markup-service.ts
- Return { success, extraction: SchemaExtractionResult }
- Security: AUTHENTICATED_WRITE policy, professional plan required

**3. GET /api/seo/schema-markup/templates**
- No request body needed
- Call getSchemaTemplates() from schema-markup-service.ts
- Return { success, templates: SchemaTemplate[], categories: string[] }
- Security: PUBLIC_READ policy (templates are non-sensitive)

All routes follow exact pattern from existing SEO API routes:
- Import APISecurityChecker, DEFAULT_POLICIES from @/lib/security/api-security-checker
- Import subscriptionService from @/lib/stripe/subscription-service
- Check security.allowed, return 401/403 if not
- Check subscription.plan !== 'free' for paid features
- Validate with Zod, return 400 on validation failure
- Try/catch with console.error and 500 response on unexpected errors
