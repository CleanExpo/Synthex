  <name>Task 1: Create YouTubeService extending BasePlatformService</name>
  <files>lib/social/youtube-service.ts</files>
  <action>
Create lib/social/youtube-service.ts following established patterns.

Implement all 6 BasePlatformService methods using YouTube Data API v3:

1. **syncAnalytics(days?)** — GET channel statistics from `https://www.googleapis.com/youtube/v3/channels?part=statistics,snippet&mine=true`. Return subscriberCount, viewCount, videoCount as SyncAnalyticsResult. For daily breakdown, use YouTube Analytics API `https://youtubeanalytics.googleapis.com/v2/reports` with dimensions=day, metrics=views,likes,comments,shares,subscribersGained.

2. **syncPosts(limit?, cursor?)** — GET uploads playlist: First get channel's `contentDetails.relatedPlaylists.uploads` playlist ID, then GET `https://www.googleapis.com/youtube/v3/playlistItems?part=snippet,contentDetails&playlistId={id}&maxResults={limit}&pageToken={cursor}`. For each video, GET statistics with `https://www.googleapis.com/youtube/v3/videos?part=statistics,snippet&id={ids}`. Map to SyncPostsResult.

3. **syncProfile()** — GET channel info from channels endpoint with `part=snippet,statistics,brandingSettings`. Return SyncProfileResult with channel title, thumbnail, subscriber count, video count.

4. **createPost(content)** — Resumable upload already implemented in existing route. Refactor: POST to `https://www.googleapis.com/upload/youtube/v3/videos?uploadType=resumable&part=snippet,status` with video metadata (title, description, tags, categoryId, privacyStatus). Return PostResult with video ID. Note: Requires video file URL or buffer.

5. **deletePost(postId)** — DELETE `https://www.googleapis.com/youtube/v3/videos?id={postId}`. Return true on success.

6. **getPostMetrics(postId)** — GET `https://www.googleapis.com/youtube/v3/videos?part=statistics&id={postId}`. Return viewCount, likeCount, commentCount, favoriteCount.

**API calls:** Check what the existing route uses (googleapis package or direct fetch). Follow the same approach for consistency. If using googleapis, create the youtube client with `google.youtube({ version: 'v3', auth: oauth2Client })`. If using fetch(), use REST endpoints with `Authorization: Bearer {accessToken}`.

**Token management:** Same pattern as other services — use this.credentials, call handleTokenRefresh() before API calls.

Reference: app/api/social/youtube/post/route.ts for existing API call patterns. The OAuth provider at lib/oauth/providers/youtube.ts has getChannelInfo() — reuse that logic in syncProfile().

Don't: Add new npm packages — googleapis is already installed if needed.
