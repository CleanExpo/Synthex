<task type="auto">
  <name>Task 1: Create TikTokService extending BasePlatformService</name>
  <files>lib/social/tiktok-service.ts</files>
  <action>
Create lib/social/tiktok-service.ts following the pattern of twitter-sync-service.ts and instagram-service.ts.

Implement all 6 BasePlatformService methods using TikTok Content Posting API v2:

1. **syncAnalytics(days?)** — GET user info stats. TikTok's API for creator analytics is limited. Fetch basic user info (follower_count, following_count, likes_count, video_count) from `https://open.tiktokapis.com/v2/user/info/` with fields `open_id,union_id,avatar_url,display_name,follower_count,following_count,likes_count,video_count`. Return as SyncAnalyticsResult with available metrics, null for unavailable ones.

2. **syncPosts(limit?, cursor?)** — POST to `https://open.tiktokapis.com/v2/video/list/` with `max_count` and optional `cursor`. Extract video metadata: id, title, create_time, cover_image_url, share_url, duration, like_count, comment_count, share_count, view_count. Map to SyncPostsResult format.

3. **syncProfile()** — GET user info from same endpoint as syncAnalytics. Return SyncProfileResult with display_name, avatar_url, follower/following counts.

4. **createPost(content)** — Two-step process already in existing route:
   a. POST to `https://open.tiktokapis.com/v2/post/publish/video/init/` with post_info (title, privacy_level, disable_comment, disable_duet, disable_stitch) and source_info (video_url for PULL_FROM_URL, or chunk_size for FILE_UPLOAD)
   b. Return PostResult with the publish_id. Note: TikTok processes videos asynchronously — the video won't be immediately available.

5. **deletePost(postId)** — TikTok API does not support programmatic video deletion. Return false and log a warning. Document this limitation.

6. **getPostMetrics(postId)** — Fetch video info from `/v2/video/query/` with video IDs. Return available metrics (likes, comments, shares, views).

**Token management:** Use this.credentials for access token. Call this.handleTokenRefresh() before API calls if token is near expiry. Set Authorization header as `Bearer ${accessToken}`.

**Error handling:** Follow BasePlatformService patterns — throw PlatformError for API failures, handle 401 with token refresh attempt, respect rate limits from response headers.

Reference: lib/social/instagram-service.ts for the fetch() + token refresh pattern. The existing app/api/social/tiktok/post/route.ts has working API call patterns — extract the logic into service methods.

Don't: Install any TikTok SDK — use fetch() directly like Instagram/LinkedIn.
Don't: Implement features not supported by TikTok's API (e.g., delete, detailed analytics).
