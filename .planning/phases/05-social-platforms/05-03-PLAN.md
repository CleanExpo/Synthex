<task type="auto">
  <name>Task 1: Create PinterestService extending BasePlatformService</name>
  <files>lib/social/pinterest-service.ts</files>
  <action>
Create lib/social/pinterest-service.ts using Pinterest API v5 (https://api.pinterest.com/v5/).

Implement all 6 BasePlatformService methods:

1. **syncAnalytics(days?)** — GET `https://api.pinterest.com/v5/user_account/analytics` with start_date, end_date, metric_types=IMPRESSION,PIN_CLICK,OUTBOUND_CLICK,SAVE. Requires business account. Return as SyncAnalyticsResult. If 403 (personal account), return null metrics with warning.

2. **syncPosts(limit?, cursor?)** — GET `https://api.pinterest.com/v5/pins` with page_size={limit} and optional bookmark={cursor}. Each pin has: id, title, description, link, media (images), created_at. Map to SyncPostsResult format with pagination bookmark.

3. **syncProfile()** — GET `https://api.pinterest.com/v5/user_account`. Returns username, profile_image, account_type, follower_count, following_count, pin_count, board_count. Map to SyncProfileResult.

4. **createPost(content)** — POST `https://api.pinterest.com/v5/pins` with:
   - board_id (REQUIRED — Pinterest pins must belong to a board)
   - title, description, link, alt_text
   - media_source: { source_type: "image_url", url: content.mediaUrls[0] } for image pins
   - For video: source_type: "video_id" (requires pre-upload)
   Note: board_id should come from content.metadata.boardId or a default board.

5. **deletePost(postId)** — DELETE `https://api.pinterest.com/v5/pins/{postId}`. Return true on 204.

6. **getPostMetrics(postId)** — GET `https://api.pinterest.com/v5/pins/{postId}/analytics` with start_date, end_date, metric_types=IMPRESSION,SAVE,PIN_CLICK,OUTBOUND_CLICK. Return available metrics.

**Board management:** Add a helper method `getBoards()` that calls GET `https://api.pinterest.com/v5/boards` — reuse logic from the OAuth provider's getBoards() method. This is needed for createPost to get valid board IDs.

**Headers:** All requests need `Authorization: Bearer {accessToken}` and `Content-Type: application/json`.

Reference: lib/social/instagram-service.ts for fetch pattern and error handling.
Don't: Install any Pinterest SDK.
