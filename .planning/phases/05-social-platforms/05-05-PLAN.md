<task type="auto">
  <name>Task 1: Create ThreadsService extending BasePlatformService</name>
  <files>lib/social/threads-service.ts</files>
  <action>
Create lib/social/threads-service.ts using Threads API (Meta Graph API variant).

Threads API is very similar to Instagram Graph API. Reference instagram-service.ts heavily.

Implement all 6 BasePlatformService methods:

1. **syncAnalytics(days?)** — GET `https://graph.threads.net/v1.0/me/threads_insights` with metric=views,likes,replies,reposts,quotes&period=day&since={start}&until={end}. Return as SyncAnalyticsResult. Also get follower_count from profile endpoint.

2. **syncPosts(limit?, cursor?)** — GET `https://graph.threads.net/v1.0/me/threads?fields=id,text,timestamp,media_url,media_type,shortcode,permalink,is_reply,reply_audience&limit={limit}&after={cursor}`. Map to SyncPostsResult with pagination.

3. **syncProfile()** — GET `https://graph.threads.net/v1.0/me?fields=id,username,name,threads_profile_picture_url,threads_biography`. Return SyncProfileResult.

4. **createPost(content)** — Two-step process (same as Instagram container pattern):
   a. POST `https://graph.threads.net/v1.0/me/threads` with:
      - media_type: TEXT (text only), IMAGE (image + text), VIDEO (video + text)
      - text: content.text
      - image_url: content.mediaUrls[0] (for IMAGE type)
      - video_url: content.mediaUrls[0] (for VIDEO type)
      - reply_control: 'everyone' | 'accounts_you_follow' | 'mentioned_only' (from metadata)
   b. POST `https://graph.threads.net/v1.0/me/threads_publish` with creation_id from step a
   Return PostResult with thread ID.

5. **deletePost(postId)** — DELETE `https://graph.threads.net/v1.0/{postId}`. Return true on success. (Note: May not be available — check API support, return false with warning if not supported.)

6. **getPostMetrics(postId)** — GET `https://graph.threads.net/v1.0/{postId}/insights?metric=views,likes,replies,reposts,quotes`. Return available metrics.

**Token management:** Threads uses long-lived tokens (60 days, same as Instagram). Use the OAuth provider's getLongLivedToken() pattern. Authorization: `Bearer {accessToken}`.

Reference: lib/social/instagram-service.ts — Threads follows almost identical patterns since both are Meta platforms.
Don't: Install any SDK — use fetch() like Instagram.
