  <files>
    app/layout.tsx
    next.config.mjs
    app/dashboard/layout.tsx
  </files>
  <action>
    Audit the code configuration for Core Web Vitals risk factors:

    LCP (Largest Contentful Paint — target < 2.5s):
    - Is the root layout using next/font for font loading? (eliminates render-blocking)
    - Are hero images using next/image with priority prop?
    - Is the dashboard layout doing any heavy synchronous work on mount?

    CLS (Cumulative Layout Shift — target < 0.1):
    - Are images sized with explicit width/height or aspect-ratio?
    - Are any elements causing layout shifts (async-loaded banners, etc.)?
    - Does the sidebar handle collapse animation without causing reflow?

    INP (Interaction to Next Paint — target < 200ms):
    - Are event handlers debounced where appropriate?
    - Is any synchronous heavy work blocking the main thread?

    Document findings. Fix any obvious code-level issues (e.g., missing font display swap, missing image dimensions). Do not fix issues requiring production profiling.
  </action>
  <verify>npx tsc --noEmit 2>&1 | tail -5</verify>
  <done>CWV risk factors audited. Any fixable code issues resolved. TypeScript clean.</done>
