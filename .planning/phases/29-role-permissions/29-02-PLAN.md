Create a roles management dashboard page with role CRUD, permission assignment, and user management. Add sidebar and command palette entries.

## Tasks

### Task 1: Create Roles Dashboard Page

**File:** `app/dashboard/roles/page.tsx`

**Page structure:**
- PageHeader with title "Role Management"
- "Create Role" button in header
- Role cards grid showing all roles

**Role Card displays:**
- Role name with icon (Crown for admin, Edit for editor, Eye for viewer, Shield for custom)
- Description
- Permission count badge
- User count badge
- System role indicator (cannot be deleted)
- Default role indicator
- Actions: Edit, Delete (disabled for system roles)

**Create/Edit Role Dialog:**
- Name input
- Description textarea
- isDefault checkbox
- Permissions multi-select with grouped categories:
  - Posts: create, read, update, delete, approve
  - Campaigns: create, read, update, delete, manage
  - Analytics: read, export
  - Personas: create, read, update, delete
  - Settings: read, update
  - Users: read, update, delete, invite
  - Roles: create, read, update, delete, manage
  - Integrations: create, read, update, delete
  - Billing: read, update, manage
  - Organization: read, update, delete, manage
- Save/Cancel buttons

**Role Users Dialog (click user count):**
- List of users with this role
- Each shows: avatar, name, email, granted date, expires date
- Revoke button per user
- "Add User" button opens user search

**Empty state:**
- "No custom roles yet. Use the predefined roles or create your own."

**Styling:** Glassmorphic cards, consistent with other dashboard pages

### Task 2: Add Sidebar and Command Palette Entries
