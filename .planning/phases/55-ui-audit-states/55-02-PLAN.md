    app/dashboard/competitors/page.tsx
    app/dashboard/reports/page.tsx
  </files>
  <action>
    Read each page. For each, check:
    1. **Loading state**: When `isLoading` is true, does the page show a skeleton or spinner? (Acceptable: DashboardSkeleton, skeleton HTML with animate-pulse, Loader2 spinner in a centered full-height div). NOT acceptable: returning null or rendering an empty container.
    2. **Empty state**: When data array is empty, does the page show a meaningful empty state? (Acceptable: DashboardEmptyState component, or local EmptyState with icon+title+description+CTA. NOT acceptable: rendering nothing or an empty list container).
    3. **Error state**: If the page catches errors, does it show error UI? (Acceptable: DashboardError component, toast notification, or inline error card). NOT acceptable: silently swallowing errors with console.error only.

    Fix any deficiencies found:
    - Replace `return null` during loading with `return <DashboardSkeleton />` (import from @/components/skeletons)
    - If an empty div is rendered when data is empty, replace with `<DashboardEmptyState icon={...} title="No [items] yet" description="[Helpful description]" action={{ label: "Create [item]", onClick: ... }} />`
    - If errors are swallowed silently (only console.error), add an error display: set an error state variable and render an inline error card using DashboardError (import from @/components/dashboard)

    IMPORTANT: content/drafts and content/library use Loader2 spinners — these are acceptable, do NOT replace them with skeletons. Only fix pages that return null or blank.
    IMPORTANT: Do not refactor pages beyond fixing the specific state issue. Preserve all existing logic.
  </action>
  <verify>npx tsc --noEmit 2>&1 | tail -5</verify>
  <done>All 8 pages audited. Any page with null-return during loading or blank empty state now shows appropriate feedback. TypeScript compiles clean.</done>
