# Plan 79-02: MRR from Stripe API

## objective

Replace the hardcoded `mrr = 0` in `/api/admin/platform-stats` with a real MRR calculation sourced from the Stripe Subscriptions API. The current implementation has a TODO comment: "MRR estimate: $0 until Stripe amount field is added to schema (SYN-future)". Rather than adding a column to the Prisma schema and keeping it in sync (which creates drift risk), this plan fetches live MRR directly from Stripe, cached for 5 minutes to avoid rate limit pressure.

## context

### Discovery Findings

**Gap**: The `Subscription` Prisma model stores `plan` (free/professional/business/custom) and `stripePriceId` but NOT the actual dollar amount. Adding an `amount` column would require schema migration, webhook sync on every price change, and creates a single-source-of-truth problem (Stripe is the source of truth for pricing).

**Alternative approach**: Query Stripe directly for active subscriptions and sum their amounts. The Stripe `subscriptions.list` endpoint returns `items.data[0].price.unit_amount` (in cents). Divide by 100 for AUD dollars.

**Stripe API consideration**: `stripe.subscriptions.list({ status: 'active' })` returns paginated results. For early-stage (< 100 subscribers), a single page suffices. For scale, we paginate with `auto_paging_each_async`.

**Caching**: Stripe recommends not calling their API on every page load. Cache the MRR result in a module-level variable with a 5-minute TTL. The Platform Health tab already refreshes every 60 seconds via SWR `refreshInterval`, so a 5-minute server-side cache is appropriate.

### Architecture Decision

- Fetch MRR from Stripe API, not from the database
- Cache result server-side with 5-minute TTL (simple in-memory cache, appropriate for serverless with ~5-min function lifetime)
- Graceful degradation: if Stripe is not configured (`stripe === null`), fall back to plan-based estimate using `PRODUCTS` config prices
- Add plan-based MRR as a separate field so the UI can show both "Stripe MRR" and "Estimated MRR" (useful before Stripe is fully configured)

## tasks

### Task 1: Create MRR calculation service -- `lib/admin/mrr-calculator.ts`

**Purpose**: Encapsulate MRR calculation logic with caching.

```ts
import { stripe } from '@/lib/stripe/config';
import { prisma } from '@/lib/prisma';
import { PRODUCTS } from '@/lib/stripe/config';

interface MRRResult {
  stripeMrr: number;      // Real MRR from Stripe API (AUD cents -> dollars)
  estimatedMrr: number;   // Plan-based estimate from DB
  currency: string;       // 'aud'
  activeCount: number;    // Number of active subscriptions from Stripe
  calculatedAt: string;   // ISO timestamp
}

// In-memory cache with TTL
let cachedResult: MRRResult | null = null;
let cachedAt = 0;
const CACHE_TTL_MS = 5 * 60 * 1000; // 5 minutes

export async function calculateMRR(): Promise<MRRResult> {
  const now = Date.now();
  if (cachedResult && now - cachedAt < CACHE_TTL_MS) {
    return cachedResult;
  }

  let stripeMrr = 0;
  let activeCount = 0;

  // 1. Real Stripe MRR (if configured)
  if (stripe) {
    try {
      const subscriptions = await stripe.subscriptions.list({
        status: 'active',
        limit: 100,
        expand: ['data.items.data.price'],
      });

      for (const sub of subscriptions.data) {
        for (const item of sub.items.data) {
          if (item.price?.unit_amount) {
            // Stripe amounts are in cents
            // For yearly plans, divide by 12 to get monthly
            const interval = item.price.recurring?.interval;
            const amount = item.price.unit_amount / 100;
            if (interval === 'year') {
              stripeMrr += amount / 12;
            } else {
              stripeMrr += amount;
            }
          }
        }
        activeCount++;
      }

      // Handle pagination if > 100 subscriptions
      if (subscriptions.has_more) {
        let lastId = subscriptions.data[subscriptions.data.length - 1]?.id;
        while (lastId) {
          const more = await stripe.subscriptions.list({
            status: 'active',
            limit: 100,
            starting_after: lastId,
            expand: ['data.items.data.price'],
          });
          for (const sub of more.data) {
            for (const item of sub.items.data) {
              if (item.price?.unit_amount) {
                const interval = item.price.recurring?.interval;
                const amount = item.price.unit_amount / 100;
                if (interval === 'year') {
                  stripeMrr += amount / 12;
                } else {
                  stripeMrr += amount;
                }
              }
            }
            activeCount++;
          }
          lastId = more.has_more
            ? more.data[more.data.length - 1]?.id
            : undefined;
        }
      }
    } catch (error) {
      console.error('Failed to fetch Stripe MRR:', error);
      // Fall through to estimate
    }
  }

  // 2. Plan-based estimate from DB (fallback or comparison)
  const planCounts = await prisma.subscription.groupBy({
    by: ['plan'],
    where: { status: 'active' },
    _count: { plan: true },
  });

  const PLAN_PRICES: Record<string, number> = {
    pro: PRODUCTS.pro.price,
    professional: PRODUCTS.pro.price,
    growth: PRODUCTS.growth.price,
    business: PRODUCTS.growth.price,
    scale: PRODUCTS.scale.price,
    custom: PRODUCTS.scale.price,
  };

  let estimatedMrr = 0;
  for (const row of planCounts) {
    const price = PLAN_PRICES[row.plan] ?? 0;
    estimatedMrr += price * row._count.plan;
  }

  const result: MRRResult = {
    stripeMrr: Math.round(stripeMrr * 100) / 100,
    estimatedMrr: Math.round(estimatedMrr * 100) / 100,
    currency: 'aud',
    activeCount,
    calculatedAt: new Date().toISOString(),
  };

  cachedResult = result;
  cachedAt = now;
  return result;
}
```

### Task 2: Update `/api/admin/platform-stats` to use MRR calculator

```ts
// Inside the GET handler, after the Promise.all for counts:
const mrrResult = await calculateMRR();

return NextResponse.json({
  success: true,
  data: {
    totalUsers,
    activeSubscriptions,
    trialingSubscriptions,
    cancelledSubscriptions,
    freeUsers: Math.max(0, freeUsers),
    mrr: mrrResult.stripeMrr || mrrResult.estimatedMrr,
    mrrDetails: {
      stripeMrr: mrrResult.stripeMrr,
      estimatedMrr: mrrResult.estimatedMrr,
      currency: mrrResult.currency,
      stripeActiveCount: mrrResult.activeCount,
      calculatedAt: mrrResult.calculatedAt,
    },
  },
});
```
