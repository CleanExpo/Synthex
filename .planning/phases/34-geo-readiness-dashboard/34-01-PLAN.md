  <action>
Create lib/seo/geo-readiness-service.ts that wraps the existing lib/geo/ engine into dashboard-ready functions:

1. `analyzeReadiness(contentText: string, contentUrl?: string, platform?: string)` — Calls analyzeGEO from lib/geo/geo-analyzer.ts, then enriches the result with:
   - A readiness tier: 'ready' (>=80), 'almost' (>=60), 'needs-work' (>=40), 'not-ready' (<40)
   - A readiness summary string per dimension (e.g., "Your citability score is strong at 85/100")
   - Platform-specific readiness flags (boolean per platform: google_aio, chatgpt, perplexity, bing_copilot — ready if platform score >= 60)

2. `getAnalysisHistory(userId: string, limit?: number)` — Queries GEOAnalysis table via Prisma, returns recent analyses ordered by createdAt DESC. Default limit 20. Returns: id, contentUrl, platform, overallScore, citabilityScore, structureScore, multiModalScore, authorityScore, technicalScore, createdAt. Do NOT return contentText (too large for list views).

3. `getScoreTrends(userId: string, days?: number)` — Queries GEOAnalysis table, groups by date (createdAt), calculates daily average scores for overall, citability, structure, multiModal, authority, technical. Default 30 days. Returns array of { date: string, overall: number, citability: number, structure: number, multiModal: number, authority: number, technical: number }.

For demo data fallback: If no GEOAnalysis records exist for the user, return demo data with realistic scores. Follow the pattern from lib/seo/pagespeed-service.ts.

Import analyzeGEO from '@/lib/geo/geo-analyzer'. Import prisma from '@/lib/prisma'.
Do NOT duplicate any logic from lib/geo/ — this service is a thin wrapper that adds readiness assessment on top.

   - Calls analyzeReadiness from geo-readiness-service
   - Stores result in GEOAnalysis table via prisma.gEOAnalysis.create (same as existing /api/geo/analyze but uses the readiness wrapper)
   - Returns the readiness-enriched result

   - Auth: getUserIdFromRequest(request), return 401 if not authenticated
   - Query param: limit (optional, default 20, max 100)
   - Calls getAnalysisHistory from geo-readiness-service
   - Returns { analyses: [...], total: number }
