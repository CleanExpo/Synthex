# Phase 63-01 Plan: Parallel Agent Execution

## Goal
Enable N workflow executions simultaneously via BullMQ concurrency control.
Partial success model — one failing execution does not block others.

## Architecture
- BullMQ worker already has `concurrency: 5` (verified in Phase 62)
- Add `WORKFLOW_PARALLEL` queue name alongside existing `WORKFLOW_STEPS`
- `parallel-executor.ts` uses `Promise.allSettled` for partial success
- `batchId` field added to `WorkflowExecution` model
- New batch API routes and `ParallelExecutionWidget` dashboard component
