<task id="1" name="Create components/voice/VoiceFingerprintCard.tsx">
Displays a VoiceFingerprint with visual metric bars and trait badges.

Layout:
- Header: "Voice Fingerprint" title + word count badge
- Metrics grid (2-col on desktop, 1-col mobile):
  - Sentence Rhythm: mean ± stdDev words, with a visual distribution hint
  - Vocabulary Diversity: TTR as % with label (High/Medium/Accessible)
  - Reading Level: Flesch score with grade description
    (90-100: 5th grade, 70-90: 8th grade, 60-70: 10th grade, 50-60: College student, < 50: College graduate)
  - First Person Rate: % with label (Personal/Institutional)
  - Question Rate: per 100 words
  - Em Dash Usage: per 100 words with "High/Low editorial style" label
  - Adverb Density: per 100 words with "Lean/Modifier-heavy" label
  - Passive Voice: % with "Active/Passive tendency" label
- Trait badges row: auto-generated from fingerprint values
  Examples: "Conversational", "Academic", "Varied Rhythm", "First-person", "Punchy"

Props: { fingerprint: VoiceFingerprint; className?: string }
</task>

<task id="2" name="Create components/voice/SlopScanResults.tsx">
Displays SlopScanResult with match list and severity indicators.

Layout:
- Header bar: overall severity badge (Clean / Warning / Error) + counts
- Slop density: "X matches per 100 words" with colour coding
- If clean: green checkmark + "No AI tell-phrases detected"
- If matches: scrollable list grouped by category:
  - Category header (Transition, Filler, Overused Word, etc.)
  - Each match: the phrase in a highlighted chip + severity dot + optional suggestion
- Footer: "Powered by SLOP_PATTERNS_V1" (versioning for user trust)

Severity colours:
- error: red-500/20 bg, red-400 text
- warning: amber-500/20 bg, amber-400 text
- clean: emerald-500/20 bg, emerald-400 text

Props: { result: SlopScanResult; className?: string }
</task>

<task id="3" name="Create components/voice/ContentCapsulePreview.tsx">
Side-by-side or before/after view of original content vs capsule format.

Layout:
- Extractability score prominently at top: large number + label + color ring
  (0-39: Poor, 40-59: Fair, 60-79: Good, 80-100: Excellent)
- Toggle: "Original" | "Capsule Format" (tab or toggle)
- Original view: renders the original text with word count
- Capsule view: renders each CapsuleSection:
  - Heading (H2 style, question-format badge if ends with ?)
  - Answer block (styled prose, word count badge showing 75-120 ✓ or actual count)
  - Supporting bullets
  - isCapsuleReady badge (green "Ready" or amber "Needs Refinement")
- Summary footer: "X of Y sections are capsule-ready"

Props: { result: ContentCapsuleResult; originalText: string; className?: string }
</task>

<task id="4" name="Create app/dashboard/voice/page.tsx">
Voice dashboard page — tabbed: Fingerprint | Capsule | Slop Scan

Server component wrapper (thin, passes auth check).
Client component: VoiceDashboardClient (use client).

Layout:
- PageHeader: title "Voice Engine", subtitle "Fingerprint your writing style, eliminate AI slop, format content for AI citation"
- Three tabs: "Fingerprint" | "Content Capsule" | "Slop Scan"

Fingerprint Tab:
- Left panel: Textarea for sample text (min 200 words enforced with live counter)
  - Character/word count below textarea: "X words — {minimum 200 required | ready to analyse}"
  - "Save Profile" toggle + name input (for plan-gated saving)
  - "Analyse" button (disabled until 200+ words)
- Right panel: VoiceFingerprintCard (shows after analysis) or EmptyState
- Below right panel: WritingContext copyable block (if fingerprint exists)
  - "Copy Writing Context" button → copies systemPrompt to clipboard

Content Capsule Tab:
- Left panel: Textarea for content to format
  - "Format as Capsule" button
- Right panel: ContentCapsulePreview (shows after formatting) or EmptyState

Slop Scan Tab:
- Full-width: Textarea for content to scan
  - "Scan for AI Slop" button
- Results: SlopScanResults component below

State management: separate useState for each tab's input/result.
API calls: useApi() hook for POST requests.
</task>
