<task id="1" name="Create lib/voice/capsule-formatter.ts">
Content Capsule Technique implementation. Transform arbitrary content into AI-extractable format.

Algorithm:
1. Split content into sections by H2/H3 headings (## or ###)
2. For each section:
   a. Extract heading text — check if it's already a question (ends with ?)
   b. Count words in section body
   c. isCapsuleReady = wordCount >= 75 && wordCount <= 120
   d. Extract bullet points (lines starting with - or *)
   e. The "answerBlock" is the first non-bullet paragraph, trimmed
3. Score extractability:
   - +20 pts if ≥50% of headings are question-format
   - +20 pts if ≥50% of sections are 75-120 words
   - +20 pts if ≥30% of sections have bullet evidence
   - +20 pts if first section has a direct answer (short first sentence ≤20 words)
   - +20 pts if headings contain question words (what, how, why, when, where, which)
4. Export formatAsCapsule(text: string): ContentCapsuleResult

Content type detection (to avoid applying capsule to narrative/opinion):
- Count headings — if < 1 heading per 300 words, content may not be informational
- If headings < 2 total, return capsule with isAlreadyCapsuleReady=false and sections=[]
  (signal to UI that this content type may not benefit from capsule format)
</task>

<task id="2" name="Create lib/voice/context-builder.ts">
Build a rich writing context system prompt from a VoiceFingerprint.
Export buildWritingContext(fingerprint: VoiceFingerprint, compact?: boolean): WritingContextResult.

Full context (compact=false, target ~400 tokens):
```
WRITING STYLE REQUIREMENTS — MATCH EXACTLY:

Sentence rhythm: Average {mean} words per sentence (range {mean-stdDev}-{mean+stdDev} words).
Vary sentence length naturally — do not use uniform length.

Vocabulary level: {High|Medium|Accessible} diversity (TTR {ttr}).
{Use sophisticated vocabulary.|Prefer simple, direct words.}

Readability target: Flesch Reading Ease {score} ({grade-level description}).

Voice characteristics:
{conditional bullets based on fingerprint values}
- Write in first person where appropriate. [if firstPersonRate > 0.02]
- Use em dashes for parenthetical asides — it is part of this voice. [if emDashRate > 0.5]
- Include rhetorical questions to engage the reader. [if questionRate > 0.5]
- Write lean: avoid unnecessary adverbs. [if adverbDensity < 0.03]
- Favour shorter paragraphs: average {avgParagraphLength} words per paragraph. [if < 80]
- Use longer, developed paragraphs: average {avgParagraphLength} words per paragraph. [if > 150]

PROHIBITED (AI slop — never use):
delve, comprehensive, robust, leverage, foster, enhance, dynamic, innovative,
"in conclusion", "it's important to note", "it's worth noting", "needless to say",
"in today's fast-paced world", "as we navigate", "tapestry", "nuance"
```

Compact context (compact=true, target ~150 tokens):
Single-line summary of key metrics + short prohibited list.

Token estimate: character_count / 4 (rough approximation).
Cap full context at 500 tokens — if estimate > 500, auto-switch to compact.
</task>
